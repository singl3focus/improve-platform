# Доменные сущности и бизнес-объекты

## Общая модель продукта
- Продукт — это личная обучающая платформа, объединяющая:
  - roadmap развития,
  - отслеживание прогресса,
  - материалы для обучения,
  - задачи,
  - уведомления по дедлайнам.
- MVP ориентирован только на личное использование.
- Каждый пользователь работает только со своими данными.
- Shared-доступ и совместная работа не входят в MVP.

## Roadmap
- У пользователя одна основная roadmap.
- Roadmap не является линейным списком.
- Логическая модель roadmap — направленный ациклический граф (DAG).
- Узлы графа — темы.
- Между темами могут существовать зависимости.
- Тему нельзя начать, если не завершены все ее зависимости.
- Roadmap используется для развития, навигации по обучению и отслеживания прогресса.

## Этапы
- Этап не является самостоятельной бизнес-сущностью roadmap.
- Этап существует только как UI-группа для объединения тем.
- Этап имеет:
  - название
- Этап не имеет:
  - собственной бизнес-логики,
  - дат,
  - отдельного жизненного цикла.
- Этап не может быть пустым.
- Тему можно переносить между этапами.

## Темы
- Тема — основная единица прогресса в системе.
- Каждая тема всегда принадлежит одному этапу.
- Тема может содержать:
  - название,
  - описание,
  - статус,
  - даты,
  - чеклист,
  - материалы,
  - зависимости от других тем.
- Прогресс считается только на уровне темы.
- Прогресс темы вычисляется автоматически.
- Ручное выставление прогресса темы не допускается.

## Статусы темы
- Для темы фиксируются следующие статусы:
  - не начато
  - в работе
  - на паузе
  - завершено
- Статус `завершено` должен устанавливаться автоматически, когда выполнены все checklist items темы.
- Тему нельзя перевести в активную работу, если зависимости темы не завершены.

## Чеклист темы
- У темы обязательно может быть чеклист.
- Чеклист влияет на прогресс темы.
- Чеклист состоит из отдельных пунктов.
- Пункты чеклиста фактически являются задачами.
- Отдельная сущность DoD не используется.
- Завершенность темы определяется через выполнение всех пунктов чеклиста.

## Прогресс темы
- Прогресс темы рассчитывается автоматически на основе выполнения checklist items.
- Ручной ввод процента прогресса темы не допускается.
- Если все checklist items выполнены, тема считается завершенной автоматически.

## Даты темы
- У темы могут быть:
  - дата начала,
  - целевая дата,
  - дата завершения.
- У этапа дат нет.

## Задачи
- Задача является отдельной сущностью.
- Задача может существовать:
  - как самостоятельная задача,
  - как пункт чеклиста внутри темы.
- Для MVP пункты чеклиста и задачи должны использовать единый поток и максимально общую модель.
- Задачи нужны как для поддержки roadmap, так и для отдельного kanban-флоу.

## Канбан задач
- Задачи визуализируются как kanban.
- Статусы задач:
  - новая
  - в работе
  - на паузе
  - выполнена
- Статус `просрочена` не хранится как отдельный статус.
- Просрочка вычисляется автоматически по дедлайну и отображается визуально.

## Дедлайны
- Дедлайн есть у задачи.
- Дедлайн не является обязательным для темы в рамках логики уведомлений.
- Уведомления строятся вокруг задач и их дедлайнов.

## Уведомления
- Рядом с backend должен работать Telegram-бот.
- Telegram-бот нужен только для уведомлений.
- Управление задачами и прогрессом через Telegram-бот в MVP не требуется.
- Формат уведомлений:
  - один раз в день утром,
  - ежедневная сводка на день,
  - перечисление того, что предстоит сделать по дедлайнам.

## Материалы
- Материал — отдельная сущность.
- Материал может существовать сам по себе.
- Материал привязывается только к одной теме.
- Одна тема может иметь много материалов.
- Один материал может относиться только к одной теме.
- Для материалов нужен порядок отображения внутри темы.

## Поля материала
- Для уникальности и гибкости в MVP материалы не привязываются к жестким типам.
- Базовые поля материала:
  - название
  - описание
- В описании может содержаться ссылка, комментарий, источник и любая дополнительная информация.

## Прогресс по материалам
- У материала есть собственный прогресс.
- Прогресс материала выставляется вручную.
- Прогресс материала и прогресс темы — разные сущности.
- Прогресс темы не должен напрямую подменяться ручным прогрессом материала.

## Зависимости между темами
- Зависимости между темами обязательны для MVP.
- Roadmap должна поддерживать DAG-модель.
- У темы могут быть prerequisite topics.
- Начало темы запрещено, если ее зависимости не завершены.

## История изменений
- История изменений обязательна.
- Нужны два типа истории:
  - техническая история,
  - бизнес-история.
- Техническая история включает фиксацию того, кто и когда изменил данные.
- Бизнес-история включает события предметной области, например:
  - изменение статуса темы,
  - изменение прогресса,
  - выполнение checklist item,
  - перенос темы между этапами,
  - изменение дедлайна задачи.

## Пользовательская модель
- В MVP пользователь — обычный user без ролей.
- Система мультипользовательская на уровне архитектуры.
- Каждый пользователь видит только свои roadmap, темы, задачи и материалы.

## Удаление данных
- Сущности удаляются физически.
- Soft delete и архивирование в MVP не используются.
- При этом история изменений как отдельный механизм должна сохраняться по правилам реализации.

# Ключевые бизнес-правила и инварианты

- У пользователя одна roadmap.
- Roadmap представляет собой DAG из тем.
- Циклы в зависимостях тем запрещены.
- Тема всегда принадлежит одному этапу.
- Этап является только UI-группой и не существует без тем.
- Тему нельзя перевести в "в работе", если ее зависимости не завершены.
- Прогресс темы считается только по checklist items.
- Ручное изменение прогресса темы запрещено.
- Если все checklist items темы выполнены, тема автоматически получает статус "завершено".
- Задача может быть самостоятельной или являться checklist item темы.
- Дедлайн влияет только на задачи.
- Просрочка задачи вычисляется динамически и не хранится отдельным статусом.
- Материал принадлежит только одной теме.
- Прогресс материала задается вручную и не определяет прогресс темы.
- Все данные пользователя изолированы и недоступны другим пользователям.
