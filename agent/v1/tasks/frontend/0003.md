### Change Summary
**Problem**
- После первого ревью в dashboard оставались незакрытые `empty`-состояния в hero-блоках `daily summary` и `roadmap progress`: при `mode=empty` рендер был неинформативным.

**Root Cause Hypothesis**
- В `frontend/src/components/dashboard-view.tsx` были реализованы `loading/error/success`, но не было явной ветки `empty` для `daily summary` и `progress`.
- Для `progress` отображалась строка `0% complete (0/0 topics)` без отдельного UX-состояния пустого roadmap.

**Evidence**
- Изменения внесены в `frontend/src/components/dashboard-view.tsx`:
  - добавлены `hasDailySummaryContent` и `hasRoadmapProgress`;
  - добавлены явные сообщения `DashboardEmpty` для `daily summary` и `roadmap progress`.
- Поиск по файлу подтверждает новые empty-ветки:
  - `Daily summary is empty...`
  - `Roadmap progress is empty...`
- API empty-режимы для hero-блоков подтверждены командами:
  - `GET /api/dashboard/daily-summary?mode=empty` → `200`
  - `GET /api/dashboard/progress?mode=empty` → `200`

**Solution**
- В `daily summary` добавлена явная `empty`-ветка: если `headline` и `todayFocus` пустые, показывается информативный `DashboardEmpty`.
- В `roadmap progress` добавлена явная `empty`-ветка: при `totalTopics === 0` показывается `DashboardEmpty`, а прогресс-бар не рендерится.
- Текст заголовка прогресса для пустого roadmap изменён с `0% complete (0/0 topics)` на `No roadmap topics yet.`.
- Обработка `loading/error` и кликабельность hero-карточек сохранены без изменения поведения.

**Tests / Verification**
- Запущено:
  - `npm run lint` — PASS.
  - `npm run build` — PASS (`Compiled successfully`, `Generating static pages (23/23)`).
  - `Invoke-WebRequest -UseBasicParsing /api/dashboard/daily-summary?mode=empty` — PASS (`200`).
  - `Invoke-WebRequest -UseBasicParsing /api/dashboard/progress?mode=empty` — PASS (`200`).
  - Статическая проверка `dashboard-view.tsx` на наличие explicit empty-веток в hero-блоках — PASS.
- Не запускалось локально:
  - Полный ручной UI-check в браузере на `/dashboard` под авторизованной сессией.

**Impact**
- UI/UX: hero-блоки dashboard теперь явно обрабатывают `empty` для `daily summary` и `roadmap progress`.
- API/данные: контракт API не менялся.
- Безопасность: изменений нет.
- Обратная совместимость: да, изменение локально в UI-логике dashboard.

**Risks & Rollback**
- Риск: тексты empty-состояний на английском, может потребоваться локализация.
- Риск: без ручного браузерного прогона не подтвержден визуальный вид empty-блоков в реальном UI.
- Rollback plan:
  - откатить изменения в `frontend/src/components/dashboard-view.tsx` (удалить helper-функции и empty-ветки hero-блоков).
