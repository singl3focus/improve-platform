## TSK-0054 — Перевести roadmap в строгий top-down граф без zoom/pan контролов
- Status: Finished
- Type: Business
- Approval: Approved

### Context
- Owner требует видеть roadmap как полноценный вертикальный граф (сверху вниз), а не как свободную сцену с управлением камерой.
- Текущая реализация использует свободную сцену с `zoom in / zoom out / recenter` и гибким wrap-расположением карточек, что не соответствует целевому UX.

### Acceptance Criteria
- [x] На `/roadmap` граф отображается строго сверху вниз (top-down), с читаемой иерархией уровней.
- [x] Расстояние между уровнями и между соседними нодами фиксированное и предсказуемое (не плавающее от ширины контейнера).
- [x] Контролы `zoom in`, `zoom out`, `recenter` отсутствуют в UI.
- [x] Навигация кликом по теме, редактирование темы, создание/удаление зависимостей продолжают работать.
- [x] На mobile используется корректная адаптация top-down представления без возврата к старым колонкам/stage-lanes.

### Constraints
- Не менять backend API в рамках этой задачи.
- Не возвращать stage-lane/колоночный layout.
- Не внедрять в этой задаче новый UX создания темы через контекст-ноду (выносится в отдельную задачу).

### Change Summary
**Problem**
- Roadmap-экран оставался сценой со свободной камерой и кнопками `zoom in / zoom out / recenter`, тогда как требовался строгий top-down граф с фиксируемым расстоянием между уровнями и нодами.

**Root Cause Hypothesis**
- Позиционирование нод было реализовано через `flex-wrap` без явной привязки к оси уровней.
- Логика viewport (`scale + offset`) и отдельные контролы камеры были встроены прямо в `RoadmapView`.
- Геометрия стрелок была ориентирована на горизонтальную схему (из правой грани в левую), а не на вертикальный поток сверху вниз.

**Evidence**
- `frontend/src/components/roadmap-view.tsx`: обработчики `zoomGraph`, `handleGraphWheel`, `handleGraphPointer*`, `sceneTransform` и блок `roadmap-graph-controls`.
- `frontend/app/globals.css`: `roadmap-graph-controls` и `roadmap-graph-nodes` с `display: flex; flex-wrap: wrap;`.
- `frontend/src/components/roadmap/use-roadmap-graph-layout.ts`: старые якоря связей по горизонтали (right -> left).

**Solution**
- Удалена UI-часть контролов камеры и обработчики pan/wheel/zoom из `RoadmapView`.
- Переключено позиционирование тем на фиксированный top-down grid:
  - `gridRow = позиция этапа`,
  - `gridColumn = позиция темы внутри этапа`,
  - фиксированные `row-gap` и `column-gap`.
- Пересчитаны якоря стрелок на вертикальный поток (низ родителя -> верх потомка).
- Обновлена функция `buildConnectionPath` под вертикальную bezier-кривую.
- Адаптирован unit-тест `buildConnectionPath` под новое поведение.
