## TSK-0027 — Изменение прогресса материала через ползунок
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Сейчас прогресс материала меняется через `select` с фиксированными значениями.
- По требованию owner прогресс материала должен меняться ползунком.
- Текущее решение с `select` ухудшает UX и точность настройки прогресса.

### Acceptance Criteria
- [x] В карточке материала на `/materials` управление прогрессом заменено с `select` на `input[type=range]`.
- [x] Ползунок поддерживает диапазон `0..100` и показывает текущее значение в процентах рядом с контролом.
- [x] Обновление прогресса сохраняется через текущий BFF API (`PATCH /api/materials/{materialId}`).
- [x] При ошибке сохранения прогресса значение в UI откатывается и пользователь видит сообщение об ошибке.
- [x] Ползунок доступен с клавиатуры (tab/focus/arrow keys) и корректно работает на desktop/mobile.

### Constraints
- Не менять backend-контракт материалов.
- Не ломать существующие сценарии создания/редактирования/удаления материалов.
- Не превращать задачу в редизайн всей страницы библиотеки.

### Change Summary
**Problem**
- В карточке материала прогресс изменялся только через `select` с фиксированными шагами, что ограничивало точность и ухудшало UX.

**Root Cause Hypothesis**
- Контрол обновления прогресса был реализован как выпадающий список вместо диапазонного ползунка.
- Локальный UI state не отражал промежуточное изменение прогресса с явной стратегией отката при ошибке запроса.

**Evidence**
- `frontend/src/components/materials-library-view.tsx`: блок `materials-progress-select-wrap` использовал `<select>` с шагом 10%.
- `frontend/src/components/hooks/use-materials-library-view-model.ts`: `handleProgressUpdate` отправлял PATCH без optimistic update/rollback в query cache.
- Проверки:
  - `cd frontend && npm run lint` → PASS (`next lint`: No ESLint warnings or errors)
  - `cd frontend && npx tsc --noEmit` → PASS (exit code 0)

**Solution**
- В `frontend/src/components/materials-library-view.tsx` заменён `select` на `input[type=range]` с `min=0`, `max=100`, `step=1`.
- Рядом с ползунком добавлен текущий процент (`strong` в `materials-progress-range-head`).
- В `frontend/src/components/hooks/use-materials-library-view-model.ts` добавлен optimistic update текущего query cache перед PATCH.
- При ошибке PATCH добавлен rollback на предыдущее состояние cache + вывод сообщения ошибки через существующий `mutationError`.
- В `frontend/app/globals.css` добавлены/обновлены стили для `materials-progress-range-*` без изменения остального layout.

**Tests / Verification**
- Запущено:
  - `cd frontend && npm run lint` — PASS
  - `cd frontend && npx tsc --noEmit` — PASS
- Не запускалось локально:
  - ручная проверка в браузере (desktop/mobile, keyboard navigation, error rollback)

**Impact**
- Влияет на UI карточки материалов на `/materials` и UX изменения прогресса.
- API контракт не изменён: используется текущий PATCH `/api/materials/{materialId}`.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: при частых изменениях ползунка возможна серия PATCH-запросов.
- Откат: вернуть изменения в `frontend/src/components/materials-library-view.tsx`, `frontend/src/components/hooks/use-materials-library-view-model.ts`, `frontend/app/globals.css`.

**Suggested Commit Message**
