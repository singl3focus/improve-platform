## TSK-0004 — Экран Roadmap с визуализацией DAG-зависимостей
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-14
- Updated: 2026-03-14

### Context
- Roadmap — ключевой экран продукта и должен показывать зависимости тем как понятную карту.
- Пользователь должен сразу видеть блокировки и путь прохождения.
- Для визуального направления использовать `design/_2/code.html` как референс структуры и подачи DAG.

### Acceptance Criteria
- [x] Экран отображает темы, сгруппированные по этапам, с визуальными связями зависимостей.
- [x] Для темы показаны: статус, прогресс, признак блокировки, счетчики задач/материалов.
- [x] Заблокированные темы визуально отличаются и содержат понятную причину блокировки.
- [x] Навигация из карточки темы открывает экран соответствующей темы.
- [x] Компоновка roadmap (этапы, карточки, связи, легенда статусов) визуально согласована с `design/_2/code.html` без обязательного полного совпадения по пикселям.

### Constraints
- Этапы в UI-группировке без отдельной бизнес-логики на клиенте.
- Визуализация должна оставаться читаемой при росте количества тем.
- Не использовать тяжелый графовый редактор в MVP.
- `design/_2/code.html` — только визуальный референс; функциональные требования берутся из задачи и `docs/bussines/v1/domain.md`.
- Не переносить в MVP нерелевантные элементы из референса (share/community/upgrade и т.п.), если их нет в задачах.

### Change Summary
**Problem**
- На `/roadmap` была только заглушка без DAG-визуализации, статусов тем, блокировок и навигации по темам.
- После первого ревью в задаче оставался незакрытый ReMake-пункт: AC5 был подтвержден только статикой кода, без runtime-доказательства визуальной согласованности на desktop/mobile.

**Root Cause Hypothesis**
- Для roadmap не был реализован отдельный экран и источник данных; отсутствовали UI-компонент с зависимостями и API payload для тем/этапов.
- В глобальных стилях не было набора классов для stage-компоновки, карточек тем, blocked-вариантов и легенды статусов.
- Верификация до ReMake не включала runtime UI-smoke сценарий `/roadmap` с проверкой визуальной компоновки и кликабельной навигации карточек.

**Evidence**
- До изменений `frontend/app/(private)/roadmap/page.tsx` рендерил только placeholder-блок.
- Реализация roadmap-view и навигации: `frontend/src/components/roadmap-view.tsx` (ключевые места: `RoadmapView`, `roadmap-connections`, blocked-card, `router.push('/topics?topicId=...')`).
- Данные roadmap с этапами/зависимостями/blocked-причинами добавлены в `frontend/app/api/roadmap/route.ts`.
- Стили для этапов, карточек, линий зависимостей и легенды добавлены в `frontend/app/globals.css`.
- Runtime smoke подтверждает режимы API: `default=200`, `empty=200`, `error=500`.
- ReMake-проверка выполнена runtime UI-smoke на `/roadmap` (desktop + mobile) с результатом:
  - `desktop`: `stages=3`, `connections=9`, `legendVisible=true`, `blockedReasonVisible=true`, переход по клику `-> /topics?topicId=topic-cli-basics`;
  - `mobile`: `stages=3`, `roadmap-connections display=none`, `legendVisible=true`, `blockedReasonVisible=true`, переход по клику `-> /topics?topicId=topic-cli-basics`.

**Solution**
- Добавлен клиентский компонент `RoadmapView` с загрузкой данных из `/api/roadmap` и обработкой состояний `loading/error/empty/success`.
- Реализована визуализация этапов roadmap:
  - группировка тем по stage-колонкам;
  - SVG-линии зависимостей между карточками на desktop;
  - legend-блок со счетчиками статусов.
- Для карточки темы добавлены:
  - статус, прогресс-бар, счетчики задач и материалов;
  - blocked-индикатор и текст причины блокировки;
  - список prerequisites;
  - навигация на соответствующий topic-контекст (`/topics?topicId=...`).
- Добавлен mock endpoint `/api/roadmap` с payload этапов и зависимостей, плюс режимы `?mode=empty` и `?mode=error`.
- Страница `app/(private)/roadmap/page.tsx` переведена с заглушки на `RoadmapView`.
- Для ReMake выполнена дополнительная runtime-верификация AC5 (desktop/mobile + click-navigation) в браузерном сценарии с mock auth backend; код roadmap-модуля не менялся.

**Tests / Verification**
- Запущено:
  - `npm run lint` — PASS.
  - `npm run build` — PASS (после повторного запуска вне sandbox из-за `spawn EPERM`).
  - Runtime smoke `/api/roadmap` (`default/empty/error`) под `next start` — PASS (`200/200/500`).
  - Статическая проверка ключевых roadmap-веток в компоненте (`rg`) — PASS.
  - Runtime UI smoke `/roadmap` (desktop + mobile + click-navigation на `/topics?topicId=...`) — PASS:
    - `desktop`: `stages=3`, `connections=9`, `legendVisible=true`, `blockedReasonVisible=true`, navigation PASS.
    - `mobile`: `stages=3`, `roadmap-connections display=none`, `legendVisible=true`, `blockedReasonVisible=true`, navigation PASS.
- Не запускалось локально:
  - Отдельный ручной визуальный проход через GUI-браузер (проверка выполнена headless smoke-сценарием).

**Impact**
- API: добавлен новый frontend endpoint `/api/roadmap` для roadmap-данных и режимов smoke-проверки.
- UI: вместо placeholder появился полноценный roadmap-экран с DAG-связями, blocked-индикацией и переходами в topic-контекст.
- Безопасность/данные: критичных изменений безопасности и миграций данных нет.
- Обратная совместимость: да, изменения изолированы в frontend roadmap-модуле.
- ReMake (текущий проход): функциональных изменений кода нет, обновлена только проверка и документирование AC5.

**Risks & Rollback**
- Риск: текущее отображение зависимостей использует client-side вычисление координат карточек; при сильных изменениях layout возможна визуальная деградация линий.
- Риск: `RoadmapView` пока использует mock API (`/api/roadmap`), реальный backend-контракт может отличаться.
- Риск текущего ReMake-шага: ручной GUI-smoke не выполнялся, визуальная часть подтверждена headless runtime проверкой.
- Rollback plan:
  - вернуть `frontend/app/(private)/roadmap/page.tsx` к placeholder-версии;
  - удалить `frontend/src/components/roadmap-view.tsx`, `frontend/src/lib/roadmap-types.ts`, `frontend/app/api/roadmap/route.ts`;
  - убрать roadmap-стили из `frontend/app/globals.css`.
