## TSK-0021 — Перенос карточек задач между колонками через drag-and-drop (desktop)
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-15
- Updated: 2026-03-15

### Context
- На канбан-доске `/tasks` статус карточки меняется через выпадающий список, а не переносом мышкой.
- По требованию owner карточки должны перетаскиваться между колонками через drag-and-drop.
- Нужно внедрить desktop DnD-UX поверх существующего API обновления статуса.

### Acceptance Criteria
- [x] На `/tasks` карточки можно перетащить мышкой из одной колонки в другую (desktop сценарий).
- [x] Drop в колонку инициирует обновление статуса через существующий BFF endpoint смены статуса.
- [x] После drop карточка отображается в целевой колонке без перезагрузки страницы.
- [x] При ошибке обновления статуса UI откатывается к предыдущему состоянию и показывает сообщение об ошибке.
- [x] Управление через `select` не используется как основной механизм переноса карточек.

### Constraints
- Скоуп только desktop mouse DnD (без обязательной touch-поддержки).
- Не внедрять тяжелые UI-рефакторы вне канбан-доски.
- Предполагается выполнение после backend задачи `TSK-0016`.

### Change Summary
**Problem**
- На канбан-доске `/tasks` смена статуса была реализована через `select` внутри карточки, а не через drag-and-drop.
- Требовался desktop DnD-перенос между колонками с использованием существующего API смены статуса и откатом UI при ошибке.

**Root Cause Hypothesis**
- В UI-компоненте доски отсутствовали `draggable`-карточки и drop-обработчики колонок.
- Несмотря на наличие API и optimistic update в view-model, DnD-события не были подключены к `handleStatusChange`.

**Evidence**
- `frontend/src/components/tasks-kanban-view.tsx:205` — колонка стала drop-target (`onDragOver/onDragEnter/onDragLeave/onDrop`).
- `frontend/src/components/tasks-kanban-view.tsx:244` — карточки сделаны `draggable` с обработчиками `onDragStart/onDragEnd`.
- `frontend/src/components/tasks-kanban-view.tsx:433` — `handleColumnDrop` вызывает `handleStatusChange(taskId, status)` при переносе.
- `frontend/src/components/hooks/use-tasks-board-view-model.ts:340` — `handleStatusChange` используется как единая точка обновления статуса.
- `frontend/src/components/hooks/use-tasks-board-view-model.ts:349` и `:400` — сохранение `previousTasks` и rollback при ошибке.
- `frontend/scripts/mvp-verification-smoke.mjs:684` — обновлён smoke-сценарий PATCH для DnD-переноса; `:697-702` — добавлена проверка статуса задачи после повторного `GET /api/tasks`.
- Команды:
  - `npm run lint` → PASS.
  - `npm run build` → PASS (после повторного запуска вне sandbox).
  - `npm run verify:mvp` → FAIL (2 не-связанных сценария, DnD-сценарий PASS).
  - `npx tsc --noEmit` → PASS.

**Solution**
- В `TasksKanbanView` реализован desktop DnD на нативных HTML5 событиях:
  - состояние перетаскивания (`draggedTaskId`, `dragOverStatus`),
  - drag/drop-обработчики карточек и колонок,
  - вызов существующего `handleStatusChange` на drop.
- Удалён `select` как основной механизм переноса карточек; вместо него добавлена текстовая подсказка про drag-and-drop.
- Добавлена визуальная обратная связь:
  - подсветка колонки при наведении перетаскиваемой карточки,
  - стиль перетаскиваемой карточки.
- В `useTasksBoardViewModel` добавлена защита от лишнего запроса при переносе в ту же колонку (early return при неизменном статусе).
- В `verify:mvp` обновлена проверка PATCH-сценария: после изменения статуса выполняется повторный `GET /api/tasks` и проверяется сохранённый новый статус.

**Tests / Verification**

- Запущено `npm run build` — PASS (после повторного запуска вне sandbox; в sandbox был `spawn EPERM`).
