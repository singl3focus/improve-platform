## TSK-0014 — Добавить в frontend переключатели языка (RU/EN) и темы (Light/Dark)
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-15
- Updated: 2026-03-15

### Context
- Owner запросил пользовательские переключатели языка (русский/английский) и темы (светлая/темная).
- Сейчас в проекте отсутствуют механизмы i18n и dark theme: `html lang="en"`, `color-scheme: light`, settings-страница содержит только placeholder.
- Нужна управляемая настройка интерфейса с сохранением выбора между перезагрузками.

### Acceptance Criteria
- [x] В `Settings` добавлены рабочие controls: `Language: RU/EN` и `Theme: Light/Dark`.
- [x] Выбранный язык и тема сохраняются (например, localStorage/cookie) и восстанавливаются после перезагрузки.
- [x] `html[lang]` синхронизируется с выбранным языком (`ru`/`en`).
- [x] UI поддерживает оба языка минимум на auth + navigation + dashboard + settings (без смешения RU/EN в одном активном режиме).
- [x] Темная тема применяет альтернативные CSS-переменные/стили ко всем ключевым экранам (auth, dashboard, roadmap, topics, tasks, materials, settings) без проблем читаемости.

### Constraints
- Не менять backend API-контракты.
- Не оставлять частично нерабочие переключатели (только UI без реального эффекта).
- Поддержать корректный рендер на desktop и mobile в обеих темах.
- Избегать дублирования строк: использовать единый словарь/слой локализации.

### Change Summary
**Problem**
- ReMake update (2026-03-15): reviewer found RU/EN mixed content on dashboard because `daily-summary` response strings were always generated in English.
- В frontend отсутствовали реальные пользовательские переключатели языка и темы.
- `Settings` был placeholder-экраном, `html lang` был статично `en`, а UI работал только в светлой теме.

**Root Cause Hypothesis**
- ReMake update (2026-03-15): `frontend/app/api/dashboard/daily-summary/route.ts` created hardcoded English `headline/todayFocus` strings on the server instead of using localized copy from `ui-copy`.
- Не было единого слоя предпочтений пользователя (state + persistence + синхронизация с DOM).
- Тексты интерфейса в auth/navigation/dashboard/settings были захардкожены напрямую в компонентах.
- Глобальные стили не имели dark-переменных и согласованных токенов для ключевых экранов.

**Evidence**
- ReMake update (2026-03-15):
  - `frontend/app/api/dashboard/daily-summary/route.ts` no longer returns `headline/todayFocus`; it returns `nextTaskTitle` + counts only.
  - `frontend/src/components/dashboard-view.tsx` now renders summary text through localized `dashboardCopy` metrics.
  - `frontend/src/lib/dashboard-types.ts` updated to new `DashboardDailySummary` shape.
  - `rg "Today focus|Upcoming tasks" frontend/app/api/dashboard/daily-summary/route.ts` -> no matches.
- Добавлен новый provider `frontend/src/components/providers/user-preferences-provider.tsx`:
  - хранит `language/theme`,
  - читает/пишет `localStorage`,
  - синхронизирует `document.documentElement.lang` и `document.documentElement.dataset.theme`.
- Добавлен единый словарь `frontend/src/lib/ui-copy.ts` (RU/EN) для auth/navigation/dashboard/settings.
- `frontend/app/layout.tsx`: подключен `UserPreferencesProvider` глобально.
- `frontend/app/(private)/settings/page.tsx`: placeholder заменен на рабочие controls `Language` и `Theme`.
- `frontend/src/components/auth/auth-form.tsx`, `frontend/src/components/layout/private-shell.tsx`, `frontend/src/components/dashboard-view.tsx`: тексты переведены на словарь и переключаются по выбранному языку.
- `frontend/app/globals.css`: добавлены `:root[data-theme="dark"]` и альтернативные CSS-переменные/стили для auth/dashboard/roadmap/topics/tasks/materials/settings.
- Команды проверки:
  - `npm run lint` -> PASS.
  - `npm run build` -> PASS (после повторного запуска вне sandbox).

**Solution**
- ReMake update (2026-03-15):
  - Removed server-side user-facing summary strings from `daily-summary` BFF route.
  - Switched dashboard summary text composition to client-side localized copy (`ui-copy`) in `dashboard-view`.
  - Kept existing metrics/empty behavior; changed only summary payload/rendering path.
- Реализован общий клиентский слой предпочтений:
  - `language: ru|en`,
  - `theme: light|dark`,
  - persistent хранение в `localStorage`,
  - автоматическая синхронизация `html[lang]` и `html[data-theme]`.
- Внедрен единый словарь переводов и подключен в ключевые блоки:
  - auth: форма логина/регистрации и fallback auth-ошибки,
  - navigation: sidebar/header/logout,
  - dashboard: заголовки, подписи, статусы, retry/open, дата/строки без RU/EN-смешения,
  - settings: labels/options/description.
- В `Settings` добавлены рабочие select-контролы для языка и темы с мгновенным применением.
- Переведена тема на токены и добавлены dark-override’ы для читаемости на всех ключевых экранах приватной и auth-зоны.

  - Browser smoke с авторизацией и сменой настроек в `Settings` (`next start` + headless `playwright-core`) — PASS (`32/32` проверок).

  - На каждом шаге подтверждены: корректные `html[lang]`/`html[data-theme]`, сохранение `localStorage`-предпочтений, отсутствие смешения RU/EN токенов, контраст текста `>= 3.0`.

**Impact**
- ReMake update (2026-03-15):
  - Dashboard daily summary no longer forces English server text in RU mode.
  - Internal frontend BFF response shape changed from `headline/todayFocus` to `nextTaskTitle` + counts, synchronized with dashboard component.
- UI/UX: добавлены реальные пользовательские переключатели языка и темы в `Settings`.
- Локализация: auth + navigation + dashboard + settings работают через единый словарь, без дублирования строк.
- Темизация: добавлена dark theme через CSS-переменные и override-стили для ключевых экранов.
- Backend API-контракты: без изменений.
- Обратная совместимость: да; по умолчанию приложение продолжает работать в light/en до выбора пользователя.

**Risks & Rollback**
- ReMake final update (2026-03-15):
  - Risk: ручная проверка в реальном браузере на узких mobile viewport не выполнялась отдельно от headless smoke.
  - Rollback: restore previous `DashboardDailySummary` fields (`headline/todayFocus`) in `dashboard-types`, `daily-summary` route, and `dashboard-view`.
- Риск: возможен краткий initial render в дефолтной теме/языке до гидрации client-provider.
- Rollback plan:
  - убрать `UserPreferencesProvider` из `frontend/app/layout.tsx`,
  - вернуть `frontend/app/(private)/settings/page.tsx` к placeholder,
  - удалить `frontend/src/components/providers/user-preferences-provider.tsx` и `frontend/src/lib/ui-copy.ts`,
  - откатить локализационные правки в `auth-form`, `private-shell`, `dashboard-view`,
  - удалить dark-theme переменные/override’ы из `frontend/app/globals.css`.
