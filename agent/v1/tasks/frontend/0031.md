## TSK-0031 — Добавить горизонтальную ленту календаря на dashboard
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- На dashboard отсутствует визуальная лента дней под приветствием.
- По решению owner нужна горизонтальная календарная лента под приветствием и над блоком графиков.
- На текущем этапе требуется только визуальная реализация без фильтрации данных.

### Acceptance Criteria
- [x] На `/dashboard` добавлена горизонтальная лента дней между приветствием и графиками.
- [x] Лента содержит выделение текущего дня и корректно отображается на RU/EN локалях.
- [x] Лента адаптивна: на mobile доступен горизонтальный скролл без поломки сетки.
- [x] Лента не меняет данные dashboard (только визуальный компонент).

### Constraints
- Не подключать бизнес-фильтрацию по дате в этой задаче.
- Не изменять API-контракты и backend.

### Change Summary
**Problem**
- На dashboard отсутствовала визуальная лента дней под приветствием, запрошенная по макетам.

**Root Cause Hypothesis**
- Файл `dashboard-calendar-ribbon.tsx` оказался в невалидной кодировке (UTF-16/binary для toolchain), из-за чего `lint` и `build` не могли его парсить.
- В компоненте использовались utility-классы и inline-style, что не соответствовало требованию ревьюера опираться на стили проекта.

**Evidence**
- При проверке фиксировались ошибки:
  - `npm run lint`: `Parsing error: File appears to be binary`.
  - `npm run build`: `stream did not contain valid UTF-8`.
- После пересоздания файла в UTF-8 количество `null`-байтов стало `0`, и команды `lint`/`build` проходят.

**Solution**
- Файл `frontend/src/components/dashboard-calendar-ribbon.tsx` пересоздан в валидной UTF-8 кодировке без бинарных артефактов.
- Компонент ленты переписан на проектные CSS-классы (без utility-классов и без `dangerouslySetInnerHTML`).
- В `frontend/app/globals.css` добавлены стили `dashboard-calendar-ribbon*` для desktop/mobile, включая горизонтальный скролл и выделение текущего дня.
- Интеграция ленты в `DashboardView` сохранена между `dashboard-personal` и блоками графиков.

**Tests / Verification**
- Запущен `cd frontend && npm run lint` — PASS.
- Запущен `cd frontend && npm run build` — PASS.
- Manual сценарий RU/EN и mobile scroll в этом ReMake-цикле не запускался.

**Impact**
- Добавлен новый визуальный компонент, на API и данные не влияет.

**Risks & Rollback**
- Компонент использует локальное время браузера, что штатно для визуальных лент, рисков влияния на бизнес-логику нет. Откат: удаление компонента из `dashboard-view.tsx`.

**Suggested Commit Message**
- `frontend: add horizontal calendar ribbon to dashboard view`
