## TSK-0042 — Roadmap: кнопки «Создать этап» и «Добавить тему» под заголовком с открытием форм в modal
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Сейчас формы создания этапа и темы встроены в страницу как отдельные панели.
- По требованию owner под блоком «Дорожная карта обучения ...» должны быть две кнопки: «Создать этап» и «Добавить тему», а сами формы должны открываться во всплывающих окнах (modal), без перехода на отдельные страницы.

### Acceptance Criteria
- [x] Под заголовком и подзаголовком roadmap размещены две кнопки: «Создать этап» и «Добавить тему».
- [x] Клик по «Создать этап» открывает modal с текущим содержимым формы создания этапа (поля/валидации/submit-flow сохранены).
- [x] Клик по «Добавить тему» открывает modal с текущим содержимым формы создания темы (поля/валидации/submit-flow сохранены).
- [x] Inline-панели создания этапа/темы удалены из основного потока страницы.
- [x] Закрытие modal поддержано через кнопку отмены и клавишу Esc, без смены маршрута `/roadmap`.
- [x] После успешного создания modal корректно закрывается, данные roadmap обновляются.

### Constraints
- Не менять backend API-контракт stage/topic create.
- Не переносить create-flow на отдельные маршруты.
- Не ломать empty-state quick-create сценарий для полностью пустого roadmap.

### Change Summary
**Problem**
- Формы создания этапа и темы рендерились inline-панелями в основном потоке roadmap-страницы, а требование owner — запускать эти create-flow из двух кнопок под заголовком с открытием modal.

**Root Cause Hypothesis**
- Roadmap create UI был реализован как постоянные секции (`StageMutationPanel` и `TopicMutationPanel`) без отдельного modal-state и без общего механизма закрытия create-форм по `Esc`.

**Evidence**
- До изменения create-панели рендерились inline внутри success-ветки roadmap и занимали место в основном потоке (`frontend/src/components/roadmap-view.tsx`).
- После изменения:
  - добавлены header-кнопки открытия create-modal (`roadmap-header-actions`);
  - create-формы этапа/темы рендерятся в `roadmap-modal-overlay`;
  - добавлено закрытие create-modal через `Esc` и cancel-кнопки.
- Команды проверки:
  - `Set-Location frontend; npm run lint` -> PASS;
  - `Set-Location frontend; npm run build` -> PASS.

**Solution**
- В `frontend/src/components/roadmap-view.tsx` добавлены две кнопки под заголовком roadmap: `Создать этап` и `Добавить тему`.
- Inline-рендер `StageMutationPanel`/`TopicMutationPanel` из основного потока удалён; вместо этого формы открываются в modal.
- Добавлены состояния и управляющие обработчики для create-modal этапа/темы:
  - open/close,
  - возврат фокуса на кнопку-триггер,
  - autofocus на title input при открытии.
- Существующие create submit-flow и валидации сохранены; формы переиспользуют текущие обработчики `handleStageCreate` и `handleTopicCreate`.
- После успешного создания modal автоматически закрывается, затем вызывается `roadmap.reload()`.
- Обновлена обработка `Esc`: закрывает create-modal (и существующий edit-stage modal), без смены маршрута `/roadmap`.
- В `frontend/app/globals.css` добавлены стили блока кнопок под заголовком (`.roadmap-header-actions`) и mobile-адаптация.

**Tests / Verification**
- Запущено:
  - `Set-Location frontend; npm run lint` -> PASS.
  - `Set-Location frontend; npm run build` -> PASS.
- Не запускалось локально:
  - ручной UI smoke `/roadmap` для интерактивной проверки открытия/закрытия modal и успешного create-flow с сохранением маршрута.

**Impact**
- UI/UX roadmap: create-flow stage/topic переведён в modal-сценарий, inline-панели убраны.
- API и контракты backend для create stage/topic: без изменений.
- Роутинг: без изменений, `/roadmap` сохраняется.
- Empty-state quick-create: сохранён.

**Risks & Rollback**
- Риск: без ручного smoke может остаться edge-case по фокусу/закрытию modal в отдельных браузерах.
- Rollback plan:
  - вернуть inline-рендер `StageMutationPanel`/`TopicMutationPanel` в success-ветку roadmap;
  - убрать modal-state/handlers для create-flow из `frontend/src/components/roadmap-view.tsx`;
  - удалить стили `.roadmap-header-actions` в `frontend/app/globals.css`.

**Suggested Commit Message**
- `frontend: move roadmap stage/topic create forms to header-triggered modals`
