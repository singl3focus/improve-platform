## TSK-0065 — Перенести language/theme в мини-иконки в левый нижний угол и убрать кнопку Settings
- Status: Finished
- Type: Business
- Approval: Approved

### Context
- Owner запросил показывать переключатели языка и темы как компактные иконки в левом нижнем углу интерфейса.
- Текущая реализация использует глобальный overlay-блок и отдельную кнопку перехода в `/settings`, что не соответствует требованию.

### Acceptance Criteria
- [x] На desktop в private-shell language/theme отображаются как мини-иконки в левом нижнем блоке sidebar (ниже основного меню).
- [x] На mobile language/theme находятся в самом низу меню/панели навигации и не перекрывают контент.
- [x] Кнопка/пункт Settings удален(а) из sidebar-навигации в рамках этой задачи.
- [x] Переключение языка и темы сохраняет текущее поведение (мгновенное применение, сохранение предпочтений, синхронизация `html[lang]` и `html[data-theme]`).
- [x] Визуальный и функциональный smoke по `/login`, `/register`, `/dashboard`, `/roadmap`, `/tasks`, `/materials` подтверждает отсутствие регрессий навигации и доступности controls.

### Constraints
- Не ломать существующий `UserPreferencesProvider` и словари локализации.
- Не добавлять обратно отдельный settings-экран как обязательную точку для language/theme.
- Не использовать git-команды.

### Change Summary
**Problem**
- Language/theme controls были вынесены в глобальный overlay в правом верхнем углу, а в sidebar оставалась отдельная ссылка Settings.
- Это расходилось с требованием: компактные controls в левом нижнем блоке private-sidebar и отсутствие Settings-кнопки в навигации.

**Root Cause Hypothesis**
- Preferences controls изначально были реализованы как универсальный overlay-компонент для всех роутов, а private-shell продолжал использовать отдельный нижний nav-блок под `/settings`.

**Evidence**
- `frontend/src/components/layout/private-shell.tsx`: присутствовал `BOTTOM_NAV_ITEMS` c `{ href: "/settings" }`.
- `frontend/app/layout.tsx` + `frontend/src/components/layout/global-ui-controls.tsx`: controls всегда рендерились как fixed overlay.
- `frontend/src/components/layout/preferences-controls.tsx`: существовал только select-вариант, не mini-icon формат для sidebar.
- Проверки:
  - `npm run lint` -> PASS.
  - `npm run build` -> PASS.

**Solution**
- Удален settings nav-item из private sidebar.
- В нижний блок sidebar встроен `PreferencesControls` с новым `variant="mini"`.
- В `PreferencesControls` добавлен mini-режим с компактными кнопками для language/theme:
  - язык переключается RU/EN,
  - тема переключается light/dark,
  - используются существующие `setLanguage`/`setTheme` из `UserPreferencesProvider`.
- В `global-ui-controls` добавлен route-gating: на private роутерах overlay не рендерится, чтобы не дублировать controls.
- Добавлены CSS-стили для mini-controls и mobile-раскладки в нижней части sidebar.

**Tests / Verification**
- `Set-Location frontend; npm run lint` -> PASS.
- `npm run build` -> PASS.
- Manual smoke по `/login`, `/register`, `/dashboard`, `/roadmap`, `/tasks`, `/materials` (desktop + mobile) -> PASS.
- Проверка переключения language/theme с валидацией `html[lang]` и `html[data-theme]` + reload persistence -> PASS.

**Impact**
- Навигация private-shell теперь без Settings-пункта.
- Language/theme controls перенесены в левый нижний блок sidebar и остаются доступными на mobile в нижней части меню.
- Поведение сохранения предпочтений не менялось (используется существующий provider).
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: мини-режим может потребовать дополнительной UX-полировки для текстов/иконок в отдельных темах.
- Rollback: вернуть settings nav-item в `private-shell.tsx`, убрать `variant="mini"`, вернуть прежний overlay-only сценарий.
