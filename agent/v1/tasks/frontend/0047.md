### Context
- Owner подтвердил, что текущий roadmap должен быть заменен целиком на полноценный графовый экран, а не дополнен частично.
- Текущая реализация отображает этапы как колонки с карточками, что не соответствует целевому сценарию визуального графа из `design/example_graph.html`.

### Acceptance Criteria
- [x] Экран `/roadmap` полностью использует графовый layout (сцена с узлами и связями) вместо текущего колоночного представления.

- [x] Drag/клик сценарии зависимостей корректно работают в новом графовом layout.
- [x] Экран остается рабочим на desktop и mobile (без критических наложений, с доступной навигацией).
- [x] Шрифт приложения не меняется.

### Constraints
- Не менять backend API-контракты и бизнес-правила доменов backend.
- Не добавлять новые зависимости без явной необходимости.
- Изменения ограничить scope экрана roadmap и связанных frontend-модулей.

### Change Summary
**Problem**
- Экран roadmap оставался в формате stage-columns, тогда как owner подтвердил полную замену на graph-сцену по референсу `design/example_graph.html`.

**Root Cause Hypothesis**
- Основной UI roadmap был построен как grid колонок (`roadmap-stage-columns` / `roadmap-stage-column`), поэтому визуально и структурно не соответствовал целевому graph-view.

**Evidence**
- До изменения layout рендерился через колонки в `frontend/src/components/roadmap-view.tsx`.
- После изменения рендер переведен на сцену с lane-группировкой и узлами: `roadmap-graph-scene`, `roadmap-stage-lane`, `roadmap-stage-lane-nodes` в `frontend/src/components/roadmap-view.tsx`.
- Стили графовой сцены, lane-watermark и mobile-адаптации добавлены/обновлены в `frontend/app/globals.css`.
- `Set-Location frontend; npm run lint` -> PASS.
- `Set-Location frontend; npm run build` -> PASS.

**Solution**
- Заменен roadmap-layout в `RoadmapView`: вместо grid-колонок реализована graph-сцена с вертикальными stage-lanes и узлами внутри каждого этапа.
- Сохранены существующие сценарии и обработчики без изменения бизнес-логики:
  - открытие темы (`router.push('/topics?topicId=...')`),
  - create/update/delete этапов и тем,
  - create/delete зависимостей,
  - drag/click dependency interaction.
- Обновлены roadmap-стили под graph-направление:
  - сцена с атмосферным фоном и прокруткой,
  - lane-блоки с watermark названиями этапов,
  - sticky legend на desktop,
  - mobile-правила для новой структуры (`roadmap-graph-scene`, `roadmap-stage-lane-nodes`).

**Tests / Verification**
- Запущено:
  - `npm run lint` — PASS.
  - `npm run build` — PASS.
  - `npm run verify:mvp` — PASS.
  - ручной browser smoke `/roadmap` (desktop/mobile + интеракции click/drag dependency) — PASS.

**Impact**
- UI: экран `/roadmap` визуально переведен на graph-сцену вместо stage-columns.
- API/контракты backend: без изменений.
- Безопасность/данные: без изменений.
- Обратная совместимость: да, пользовательские сценарии roadmap сохранены в рамках текущего frontend-модуля.

**Risks & Rollback**
- Риск: возможны точечные расхождения визуального позиционирования узлов/связей на редких viewport вне проверенных desktop/mobile размеров.
- Rollback plan:
  - вернуть предыдущую версию layout в `frontend/src/components/roadmap-view.tsx`;
  - вернуть предыдущие roadmap-стили в `frontend/app/globals.css`.
