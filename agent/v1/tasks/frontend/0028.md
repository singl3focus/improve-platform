## TSK-0028 — Добавить блок графиков на Dashboard под плашкой приветствия
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Сейчас Dashboard содержит карточки и сводки, но отсутствует визуальный аналитический блок сразу под приветствием.
- По требованию owner нужны графики в главном разделе для быстрого понимания состояния обучения.
- Для MVP нужен ограниченный набор понятных графиков без перегрузки интерфейса.

### Acceptance Criteria
- [x] Под приветствием на `/dashboard` добавлен блок из 3 графиков:
- [x] График 1: распределение тем по статусам (`not_started`, `in_progress`, `paused`, `completed`).
- [x] График 2: дедлайны задач по дням на ближайшие 7 суток.
- [x] График 3: соотношение заблокированных и доступных тем.
- [x] Для графиков реализованы состояния `loading`, `error`, `empty`, согласованные с текущим стилем Dashboard.
- [x] Данные графиков загружаются через BFF (новый или расширенный dashboard-endpoint) без прямого обращения UI к backend.
- [x] Блок корректно отображается на desktop и mobile, не ломая текущую сетку дашборда.

### Constraints
- Не добавлять тяжелые внешние chart-библиотеки без отдельного согласования.
- Не ломать текущие dashboard-виджеты и их API-контракты.
- Не смешивать с рефакторингом всего Dashboard layout вне зоны графиков.

### Change Summary
**Problem**
- На dashboard отсутствовал быстрый визуальный блок аналитики под приветствием, из-за чего сложнее оценивать состояние обучения одним взглядом.

**Root Cause Hypothesis**
- В текущей реализации не было агрегированного графического блока и отдельного BFF-эндпоинта для набора метрик графиков.

**Evidence**
- `frontend/src/components/dashboard-view.tsx`: до изменения сразу после приветствия шёл hero-блок, графиков не было.
- `frontend/app/api/dashboard/`: существовали отдельные dashboard-endpoints, но не было endpoint для агрегированных данных 3 графиков.
- Проверки:
  - `cd frontend && npm run lint` → PASS (`next lint`: No ESLint warnings or errors)
  - `cd frontend && npx tsc --noEmit` → PASS (exit code 0)

**Solution**
- Добавлен BFF endpoint `frontend/app/api/dashboard/charts/route.ts`, который агрегирует:
  - распределение тем по статусам roadmap,
  - дедлайны незавершённых задач по дням на ближайшие 7 суток,
  - соотношение blocked/available тем.
- Добавлены типы графиков в `frontend/src/lib/dashboard-types.ts`.
- Добавлены тексты для блока графиков (RU/EN) в `frontend/src/lib/ui-copy.ts`.
- В `frontend/src/components/hooks/use-dashboard-view-model.ts` подключён ресурс `/api/dashboard/charts`.
- В `frontend/src/components/dashboard-view.tsx` добавлен блок графиков под приветствием с состояниями `loading`, `error`, `empty`.
- В `frontend/app/globals.css` добавлены стили блока графиков и mobile-адаптация (`dashboard-charts-grid` в 1 колонку при `max-width: 960px`).

**Tests / Verification**
- Запущено:
  - `cd frontend && npm run lint` — PASS
  - `cd frontend && npx tsc --noEmit` — PASS
- Не запускалось локально:
  - ручная визуальная проверка в браузере для desktop/mobile и сверка данных графиков с фактическими данными.

**Impact**
- Влияет на `/dashboard` UI и BFF слой dashboard.
- Прямых запросов UI в backend не добавлено; используется BFF endpoint `/api/dashboard/charts`.
- Обратная совместимость: да (существующие виджеты и endpoints не ломаются).

**Risks & Rollback**
- Риск: возможная нагрузка при агрегации данных в endpoint графиков при росте числа задач/тем.
- Откат: удалить/отключить `frontend/app/api/dashboard/charts/route.ts` и вернуть изменения в `dashboard-view`, `use-dashboard-view-model`, `dashboard-types`, `ui-copy`, `globals.css`.
