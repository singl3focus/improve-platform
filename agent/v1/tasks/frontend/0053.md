## TSK-0053 — Добавить zoom/pan/recenter для граф-сцены roadmap
- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-17

### Context
- После перехода к node-only визуалу roadmap должен ощущаться как полноценный граф: приближение, отдаление и перемещение сцены.
- Без zoom/pan даже компактные карточки остаются неудобны при большом числе тем и связей.

### Acceptance Criteria
- [x] На `/roadmap` есть рабочие контролы `zoom in`, `zoom out`, `recenter`.
- [x] Масштабирование применяет трансформацию граф-сцены и не ломает геометрию линий связей.
- [x] Перемещение (pan) граф-сцены работает мышью/трекпадом на desktop и не конфликтует с drag-созданием связи между темами.
- [x] После `recenter` граф возвращается в исходную позицию и базовый масштаб.
- [x] Клик по карточке, редактирование темы и link/unlink связей продолжают работать при измененном масштабе.
- [x] Добавлены/обновлены тесты helper-функций графа (координаты/нормализация/путь) для сценариев с трансформацией сцены.

### Constraints
- В этой задаче не реализовывать отдельный режим `fit-to-screen`.
- В этой задаче не реализовывать pinch-жесты на mobile; только кнопочные контролы масштаба.
- Не менять бизнес-правила roadmap (DAG-ограничения, статусы тем, доменные переходы).

### Change Summary
**Problem**
- После перехода на node-only roadmap-сцену не было инструментов навигации по графу при большом количестве тем.
- Отсутствовали zoom/pan/recenter, из-за чего обзор графа и работа со связями были ограничены.
- В ReMake-проверке выявлен UX-дефект: при сильном смещении сцены тема могла перекрывать кликабельность контролов графа (`Recenter`/zoom).

**Root Cause Hypothesis**
- Граф-сцена рендерилась статично, без состояния трансформации и контролов управления viewport.
- Helper-слой не покрывал вычисления для сценариев масштабирования (clamp/anchor offset).

**Evidence**
- В `frontend/src/components/roadmap-view.tsx` добавлены контролы и состояние трансформации сцены (zoom in/out/recenter, pan pointer/wheel).
- В `frontend/src/components/roadmap/use-roadmap-graph-layout.ts` добавлен учёт `sceneTransform` для перерасчёта линий при zoom/pan.
- В `frontend/src/lib/roadmap-graph.ts` добавлены helper-функции трансформации и расширена нормализация координат.
- В `frontend/src/lib/roadmap-graph.test.ts` добавлены тесты для трансформаций.
- Для ReMake внесён точечный фикс перекрытия контролов:
  - `frontend/src/components/roadmap-view.tsx`: контролы обёрнуты в `roadmap-graph-controls-layer`.
  - `frontend/app/globals.css`: добавлен overlay-слой `roadmap-graph-controls-layer` (`pointer-events: none`) и `pointer-events: auto` для самого блока кнопок.

**Solution**
- Добавлены контролы `zoom in`, `zoom out`, `recenter` в граф-сцене roadmap.
- Реализовано масштабирование сцены с фиксацией якоря по центру viewport и ограничением масштаба.
- Реализован pan:
  - drag мышью по пустой области графа;
  - wheel-pan для трекпада/мыши;
  - исключены конфликтные цели (`topic card`, кнопки/инпуты), чтобы не ломать существующие действия.
- Перерасчёт линий зависимостей привязан к изменению трансформации сцены, чтобы геометрия связей оставалась корректной.
- Добавлены тесты helper-функций для сценариев с трансформацией (`normalizeGraphPoint`, `clampGraphScale`, `getGraphOffsetForScale`).
- Для корректной кликабельности контролов после pan/zoom добавлен отдельный overlay-слой контролов поверх граф-сцены.
