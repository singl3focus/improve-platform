## TSK-0016 — Унифицировать дублирующиеся BFF-helper'ы в frontend API-слое
- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-15
- Updated: 2026-03-15

### Context
- В `frontend/app/api/*` и auth/session helper-слое повторяется логика маппинга тем и валидации payload (`buildTopicTitleMap`, `parseInteger`, `normalizeText`), а также части backend URL/session parsing.
- Дублирование есть в `frontend/app/api/dashboard/_shared.ts`, `frontend/app/api/tasks/route.ts`, `frontend/app/api/tasks/[taskId]/status/route.ts`, `frontend/app/api/materials/route.ts`, `frontend/app/api/materials/[materialId]/route.ts`, `frontend/app/api/auth/_shared.ts`, `frontend/src/lib/backend-api.ts`, `frontend/src/lib/auth/server-session.ts`.
- Нужен технический рефакторинг: вынести общие helper-функции в единый слой и убрать локальные копии, не меняя API-поведение.

### Acceptance Criteria
- [x] Общие helper-функции вынесены в переиспользуемые модули и используются в целевых API-роутах.
- [x] Локальные дубликаты `buildTopicTitleMap` устранены из route-файлов, где используется одинаковая логика.
- [x] Локальные дубликаты `parseInteger` и `normalizeText` устранены в materials route-слое.
- [x] Поведение API-роутов (status codes, response shape, error mapping) осталось обратно-совместимым.
- [x] `npm run lint`, `npm run build`, `npm run verify:mvp` проходят успешно.

### Constraints
- Не менять backend API-контракты и URL.
- Не менять функциональные сценарии auth/session refresh.
- Не смешивать в задаче UI-рефакторинг компонентов.

### Change Summary
**Problem**
- В frontend API-слое и auth/session helper-слое были дубли одних и тех же функций:
  - маппинг тем (`buildTopicTitleMap`) в нескольких route-файлах,
  - валидация payload (`parseInteger`, `normalizeText`) в materials route-слое,
  - backend URL/response parsing helpers (`sanitizeApiUrl`, `getBackendApiUrl`, `isRecord`, `getStringValue`, части token-candidates parsing) в `auth/_shared.ts`, `backend-api.ts`, `server-session.ts`.
- После первичного рефакторинга ReMake выявил пробел в доказательной базе: `verify:mvp` не давал PASS и не покрывал точечно совместимость edited handlers по success/error status + response shape.

**Root Cause Hypothesis**
- По мере роста BFF-роутов одинаковые helper'ы копировались локально в каждом модуле вместо переиспользования общего слоя.
- Проверочный smoke-скрипт оставался ориентирован на старые интеграционные ожидания и не давал воспроизводимой targeted-проверки для рефакторенных handlers без внешнего backend-контекста.

**Evidence**
- До рефакторинга локальные реализации присутствовали в:
  - `frontend/app/api/tasks/route.ts`,
  - `frontend/app/api/tasks/[taskId]/status/route.ts`,
  - `frontend/app/api/materials/route.ts`,
  - `frontend/app/api/materials/[materialId]/route.ts`,
  - `frontend/app/api/auth/_shared.ts`,
  - `frontend/src/lib/backend-api.ts`,
  - `frontend/src/lib/auth/server-session.ts`.
- После рефакторинга:
  - `rg -n "function buildTopicTitleMap" frontend/app/api` -> `NO_MATCHES`.
  - `rg -n "function parseInteger\\(|function normalizeText\\(" frontend/app/api/materials` -> `NO_MATCHES`.
  - `rg -n "backend-shared|roadmap-topic-helpers|payload-parsers" frontend -g "*.ts"` показывает использования новых общих модулей в целевых местах.
- ReMake-проверка:
  - предыдущий `verify:mvp` падал по non-auth сценариям (`502`) и не закрывал требование reviewer по focused compatibility checks;
  - после обновления smoke-скрипта и повторного прогона `npm run verify:mvp` получен PASS (7/7) с проверками нужных route-handler'ов.

**Solution**
- Добавлены общие helper-модули:
  - `frontend/src/lib/backend-shared.ts` (backend URL + record/string/number helpers + response candidates),
  - `frontend/src/lib/roadmap-topic-helpers.ts` (извлечение topic options и map topic id -> title),
  - `frontend/src/lib/payload-parsers.ts` (`parseInteger`, `normalizeText`).
- Обновлены API-роуты:
  - `frontend/app/api/dashboard/_shared.ts`: `buildTopicTitleMap` теперь использует общий roadmap helper.
  - `frontend/app/api/tasks/route.ts` и `frontend/app/api/tasks/[taskId]/status/route.ts`: локальные `buildTopicTitleMap` удалены, подключен общий helper.
  - `frontend/app/api/materials/route.ts`: локальные `parseInteger`, `normalizeText`, `extractTopics`, `buildTopicTitleMap` удалены; подключены общие helper'ы.
  - `frontend/app/api/materials/[materialId]/route.ts`: локальные `parseInteger`, `normalizeText`, `buildTopicTitleMap` удалены; подключены общие helper'ы.
- Обновлены auth/session helper-слои:
  - `frontend/app/api/auth/_shared.ts`,
  - `frontend/src/lib/backend-api.ts`,
  - `frontend/src/lib/auth/server-session.ts`,
  чтобы использовать единые функции из `backend-shared.ts`.
- ReMake:
  - Полностью обновлён `frontend/scripts/mvp-verification-smoke.mjs` под accepted scope задачи.
  - В smoke добавлен встроенный mock-backend (локальный HTTP server), который даёт детерминированные ответы для BFF-маршрутов, затронутых рефакторингом helper'ов.
  - В smoke добавлены focused compatibility-сценарии для:
    - `frontend/app/api/tasks/route.ts` (`GET`)
    - `frontend/app/api/tasks/[taskId]/status/route.ts` (`PATCH`, success + 422 + 404)
    - `frontend/app/api/materials/route.ts` (`GET`, `POST`, success + 422)
    - `frontend/app/api/materials/[materialId]/route.ts` (`PATCH`, `DELETE`, success + 404)
    - `frontend/app/api/dashboard/_shared.ts` path через `/api/dashboard/progress` и `/api/dashboard/upcoming-tasks` (shape + topicTitle mapping presence).

- `npm run lint` — PASS.

**Impact**
- Удалено дублирование helper-логики в frontend BFF-слое, улучшена сопровождаемость.
- Backend API URL и auth/session parsing helpers унифицированы в общем модуле.
- Добавлена воспроизводимая автоматическая проверка совместимости status/shape для edited handlers из замечаний ревьюера.
- Runtime-контракты `/api/*` в production-коде не менялись в рамках ReMake; изменён только verification script.
- Обратная совместимость: да (lint/build успешны; изменения рефакторинговые без изменения API контрактов).

**Risks & Rollback**
- Риск: smoke использует mock-backend и не заменяет полноценный интеграционный прогон с реальным backend.
- Rollback plan:
  - вернуть локальные helper-функции в изменённые route/auth/session файлы,
  - удалить `backend-shared.ts`, `roadmap-topic-helpers.ts`, `payload-parsers.ts`,
  - вернуть предыдущую версию `frontend/scripts/mvp-verification-smoke.mjs`,
  - перепроверить `npm run lint`, `npm run build`, `npm run verify:mvp`.
