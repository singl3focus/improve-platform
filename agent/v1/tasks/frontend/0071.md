## TSK-0071 — Добавить проверку сценариев для обновленного UI roadmap/tasks
- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-18

### Context
- После изменений в карточках roadmap и задач возрастает риск регрессий UI/UX (возврат description в roadmap, поломка edit-модалки задач, возврат удаленного drag-hint).
- Нужна техническая задача на проверяемую верификацию критичных сценариев, чтобы закрепить изменения.

- [x] Сценарии включают проверку отсутствия текста drag-hint в карточке задачи.
- [x] Verification в задаче разработки содержит явные PASS/FAIL результаты по каждому сценарию.

### Change Summary
**Problem**
- Для недавних изменений UI в `/roadmap` и `/tasks` не было отдельного воспроизводимого verification-набора, который явно фиксирует риск-регрессии по 4 критичным сценариям: скрытый description в roadmap-card, edit-flow задачи, поведение модалки при API-ошибке и отсутствие drag-hint в task-card.

**Root Cause Hypothesis**
- Сценарии проверялись точечно и не были формализованы в едином checklist с воспроизводимым mock backend-окружением.
- Негативный edit-flow (`PUT /api/v1/tasks/{id}` -> error) не имел явного backend-toggle для стабильной проверки UI-поведения модалки.

**Evidence**
- Добавлен checklist сценариев: `frontend/scripts/roadmap-tasks-ui-verification.md`.
- Добавлен mock backend для сценариев с toggle ошибки обновления задачи: `frontend/scripts/roadmap-tasks-ui-review-backend.mjs`.
- Добавлен npm-скрипт запуска mock backend: `frontend/package.json` -> `verify:roadmap-tasks-ui:backend`.
- Результаты runtime UI smoke:
  - `/roadmap`: description-текст `Core markup and styling skills for UI verification.` не отображается в карточке.
  - `/tasks`: edit-flow успешно обновляет карточку и закрывает модалку.
  - `/tasks` + `taskUpdateMode=error`: модалка не закрывается, ошибка отображается.
  - `/tasks`: drag-hint текст отсутствует.

**Solution**
- Добавлен документированный ручной checklist с чёткими preconditions, шагами и ожидаемыми результатами для всех 4 сценариев.
- Реализован отдельный review mock backend с минимально необходимыми endpoint-ами (`roadmap`, `tasks`, `topic metrics`) и endpoint-ом `POST /__mock/scenario` для переключения `taskUpdateMode` (`ok|error`).
- В `package.json` добавлен скрипт запуска review backend для воспроизводимой локальной проверки.

**Tests / Verification**
- Запущено:
  - `Set-Location frontend; npm run lint` — PASS.
  - `Set-Location frontend; npm run build` — PASS.
  - Manual UI checklist из `frontend/scripts/roadmap-tasks-ui-verification.md` — PASS по всем 4 сценариям.
- Не запускалось:
  - Автоматизированные e2e-тесты (Playwright/Cypress) в CI-пайплайне для этих сценариев не добавлялись в рамках задачи.

**Impact**
- UI/API контракты продукта не изменены; добавлена только verification-инфраструктура и документация сценариев.
- Улучшена воспроизводимость ручного regression-check для `/roadmap` и `/tasks`.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: сценарии остаются manual-checklist (зависят от дисциплины прогона).
- Риск: mock backend покрывает только необходимые для проверки ветки и не заменяет интеграцию с реальным backend.
- Rollback plan:
  - удалить `frontend/scripts/roadmap-tasks-ui-verification.md`;
  - удалить `frontend/scripts/roadmap-tasks-ui-review-backend.mjs`;
  - убрать `verify:roadmap-tasks-ui:backend` из `frontend/package.json`.
