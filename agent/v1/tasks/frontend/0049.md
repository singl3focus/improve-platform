### Change Summary
**Problem**
- Экран `/tasks` визуально не соответствовал выбранному дизайну из `design/_4/code.html`: фильтры были всегда раскрыты, у action-кнопок не было требуемого визуала с иконками, а заголовок и визуальный ритм Kanban отличались от целевого.

**Root Cause Hypothesis**
- Текущая реализация `TasksBoardHeader` рендерила фильтры inline в шапке и использовала базовые кнопки без выделенного дизайн-слоя иконок/действий.
- CSS для Kanban-колонок держал рамочный стиль (`border`) вместо более «воздушного» surface-оформления, близкого к референсу.

**Evidence**
- До изменений в `frontend/src/components/tasks-kanban-view.tsx` фильтры (`topic/deadline`) рендерились напрямую внутри `TasksBoardHeader`.
- После изменений в `frontend/src/components/tasks-kanban-view.tsx` добавлен drawer `TasksFiltersDrawer` с `role="dialog"`, `aria-modal`, закрытием по `Esc`, overlay и возвратом фокуса на trigger.
- В `frontend/src/components/tasks-kanban-view.tsx` добавлены отдельные SVG-иконки `TaskCreateIcon` и `TaskFilterIcon`, а заголовок RU приведен к `Мой канбан`.
- В `frontend/app/globals.css` обновлены стили Kanban: `tasks-action-button`, `tasks-filter-drawer`, `tasks-column` (без рамок), обновлённый surface-вид колонок/карточек.
- `npm run lint` -> PASS.
- `npm run build` -> PASS.
- Ручной smoke `/tasks` -> PASS (фильтр, создание, DnD, удаление).

**Solution**
- Обновлен `TasksBoardHeader`:
  - inline-фильтры убраны из шапки;
  - добавлены action-кнопки `Filter` и `Create task` с кастомными SVG-иконками;
  - добавлен визуальный active-state для кнопки фильтра.
- Реализован отдельный drawer фильтров `TasksFiltersDrawer`:
  - поля `Topic` и `Deadline`;
  - применение фильтров к текущей доске через существующий `setFilters`;
  - действия `Reset` и `OK`;
  - доступность: `role=dialog`, `aria-modal`, закрытие по `Esc`/overlay/кнопке, возврат фокуса на trigger.
- Проведён restyle Kanban в `frontend/app/globals.css`:
  - шапка и кнопки в направлении `design/_4`;
  - колонки без рамок, более мягкий фон;
  - drawer-стили для фильтров;
  - карточки получили более выразительный surface и тень.

**Tests / Verification**
- Запущено:
  - `npm run lint` — PASS.
  - `npm run build` — PASS.
  - Manual `/tasks` smoke — PASS (filter drawer + apply/reset + Esc/overlay close, create task, DnD, delete task).

**Impact**
- UI: обновлён внешний вид и UX Kanban на `/tasks`.
- API/контракты backend: без изменений.
- Данные: без изменений структуры; в ручной проверке создавалась и удалялась тестовая задача.
- Доступность: улучшено поведение фильтров через dialog-pattern с управлением фокусом.
- Обратная совместимость: да, ключевые пользовательские сценарии (`create`, `delete`, `drag-and-drop`, `filter`) сохранены.

**Risks & Rollback**
- Риск: возможны дополнительные визуальные коррекции на редких viewport при сочетании длинных заголовков/локализации.
- Rollback plan:
  - вернуть предыдущую версию `frontend/src/components/tasks-kanban-view.tsx`;
  - вернуть предыдущие styles-блоки `tasks-*` в `frontend/app/globals.css`.
