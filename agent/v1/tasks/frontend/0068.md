### Change Summary
**Problem**
- Frontend materials продолжал работать со старым контрактом (`progressPercent` как входное поле), тогда как backend перешел на `type/unit/total_amount/completed_amount` с вычисляемым `progress`.
- В ReMake-цикле требовалось подтвердить runtime smoke без `fetch failed` и явно зафиксировать ручную проверку desktop/mobile + all-types create/edit доказательствами.

**Root Cause Hypothesis**
- BFF и UI-формы материалов не были синхронизированы с backend-изменением из `TSK-0022`, поэтому теряли новые поля и позволяли редактировать устаревший `progress`.

**Evidence**
- До изменения `frontend/src/lib/materials-library-types.ts` содержал только `progressPercent` и не содержал `type/unit/totalAmount/completedAmount`.
- До изменения `frontend/src/components/materials-library-view.tsx` формы create/edit оперировали полем `progress (%)` вместо полной/выполненной меры.
- До изменения BFF в `frontend/app/api/materials/route.ts` и `frontend/app/api/materials/[materialId]/route.ts` отправлял/маппил `progress` без новых полей контракта.
- Добавлен изолированный mapping-слой `frontend/src/lib/materials-api-mapping.ts` и тесты `frontend/src/lib/materials-api-mapping.test.ts`, подтверждающие сохранение `type/unit/totalAmount/completedAmount/progressPercent` и корректный snake_case payload в backend.
- Обновлен smoke `frontend/scripts/mvp-verification-smoke.mjs` под новый materials-контракт; `npm run verify:mvp` -> PASS.
- Проверка после изменений: `npm run lint`, `npm run test`, `npm run build`, `npm run verify:mvp` -> PASS.
- Для воспроизводимости smoke отдельно поднят frontend runtime на `http://127.0.0.1:3025`, затем повторно выполнен `npm --prefix frontend run verify:mvp` -> PASS (без `fetch failed`).
- Дополнительно выполнен runtime-check all material types через frontend BFF (`POST /api/materials`, `PATCH /api/materials/{id}`): `book/article/course/video` -> unit/amount/type сохраняются без потерь.
- Выполнена ручная UI-проверка `/materials` на desktop и mobile (`<=960`): карточка показывает type badge + measure meta + progress, create-modal держит auto-unit и disabled unit-input для всех 4 типов.

**Solution**
- Обновлена модель материалов во frontend (`frontend/src/lib/materials-library-types.ts`):
  - добавлены `MaterialType`, `MaterialUnit`, `MATERIAL_TYPE_TO_UNIT`;
  - в `LibraryMaterial` добавлены `type`, `unit`, `totalAmount`, `completedAmount`.
- Обновлены form-утилиты (`frontend/src/lib/materials-form.ts`):
  - добавлены `parseNonNegativeInteger`, `resolveUnitByType`, `computeProgressPercent`;
  - оставлен clamp/normalize прогресса для вычисления процента из меры.
- Переработан view-model материалов (`frontend/src/components/hooks/use-materials-library-view-model.ts`):
  - draft и payload переведены на `type + total/completed`;
  - добавлена валидация `completedAmount <= totalAmount`;
  - удален старый debounce-пайплайн ручного обновления `progress` через range.
- Переработан UI материалов (`frontend/src/components/materials-library-view.tsx`):
  - в create/edit добавлены поля `type`, read-only `unit`, `totalAmount`, `completedAmount`;
  - unit автоподставляется и блокируется по выбранному типу;
  - карточки обновлены визуально: type badge + measure meta (`completed/total unit`) + пересчитанный progress.
- Обновлены стили карточек (`frontend/app/globals.css`) под направление `design/_5/code.html` с акцентом на читаемость и мета-бейджи.
- Обновлен BFF-контракт материалов:
  - `frontend/app/api/materials/route.ts` и `frontend/app/api/materials/[materialId]/route.ts` валидируют/маппят `type`, `totalAmount`, `completedAmount` и возвращают `unit`;
  - в backend-запросы отправляются `type`, `total_amount`, `completed_amount`.
- Добавлен единый mapping helper `frontend/src/lib/materials-api-mapping.ts`:
  - `mapBackendMaterialToLibraryMaterial`;
  - `toBackendCreateMaterialPayload`;
  - `toBackendUpdateMaterialPayload`.
- Добавлены unit-тесты `frontend/src/lib/materials-api-mapping.test.ts` и подключены в `frontend/tsconfig.test.json`.
- Обновлен `frontend/scripts/mvp-verification-smoke.mjs` для материалов:
  - mock backend хранит/возвращает `type/unit/total_amount/completed_amount/progress`;
  - materials smoke-сценарии проверяют create/edit без потери полей по новому контракту.
