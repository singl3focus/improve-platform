## TSK-0024 — CRUD тем в Roadmap (добавление, редактирование, удаление узлов)
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Сейчас UI Roadmap поддерживает только quick-create первой темы, полноценного управления темами в существующем roadmap нет.
- Пользователю нужно добавлять новые темы после первой, изменять их и удалять в рамках одного экрана Roadmap.
- Без CRUD тем roadmap не работает как управляемая карта развития.

### Acceptance Criteria
- [x] На экране `/roadmap` можно добавить новую тему в выбранный stage, если roadmap уже существует.
- [x] На экране `/roadmap` можно редактировать тему (минимум: `title`, `description`, `stage`, `position`).
- [x] На экране `/roadmap` можно удалить тему с подтверждением действия.
- [x] После create/update/delete граф и список тем обновляются без ручной перезагрузки страницы.
- [x] В BFF-слое добавлены/используются маршруты для `POST/PUT/DELETE` операций по темам с нормализованными ошибками backend.

### Constraints
- Не менять backend API-контракт `roadmap/topics`.
- Не нарушать существующие DAG-инварианты и правила блокировок.
- Не смешивать задачу с редактированием зависимостей (это отдельный TSK).

### Change Summary
**Problem**
- На `/roadmap` после первого quick-create не было CRUD по темам: отсутствовали UI-операции добавления, редактирования и удаления тем в существующей roadmap.
- В BFF отсутствовали маршруты `POST/PUT/DELETE` для тем roadmap, поэтому UI не мог выполнять эти операции через единый frontend-контур.

**Root Cause Hypothesis**
- Компонент `RoadmapView` покрывал только чтение графа и quick-create первой темы (`/api/roadmap/quick-create`), без мутаций по отдельным узлам.
- В `frontend/app/api/roadmap/*` не было маршрутов для CRUD тем после создания roadmap.

**Evidence**
- UI-CRUD добавлен в `frontend/src/components/roadmap-view.tsx`:
  - create form для темы: `roadmap-topic-mutation-panel` и `handleTopicCreate` (`:579`, `:747`);
  - edit flow (`title`, `description`, `stage`, `position`): `handleTopicUpdate` + inline `roadmap-topic-edit-form` (`:641`, `:990`);
  - delete flow с подтверждением: `handleTopicDelete` и `window.confirm` (`:689`, `:697`).
- BFF-роуты добавлены:
  - `POST /api/roadmap/topics` в `frontend/app/api/roadmap/topics/route.ts:26`;
  - `PUT/DELETE /api/roadmap/topics/[topicId]` в `frontend/app/api/roadmap/topics/[topicId]/route.ts:23`, `:110`.
- Нормализация backend-ошибок в BFF реализована через `createBackendErrorResponse` в обоих новых маршрутах (`route.ts:85`, `[topicId]/route.ts:55`, `:85`, `:120`).
- Для корректного редактирования позиции и согласованности типов:
  - `position` добавлен в frontend-контракты `BackendRoadmapTopic` и `RoadmapTopic` (`frontend/src/lib/backend-learning-contracts.ts:6`, `frontend/src/lib/roadmap-types.ts:8`);
  - mapping `position` из backend добавлен в `frontend/app/api/roadmap/route.ts:135`.
- Проверка командами:
  - `npm run lint` -> PASS.
  - `npx tsc --noEmit` -> PASS.

**Solution**
- Добавлен BFF-контур CRUD тем roadmap:
  - create-роут с валидацией входа и дефолтным `position` по выбранному stage;
  - update-роут с валидацией и сохранением текущих `start_date/target_date` из backend topic;
  - delete-роут с единым форматом ошибок.
- На `/roadmap` добавлены:
  - панель создания темы в существующий stage;
  - кнопки редактирования/удаления на карточке темы;
  - inline-форма редактирования `title/description/stage/position`;
  - отображение mutation-ошибок в стандартном `dashboard-error` блоке.
- После `create/update/delete` выполняется `roadmap.reload()`, поэтому граф и список тем обновляются без ручного refresh.
- Добавлены стили для новых roadmap-форм/действий в `frontend/app/globals.css` с мобильной адаптацией.

**Tests / Verification**
- Запущено:
  - `cd frontend && npm run lint` — PASS.
  - `cd frontend && npx tsc --noEmit` — PASS.
- Не запускалось локально:

**Impact**
- UI/UX: на `/roadmap` появился полный CRUD тем в рамках одного экрана.
- API/BFF: добавлены новые frontend API маршруты `/api/roadmap/topics` и `/api/roadmap/topics/[topicId]`.
- Контракты: backend API-контракт не изменён; frontend-типы дополнены полем `position` для темы.
- Безопасность: без изменений.
- Обратная совместимость: да (quick-create и существующая загрузка roadmap сохранены).

**Risks & Rollback**
- Риск: не выполнен ручной end-to-end smoke с реальным backend, поэтому UX-поведение CRUD не подтверждено браузерно.
- Риск: update-topic зависит от доступности `GET /api/v1/roadmap/topics/{id}` для сохранения текущих дат (`start_date/target_date`) перед `PUT`.
- Rollback plan:
  - удалить `frontend/app/api/roadmap/topics/route.ts` и `frontend/app/api/roadmap/topics/[topicId]/route.ts`;
  - откатить CRUD-часть в `frontend/src/components/roadmap-view.tsx`;
  - удалить добавленные roadmap CRUD стили в `frontend/app/globals.css`;
  - вернуть прежние типы `RoadmapTopic`/`BackendRoadmapTopic` без `position`, если потребуется.
