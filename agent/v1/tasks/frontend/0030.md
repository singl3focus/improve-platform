## TSK-0030 — Добавить иконки в левое меню и вынести кнопку настроек вниз
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Левое меню сейчас текстовое и не содержит иконок.
- По решению owner навигация должна содержать иконки, а кнопка перехода в настройки должна быть закреплена в нижней части меню.

### Acceptance Criteria
- [x] Все основные пункты левого меню отображаются с иконками.
- [x] Кнопка/пункт перехода в `/settings` расположен(а) внизу сайдбара и визуально отделен(а) от основного списка.
- [x] Переход в настройки работает по клику/клавиатуре, состояние active корректно отображается.
- [x] На mobile-режиме поведение не ломается (кнопка настроек остается доступной).

### Constraints
- Не менять маршруты навигации.
- Не вводить тяжелые UI-библиотеки ради иконок.

### Change Summary
**Problem**
- Левое меню навигации было чисто текстовым и кнопка "Настройки" находилась в общем списке, вместо того, чтобы быть логически отделенной внизу.

**Root Cause Hypothesis**
- Компонент `PrivateShell` рендерил все ссылки из одного массива без использования иконок, а `.sidebar` не имел flexbox для отталкивания элементов вниз.

**Evidence**
- В `frontend/src/components/layout/private-shell.tsx` присутствуют `MAIN_NAV_ITEMS` и `BOTTOM_NAV_ITEMS`; маршрут `settings` остается `/settings`.
- Повторный запуск команд в текущем состоянии: `npm run lint` — PASS, `npm run build` — PASS.

**Solution**
- В `globals.css` добавлено свойство `display: flex; flex-direction: column;` для `.sidebar` и класс `.sidebar-spacer` со свойством `flex-grow: 1`.
- В `nav-link` добавлен `display: flex; align-items: center; gap: 8px;` для корректного отображения иконок рядом с текстом.
- В `PrivateShell` из `lucide-react` импортированы иконки: `LayoutDashboard`, `Map`, `ListTodo`, `Library`, `Settings`.
- Массив `NAV_ITEMS` разделен на `MAIN_NAV_ITEMS` и `BOTTOM_NAV_ITEMS` (для настроек).
- Обе группы ссылок рендерятся в двух разных тегах `<nav>`, разделенных `.sidebar-spacer`.
- В `currentSection` теперь используется объединенный массив `ALL_NAV_ITEMS` для поиска текущего заголовка экрана.

**Tests / Verification**
- Запущен `cd frontend && npm run lint` — PASS.
- Запущен `cd frontend && npm run build` — PASS.
- Manual сценарий в этом ReMake-цикле не запускался.

**Impact**
- UI: Сайдбар теперь имеет иконки, страница "настройки" зафиксирована внизу.
- Обратная совместимость: Да.

**Risks & Rollback**
- Откат: вернуть старый код в `PrivateShell` (единый NAV_ITEMS без иконок) и удалить flexbox из `globals.css`.
