## TSK-0045 — Materials: убрать inline-блок «Добавить материал», вынести кнопку создания вправо над списком и открыть форму в modal
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Сейчас раздел материалов содержит отдельный inline-блок «Добавить материал» перед сеткой материалов.
- По требованию owner этот блок должен быть удалён: кнопка создания материала должна находиться справа сверху над областью материалов, а форма должна открываться во всплывающем окне с текущим содержимым.

### Acceptance Criteria
- [x] Inline-блок «Добавить материал» удалён из основного потока страницы материалов.
- [x] Кнопка «Создать материал» расположена справа в верхней части над областью материалов.
- [x] Клик по кнопке открывает modal с текущим содержимым формы создания материала (поля/валидации/submit-flow сохранены).
- [x] После заголовочного блока раздела и верхней строки управления сразу идёт список/сетка материалов, без отдельного create-блока между ними.
- [x] После успешного создания modal закрывается, новый материал появляется в текущем списке.
- [x] Закрытие modal поддержано (Cancel/Esc), маршрут не меняется.

### Constraints
- Не менять backend API-контракт материалов.
- Не менять существующий сценарий редактирования/удаления/обновления прогресса материалов.
- Не менять фильтры поиска и выбора темы.

### Change Summary
**Problem**
- На странице `/materials` форма создания материала находилась отдельным inline-блоком между header и списком карточек, что нарушало требуемую компоновку экрана.

**Root Cause Hypothesis**
- Create-flow материалов был изначально реализован только как встроенная панель (`MaterialsCreatePanel`) в основном потоке `MaterialsLibraryView`, без modal-триггера в верхней строке управления.

**Evidence**
- До изменения `frontend/src/components/materials-library-view.tsx` рендерил `MaterialsCreatePanel` сразу после `MaterialsLibraryHeader`.
- После изменения:
  - в `MaterialsLibraryHeader` добавлена кнопка `Создать материал` в правой части верхней строки;
  - inline-рендер формы убран из основного потока и перенесён в modal;
  - submit формы обёрнут в `handleCreateSubmit`, modal закрывается только при успешном создании.
- В `frontend/src/components/hooks/use-materials-library-view-model.ts` `handleCreate` переведён на `Promise<boolean>` для явного сигнала успеха.
- Командная проверка:
  - `npm run lint` → PASS;
  - `npm run build` → PASS.
- Manual smoke (фактический):
  - `/materials` на локальном сервере `:3026` подтверждает отсутствие inline create-блока;
  - create-кнопка расположена справа в header-control зоне и открывает modal;
  - `Cancel`/`Esc`/overlay закрывают modal без смены маршрута;
  - после создания тестовой темы в `/roadmap` submit создаёт материал, modal закрывается, новый элемент появляется в списке.

**Solution**
- В `frontend/src/components/materials-library-view.tsx` реализован modal create-flow материалов:
  - добавлено состояние открытия modal и trigger-ref для возврата фокуса;
  - форма создания оставлена прежней (`MaterialsCreatePanel`) и переиспользована внутри modal;
  - добавлено закрытие modal по `Cancel`, `Esc`, кнопке `×` и клику на overlay;
  - при успешном создании modal закрывается, затем данные списка обновляются существующим query invalidate.
- В `frontend/src/components/hooks/use-materials-library-view-model.ts` `handleCreate` возвращает `true/false` без изменения API-контракта.
- В `frontend/app/globals.css` добавлены стили для верхней строки управления (`filters + create button`) и modal-представления формы.

**Tests / Verification**
