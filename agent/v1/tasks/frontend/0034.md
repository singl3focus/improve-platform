## TSK-0034 — Добавить явный UI для создания, редактирования и удаления этапов roadmap
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Пользователю неочевидно, как создать новый этап roadmap.
- По решению owner нужны также редактирование и удаление этапов, а не только создание.

### Acceptance Criteria
- [x] В UI roadmap есть явные действия: «Создать этап», «Редактировать этап», «Удалить этап».
- [x] Действия сопровождаются понятными состояниями (`loading`, `error`, `success`) и confirm для удаления.
- [x] После операций список этапов и связанные темы в интерфейсе обновляются без ручной перезагрузки страницы.
- [x] Ключевые ошибки валидации отображаются пользователю читаемо.

### Constraints
- Не объединять с глубоким редизайном всего экрана roadmap.
- Не менять бизнес-правила backend по порядку/ограничениям этапов в этой задаче.

### Verification
- Command/Test:
  - `cd frontend && npm run lint`
  - Expected: линтер проходит без ошибок.
  - Result: PASS (`✔ No ESLint warnings or errors`)
  - Manual:
  - Expected: пользователь может создать, изменить и удалить этап через UI roadmap.
  - Result: NOT RUN

### Change Summary
**Problem**
- В roadmap-интерфейсе не было явных действий для CRUD этапов: пользователь не мог напрямую создавать, редактировать и удалять этапы.

**Root Cause Hypothesis**
- На frontend были реализованы CRUD-сценарии для тем и зависимостей, но отсутствовали API-прокси и UI-слой для операций над этапами.

**Evidence**
- В `frontend/src/components/roadmap-view.tsx` до изменения были формы и кнопки только для тем/зависимостей.
- В `frontend/app/api/roadmap` отсутствовали роуты `stages` для `POST/PUT/DELETE`.
- После внесения изменений `cd frontend && npm run lint` завершился успешно: `✔ No ESLint warnings or errors`.

**Solution**
- Добавлены frontend API-роуты для этапов roadmap:
  - `frontend/app/api/roadmap/stages/route.ts` (`POST`),
  - `frontend/app/api/roadmap/stages/[stageId]/route.ts` (`PUT`, `DELETE`).
- Расширен `frontend/src/components/roadmap-view.tsx`:
  - добавлены явные действия `Создать этап`, `Редактировать этап`, `Удалить этап`,
  - добавлены формы создания и редактирования этапа,
  - добавлен `confirm` перед удалением этапа,
  - добавлены состояния `loading/error/success` для мутаций этапов,
  - добавена валидация обязательного названия этапа и корректной позиции.
- Обновлен `frontend/app/globals.css` стилями для нового stage-UI и success-сообщения.
- После успешных операций вызывается `roadmap.reload()`, что обновляет этапы и связанные темы без ручного refresh страницы.

**Tests / Verification**
- Запущен `cd frontend && npm run lint` — PASS.
- Manual сценарии CRUD этапов локально не запускались в этом цикле.

**Impact**
- UI roadmap получил полноценные действия управления этапами с понятной обратной связью.
- API/бизнес-правила backend не изменялись.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: удаление этапа может возвращать backend-ошибку при бизнес-ограничениях (например, если есть связанные темы); ошибка показывается пользователю как есть.
- Rollback: удалить добавленные frontend API-роуты `frontend/app/api/roadmap/stages/*` и откатить stage-CRUD блоки в `frontend/src/components/roadmap-view.tsx` и `frontend/app/globals.css`.
