## TSK-0022 — Добавить удаление задачи с иконки на карточке и подтверждением
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-15
- Updated: 2026-03-15

### Context
- Backend уже поддерживает удаление задачи (`DELETE /api/v1/tasks/{taskID}`), но на канбан-карточке нет UI-действия удаления.
- По требованию owner на карточке должна быть иконка удаления задачи.
- Также требуется подтверждение удаления перед фактическим вызовом API.

### Acceptance Criteria
- [x] На карточке задачи в `/tasks` есть иконка/кнопка удаления.
- [x] Перед удалением показывается подтверждение действия (confirm).
- [x] Во frontend BFF добавлен endpoint удаления задачи (`DELETE /api/tasks/{taskId}`) с проксированием в backend.
- [x] После подтвержденного удаления карточка исчезает с доски без перезагрузки.
- [x] Ошибки удаления отображаются пользователю и не ломают текущее состояние доски.

### Constraints
- Не менять backend контракт удаления задачи.
- Не удалять задачу без явного подтверждения пользователя.
- Ограничить изменения экраном `/tasks` и BFF API для задач.

### Change Summary
**Problem**
- После ревью задача ушла в `ReMake`: не было проходящей end-to-end верификации удаления задачи в smoke-скрипте.
- Не было воспроизводимого сигнала на сценарий ошибки удаления (база для проверки rollback-поведения UI).

**Root Cause Hypothesis**
- В `frontend/scripts/mvp-verification-smoke.mjs` отсутствовал сценарий `create -> DELETE -> GET without task`.
- Mock backend внутри smoke-скрипта не поддерживал `DELETE /api/v1/tasks/{id}` и режим принудительной ошибки удаления.

**Evidence**
- `frontend/scripts/mvp-verification-smoke.mjs:750` — добавлен сценарий `Tasks DELETE route keeps success and removes task from board response`.
- `frontend/scripts/mvp-verification-smoke.mjs:773` — добавлен сценарий `Tasks DELETE route keeps task intact when backend delete fails`.
- `frontend/scripts/mvp-verification-smoke.mjs:93,149-160,409` — добавлен `taskDeleteMode` и принудительная ошибка удаления в mock backend.
- `frontend/src/components/tasks-kanban-view.tsx:528-529` — подтверждение удаления через `window.confirm(...)`.
- `frontend/src/components/hooks/use-tasks-board-view-model.ts:409-443` — optimistic удаление и rollback к `previousTasks` в `catch`.
- Команды:
  - `npm run lint` → PASS.
  - `npm run build` → PASS (после повторного запуска вне sandbox).
  - `FRONTEND_BASE_URL=http://127.0.0.1:3032 npm run verify:mvp` → PASS (все сценарии PASS).
  - `npx tsc --noEmit` → PASS.

**Solution**
- В mock backend smoke-скрипта добавлен обработчик `DELETE /api/v1/tasks/{id}` и режим ошибки `taskDeleteMode=error`.
- Добавлен smoke-сценарий успешного удаления:
  - создать задачу через `POST /api/tasks`,
  - удалить через `DELETE /api/tasks/{taskId}`,
  - проверить отсутствие задачи в повторном `GET /api/tasks`.
- Добавлен smoke-сценарий неуспешного удаления:
  - форсировать backend-ошибку (`taskDeleteMode=error`),
  - убедиться, что `DELETE` возвращает `500`,
  - убедиться, что задача остаётся в списке,
  - вернуть `taskDeleteMode=ok` и удалить задачу для cleanup.
- Для UI-proof зафиксированы точные точки confirm/optimistic/rollback в коде (`tasks-kanban-view.tsx`, `use-tasks-board-view-model.ts`).

**Tests / Verification**
- Запущено `npm run lint` — PASS.
- Запущено `npm run build` — PASS (после повторного запуска вне sandbox; в sandbox был `EPERM`).
- Запущено `FRONTEND_BASE_URL=http://127.0.0.1:3032 npm run verify:mvp` — PASS (все smoke-сценарии PASS, включая оба delete-сценария).
- Запущено `npx tsc --noEmit` — PASS.
- Ручной UI smoke в браузере не запускался в рамках этой сессии (воспроизводимые шаги зафиксированы в Notes).

**Impact**
- Runtime-контракты frontend/backend не изменены.
- Изменения в этом цикле ограничены smoke-скриптом верификации и документацией задачи.
