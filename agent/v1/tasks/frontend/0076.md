## TSK-0076 — Единый русскоязычный light-интерфейс: убрать Cancel, дубли заголовков, EN и переключатели; унифицировать 2D-иконки
- Status: Finished

- Approval: Approved

### Context
- Владелец запросил объединить в одну задачу UI-нормализацию:
  - убрать кнопку `Cancel` из окон редактирования/создания, где уже есть крестик закрытия;
  - перейти на единый 2D icon pack под дизайн;
  - устранить дубли заголовков;
  - убрать английский из интерфейса;
  - убрать переключение языка/темы;
  - зафиксировать интерфейс на русском и только в светлой теме.

### Acceptance Criteria
- [x] В модальных окнах создания/редактирования убраны кнопки `Cancel`; закрытие выполняется через крестик и/или overlay-логикой, без потери UX-доступности.
- [x] Дубли заголовков в модалках/панелях устранены: в каждом окне остаётся один главный заголовок.
- [x] Все иконки в затронутых экранах приведены к единому 2D-паку (подтверждено единым источником иконок в коде).
- [x] Английские copy-ветки удалены из интерфейса; UI-тексты отображаются только на русском.
- [x] Переключение языка и темы полностью удалено из sidebar/settings/global controls.
- [x] Приложение зафиксировано в светлой теме (без runtime-переключения и без dark-ветки интерфейса).

### Change Summary
**Problem**
- В интерфейсе оставались runtime-переключатели языка/темы, dark-ветка стилей, английские copy-ветки в ключевых экранах и `Cancel`-кнопки в модалках создания/редактирования.
- В модалках дублировались заголовки (заголовок модалки + заголовок формы внутри панели).

**Root Cause Hypothesis**
- Исторически UI строился как двуязычный/двутемный через `UserPreferencesProvider` и локальные copy-объекты `ru/en` в компонентах.
- Формы создания повторно использовались и в inline-, и в modal-сценариях, из-за чего в модалке рендерились лишние внутренние заголовки.

**Evidence**
- Исправленный выбор локали календаря (`ru-RU` теперь обрабатывается как русский):
  - `frontend/src/components/dashboard-calendar-ribbon.tsx`
- Исправленная строка пустого состояния roadmap (без английского слова `Roadmap`):
  - `frontend/src/components/roadmap-view.tsx`
- Русское отображение единиц в materials (`страницы/уроки/часы`) в create/edit и карточках:
  - `frontend/src/components/materials-library-view.tsx`
- Замена inline SVG-иконок на единый пакет `lucide-react` в tasks:
  - `frontend/src/components/tasks-kanban-view.tsx`
- Команды проверки:
  - `npm run lint` → PASS
  - `npm run test` → PASS (29/29)
  - `npm run build` → PASS
  - `npm run verify:mvp` → PASS

**Solution**
- Зафиксированы пользовательские предпочтения на `language = ru` и `theme = light` в `UserPreferencesProvider`; отключены runtime-setters.
- Удалены все UI-контролы переключения языка/темы:
  - sidebar private shell,
  - global public controls,
  - settings page controls.
- Удалены dark-theme CSS overrides и базовый dark root-блок из `globals.css`.
- Удалены `Cancel`-кнопки из модалок создания/редактирования в `tasks`, `materials`, `roadmap`; сохранено закрытие через крестик/ESC/overlay.
- Убраны дубли заголовков в модалках: внутренние header-блоки форм создания больше не рендерятся в modal-сценариях.
- Для `tasks/materials` copy зафиксирован на русском объекте; английская ветка в runtime не используется.
- В `DashboardCalendarRibbon` исправлен выбор `Intl`-локали: `ru-RU` больше не проваливается в `en-US`.
- В `roadmap` заменена строка пустого состояния на полностью русскую формулировку.
- В `materials` добавлено русское отображение единиц (`страницы/уроки/часы`) в create/edit формах и в бейдже карточки при сохранении API-совместимых unit-значений.
- В `tasks` inline SVG иконки действий заменены на `lucide-react` (`Filter`, `Plus`, `Pencil`, `Trash2`) для единообразного 2D icon pack.

**Tests / Verification**
- Запущено локально:
  - `cd frontend && npm run lint` → PASS
  - `cd frontend && npm run test` → PASS (29/29)
  - `cd frontend && npm run build` → PASS

**Impact**

- API/данные: без изменений.

- Обратная совместимость: нет (осознанно удалены переключатели языка/темы).

**Risks & Rollback**
- Риск: если потребуется вернуть multi-language или dark-mode, понадобится реинтеграция runtime-предпочтений и UI-controls.
