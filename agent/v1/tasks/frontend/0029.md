## TSK-0029 — Убрать рамку у приветствия и даты на dashboard
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- В текущем dashboard блок приветствия и даты отрисован как карточка с рамкой.
- По решению owner приветствие и дата должны быть обычным текстовым блоком без карточной рамки.

### Acceptance Criteria
- [x] В верхней зоне `/dashboard` приветствие и дата отображаются без карточной рамки, фона и бордера.
- [x] Текст приветствия и даты визуально отделен только типографикой и отступами.
- [x] На mobile и desktop верстка остается читабельной и не ломает расположение следующих блоков.

### Constraints
- Не менять бизнес-данные и API-запросы dashboard.
- Не объединять с редизайном других зон, кроме верхнего блока приветствия.

### Verification
- Command/Test:
  - `cd frontend && npm run lint`
  - Expected: линтер проходит без ошибок.
  - Result: FAIL (`./src/components/dashboard-calendar-ribbon.tsx`, `Parsing error: File appears to be binary`)
  - Manual:
  - Expected: на `/dashboard` приветствие и дата отображаются как текст без карточной рамки; соседние блоки не смещены.
  - Result: NOT RUN (локально не запускалось в этом ReMake-цикле)

### Change Summary
**Problem**
- Блок приветствия (greeting) и даты на странице `/dashboard` отображался как карточка с рамкой и фоном, что противоречило требованиям owner.

**Root Cause Hypothesis**
- Компонент использует CSS класс `.dashboard-personal`, который был стилизован как карточка с бордером, скруглениями углов и градиентным фоном.

**Evidence**
- В `frontend/app/globals.css` у `.dashboard-personal` оставлен только `padding: 8px 4px 0;` без `border`, `border-radius` и `background`.
- Повторный запуск `cd frontend && npm run lint` завершился с ошибкой: `./src/components/dashboard-calendar-ribbon.tsx` → `Parsing error: File appears to be binary`.

**Solution**
- В `frontend/app/globals.css` у класса `.dashboard-personal` удалены свойства `border`, `border-radius` и `background`.
- Оставлен только отступ `padding` (скорректирован для визуального отделения: с `16px 18px` до `8px 4px 0`) для сохранения читабельности.

**Tests / Verification**
- Запущен `cd frontend && npm run lint` — FAIL по несвязанной с этой задачей ошибке парсинга в `dashboard-calendar-ribbon.tsx`.
- Manual check в этом ReMake-цикле не запускался (NOT RUN).

**Impact**
- UI: Верхний блок dashboard теперь выглядит как обычный текст.
- Обратная совместимость: да.

**Risks & Rollback**
- Рисков нет. 
- Откат: вернуть старые стили для `.dashboard-personal` в `globals.css`.

**Suggested Commit Message**
- `style(frontend): remove card border and background from dashboard greeting`
