### Change Summary
**Problem**
- В shell присутствовал глобальный заголовок `h1`, который дублировал заголовки экранов.
- Кнопка logout в нижнем блоке sidebar не была центрирована.
- Переключатели темы/языка были доступны только на странице settings, а не глобально на публичных и приватных экранах.

**Root Cause Hypothesis**
- Заголовок секции был захардкожен в `PrivateShell` и рендерился для всех приватных страниц.
- Logout-экшен стилизован через `align-self: flex-start`, поэтому не центрировался.
- Контролы language/theme были реализованы локально в `settings/page.tsx` и не выводились в общем layout.

**Evidence**
- `frontend/src/components/layout/private-shell.tsx`: присутствовал `<header><h1>...</h1></header>` и `sidebar-text-action` внизу.
- `frontend/app/(private)/settings/page.tsx`: language/theme селекты находились только в settings-странице.
- `frontend/app/layout.tsx`: не было глобального компонента для language/theme контролов.
- Команды проверки:
  - `npm run lint` -> PASS.
  - `npm run build` -> PASS.

**Solution**
- Удален глобальный `h1`-header из `PrivateShell` вместе с вычислением `currentSection`.
- Изменены стили нижнего nav-блока sidebar: logout центрирован и остается под settings.
- Добавлены переиспользуемые контролы `PreferencesControls`.
- Добавлена глобальная панель `GlobalUiControls` и подключена в root layout, чтобы language/theme были доступны на всех экранах.
- Страница settings переведена на использование общего компонента контролов для единообразного поведения.

**Tests / Verification**
- `Set-Location frontend; npm run lint` -> PASS.
- `Set-Location frontend; npm run build` -> PASS (сервисный `PathNotFound` от повторного `Set-Location` в уже открытой директории не помешал успешной сборке).
- Manual UI не запускался локально в рамках задачи.

**Impact**
- UI/layout: изменены shell и глобальный слой контролов.
- Auth flow: напрямую не менялся; build и lint подтверждают отсутствие compile/lint регрессий.
- Данные/API/security: без изменений.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: на узких экранах глобальная панель может конкурировать за место с контентом.
- Rollback: удалить `GlobalUiControls` из `frontend/app/layout.tsx`, удалить новые компоненты контролов и вернуть прежний блок header/logout-стилей.
