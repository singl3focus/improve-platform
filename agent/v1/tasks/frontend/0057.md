## TSK-0057 — Исправить UX-дефекты модалок roadmap (позиция, ошибки, дубли edit)
- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-17
- Updated: 2026-03-17

### Context
- В roadmap остались UX-дефекты:
  - в карточке темы одновременно отображаются две кнопки редактирования;
  - в модалках создания этапа/темы блок `Позиция` визуально конфликтует с окном;
  - ошибки из `Создать этап` и `Добавить тему` выводятся на странице, а не в самом окне.

### Acceptance Criteria
- [x] В карточке темы остается только одна кнопка редактирования темы; редактирование этапа вынесено из карточки темы.
- [x] В модалке `Создать этап` поле/блок `Позиция` не ломает layout окна и не выходит за его границы.
- [x] В модалке `Добавить тему` поле/блок `Позиция` не ломает layout окна и не выходит за его границы.
- [x] Ошибки в сценариях `Создать этап` и `Добавить тему` отображаются внутри соответствующих модалок.
- [x] После закрытия/повторного открытия модалки старые ошибки не показываются повторно без нового fail-сценария.

### Constraints
- Не менять endpoint-ы roadmap.
- Не изменять бизнес-правила позиций на backend (валидность/порядок остаются прежними).
- Не ухудшать доступность (focus trap/ESC/кнопка закрытия сохраняются).

### Change Summary
**Problem**
- В roadmap были UX-дефекты: две визуально похожие edit-кнопки в карточке темы, конфликт layout поля `Позиция` в create-модалках и вывод create-ошибок вне модального окна.

**Root Cause Hypothesis**
- В карточке одновременно рендерились действия редактирования этапа и темы.
- Формы в create-панелях использовали общую 3-колоночную сетку, где блок `Позиция` конкурировал с остальными полями в тесной модалке.
- Ошибки `stageMutationError`/`topicMutationError` рендерились в нижней части страницы общим блоком и не были привязаны к create-модалкам.

**Evidence**
- `frontend/src/components/roadmap-view.tsx`: из карточки темы удален `roadmap-stage-edit-trigger`; оставлен один edit-триггер темы.
- `frontend/src/components/roadmap-view.tsx`: в `StageMutationPanel` и `TopicMutationPanel` добавлен `error` prop и рендер `dashboard-error roadmap-modal-error` внутри модальных панелей.
- `frontend/src/components/roadmap-view.tsx`: глобальные блоки ошибок ограничены условиями `!isTopicCreateModalOpen` и `!isStageCreateModalOpen`, чтобы create-ошибки не дублировались вне окна.
- `frontend/app/globals.css`: добавлен класс `roadmap-topic-field-position`; для create-панелей введена 2-колоночная сетка и вынос поля `Позиция` на всю ширину (`grid-column: 1 / -1`).

**Solution**
- Удалено редактирование этапа из карточки темы, чтобы убрать дублирующийся edit-UX в карточке.
- Перенесен показ ошибок create-сценариев внутрь соответствующих модалок этапа/темы.
- Стабилизирован layout модалок create: поле `Позиция` больше не конфликтует с сеткой и не выходит за границы окна.
- Добавлен явный сброс create-ошибок при закрытии модалок (`closeStageCreateModal`/`closeTopicCreateModal`), чтобы старые ошибки не попадали во внешний блок после закрытия.
