## TSK-0063 — Убрать stage-зависимость из Roadmap UI и BFF
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-17
- Updated: 2026-03-18

### Context
- Критические баги подтверждены в браузере:
  - создание этапа на Roadmap падает с `404 Not Found`;
  - добавление темы через модалку Add topic падает с `422 Stage does not exist in roadmap`.
- Причина: frontend и BFF продолжают использовать stage-сценарии, хотя backend-контракт roadmap уже stage-free.

### Acceptance Criteria
- [x] На странице Roadmap отсутствует stage-управление (Create stage, edit/delete stage, stage модалки).
- [x] Модалка Add topic работает без выбора stage и без отправки stage-полей.
- [x] Создание темы в Roadmap успешно проходит по stage-free контракту backend.
- [x] В BFF роуте создания темы удалена stage-валидация, вызывающая `Stage does not exist in roadmap`.
- [x] После изменений в Roadmap не возникает `404` по stage endpoint и `422` из-за stage-валидации при создании темы.

### Constraints
- Сохранить бизнес-инварианты topic creation (валидация title/description/position, user scope, обработка backend ошибок).
- Не изменять backend-контракты в рамках этой задачи.
- Не использовать git-команды.
