### Change Summary
**Problem**
- В `roadmap/layout/materials`-частях frontend оставалась связанная техническая проблема: layout-расчёты графа roadmap и парсинг/нормализация числовых полей materials были разбросаны по компонентам, а тестовый раннер покрывал только один модуль (`roadmap-graph`).
- После рефакторинга smoke-проверка (`verify:mvp`) падала множественными `404` в BFF-сценариях roadmap/topics/tasks/materials/dashboard при конфигурации `BACKEND_API_URL` с суффиксом `/api/v1`.

**Root Cause Hypothesis**
- Логика вычисления roadmap-связей и логика обработки числовых полей материалов не были вынесены в переиспользуемые `lib`-модули.
- `npm test` был настроен только на `.test-dist/roadmap-graph.test.js`, поэтому регрессии в `layout/materials` не отслеживались.
- В `frontend/src/lib/backend-shared.ts` функция `getBackendApiUrl(...)` конкатенировала `BACKEND_API_URL` и путь без нормализации префикса `/api/v1`, из-за чего при значении `http://127.0.0.1:8080/api/v1` формировались URL вида `/api/v1/api/v1/...`.

**Evidence**
- До изменений `frontend/src/components/roadmap/use-roadmap-graph-layout.ts` содержал inline-расчёт связей и размера графа.
- До изменений `frontend/src/components/hooks/use-materials-library-view-model.ts` содержал локальные `parsePositiveInteger`, `parseProgressPercent`, `normalizeProgressPercent`.
- До изменений `frontend/tsconfig.test.json` включал только `src/lib/roadmap-graph.ts` и `src/lib/roadmap-graph.test.ts`.
- До исправления ReMake `frontend/src/lib/backend-shared.ts` формировал backend URL прямой конкатенацией base+path.
- Проверка после изменений:
  - `npm run test` -> `17 passed, 0 failed`.
  - Новый тест `backend-shared.test.ts` подтверждает отсутствие дублирования `/api/v1`.
  - `npm --prefix frontend run verify:mvp` -> все сценарии PASS, без `404` для roadmap/topics/tasks/materials/dashboard.

**Solution**
- Вынесена layout-логика roadmap в новый модуль `frontend/src/lib/roadmap-layout.ts`:
  - `buildRoadmapConnections(...)`
  - `readGraphSize(...)`
- `frontend/src/components/roadmap/use-roadmap-graph-layout.ts` переведён на использование нового `lib`-модуля.
- Вынесена логика числовых полей materials в новый модуль `frontend/src/lib/materials-form.ts`:
  - `parsePositiveInteger(...)`
  - `parseProgressPercent(...)`
  - `normalizeProgressPercent(...)`
- `frontend/src/components/hooks/use-materials-library-view-model.ts` переведён на переиспользование `materials-form`.
- Добавлены unit-тесты:
  - `frontend/src/lib/roadmap-layout.test.ts`
  - `frontend/src/lib/materials-form.test.ts`
- Добавлен unit-тест `frontend/src/lib/backend-shared.test.ts` для `getBackendApiUrl(...)`:
  - сценарий с `BACKEND_API_URL=http://127.0.0.1:8080/api/v1` больше не даёт дублирования `/api/v1`;
  - сценарий с host-only URL (`http://127.0.0.1:8080`) сохраняет прежнее поведение.
- Расширен test-runner:
  - `frontend/tsconfig.test.json` включает новые test/target файлы;
  - `frontend/package.json` обновлён: `npm test` запускает `node --test .test-dist` (Windows-совместимо).
- В `frontend/src/lib/backend-shared.ts` обновлена `getBackendApiUrl(...)`: если base уже оканчивается на `/api/v1` и путь начинается с `/api/v1`, повторный префикс не добавляется.

**Impact**
- Затронуто:
  - Внутренняя организация frontend-кода (`roadmap` layout helper, `materials` numeric helper).
  - Unit-покрытие и конфигурация `npm test`.
  - Нормализация формирования backend URL в BFF (`backend-shared`).
- Не затронуто:
  - Публичные API-контракты frontend/backend.
  - Бизнес-логика пользовательских сценариев.
- Обратная совместимость: да.

**Risks & Rollback**
- Риски:
  - Потенциальная функциональная эквивалентность выноса helper-логики зависит от корректности unit-тестов и runtime smoke.
  - `build` в текущем окружении требует увеличенного heap.
  - Возможны расхождения, если backend URL будет использовать иной префикс версии API, отличный от `/api/v1`.
- Rollback plan:
  - удалить `frontend/src/lib/roadmap-layout.ts` и `frontend/src/lib/materials-form.ts` и вернуть прежнюю inline-логику в соответствующих компонентах;
  - удалить новые тесты `frontend/src/lib/roadmap-layout.test.ts` и `frontend/src/lib/materials-form.test.ts`;
  - вернуть прежнюю реализацию `getBackendApiUrl(...)` в `frontend/src/lib/backend-shared.ts` и удалить `frontend/src/lib/backend-shared.test.ts`;
  - вернуть исходные `frontend/tsconfig.test.json` и `frontend/package.json` для прежнего test-runner.
