## TSK-0001 — Базовый каркас frontend (Next.js + Tailwind + shadcn/ui)
- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-14
- Updated: 2026-03-14

### Context
- Сейчас frontend отсутствует, нужен стартовый каркас для MVP экранов.
- Тех-стек задан: Next.js App Router, TypeScript, Tailwind, shadcn/ui.
- Появились визуальные референсы в `design/`, каркас должен поддержать их единый стиль без копирования статического HTML.

### Acceptance Criteria
- [ ] Создано приложение на Next.js App Router с TypeScript и базовой структурой модулей.
- [ ] Подключены Tailwind CSS и shadcn/ui с единым стилевым токеном для статусов.
- [ ] Реализован базовый layout: `Sidebar + Header + Main`, маршруты-заглушки для Dashboard/Roadmap/Topics/Tasks/Materials/Settings.
- [ ] Приложение запускается локально и корректно отображается на desktop и мобильной ширине.
- [ ] Базовые UI-токены (цвета статусов, типографика, отступы) позволяют реализовывать экраны по визуальным референсам из `design/`.

### Constraints
- Без избыточного глобального state manager на старте.
- Держать стиль интерфейса в рамках `clean, modern, calm`.
- Не превращать layout в перегруженный enterprise-dashboard.
- `design/*` использовать только как визуальный референс; функциональные требования задаются задачами и `docs/bussines/v1/*`.
- Не копировать макеты из `design/` один-в-один как готовый production UI.

### Verification
- Command/Test:
  - `npm run dev` (или `pnpm dev`)
  - Expected: приложение поднимается без ошибок, базовая навигация работает.
  - Result: NOT RUN (заполняется Frontend-Developer)
  - UI smoke: переходы между разделами
  - Expected: роутинг и layout стабильны.
  - Result: NOT RUN (заполняется Frontend-Developer)
