## TSK-0025 — Управление зависимостями тем в Roadmap (связи графа)
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Визуализация зависимостей в Roadmap есть, но пользователь не может полноценно управлять связями тем из UI.
- По требованию owner roadmap должен быть “настоящим графом”: добавить ноду и создать/удалить связь между нодами.
- Без редактирования зависимостей невозможна живая настройка последовательности обучения.

### Acceptance Criteria
- [x] На `/roadmap` можно добавить зависимость между двумя существующими темами (topic -> prerequisite topic).
- [x] На `/roadmap` можно удалить существующую зависимость темы.
- [x] Ошибки backend по `self_dependency` и `cycle_detected` отображаются пользователю как понятные сообщения.
- [x] После изменения связи графовые ребра и признаки блокировки тем обновляются без ручного reload.
- [x] BFF-слой покрывает операции `POST/DELETE` зависимостей с нормализованной обработкой ошибок.

### Constraints
- Строго сохранить DAG: нельзя обходить валидацию циклов.
- Не смешивать с CRUD тем (это отдельный TSK).
- Не менять backend-контракт зависимостей (`/api/v1/roadmap/topics/{topicID}/dependencies`).

### Change Summary
**Problem**
- В roadmap отображались ребра зависимостей, но пользователь не мог управлять ими из UI.
- Не было frontend/BFF механизма для `POST/DELETE` зависимостей, поэтому невозможно было построить/исправить связи DAG на экране `/roadmap`.

**Root Cause Hypothesis**
- `RoadmapView` реализовывал только визуализацию `prerequisiteTopicIds` и не содержал форм/действий для мутаций зависимостей.
- В `frontend/app/api/roadmap/...` отсутствовали маршруты BFF для операций по `/api/v1/roadmap/topics/{topicID}/dependencies`.

**Evidence**
- В UI добавлены обработчики и панель управления зависимостями:
  - `createRoadmapDependency`/`deleteRoadmapDependency` (`frontend/src/components/roadmap-view.tsx:382`, `:406`);
  - обработка ошибок по code (`self_dependency`, `cycle_detected`) через `getDependencyErrorMessage` (`frontend/src/components/roadmap-view.tsx:297`);
  - формы и действия `handleDependencyCreate`/`handleDependencyDelete` (`frontend/src/components/roadmap-view.tsx:862`, `:891`);
  - новый блок `roadmap-dependency-mutation-panel` и remove-кнопка в dependency chips (`frontend/src/components/roadmap-view.tsx:1028`, `:1206`).
- В BFF добавлены маршруты:
  - `POST /api/roadmap/topics/[topicId]/dependencies` (`frontend/app/api/roadmap/topics/[topicId]/dependencies/route.ts:19`);
  - `DELETE /api/roadmap/topics/[topicId]/dependencies/[dependencyTopicId]` (`frontend/app/api/roadmap/topics/[topicId]/dependencies/[dependencyTopicId]/route.ts:15`).
- Нормализованная обработка backend-ошибок в BFF использует `createBackendErrorResponse` с сохранением `code` в JSON-ответе (`.../dependencies/route.ts:47`, `...[dependencyTopicId]/route.ts:29`).
- Проверка командами:
  - `npm run lint` -> PASS.
  - `npx tsc --noEmit` -> PASS.

**Solution**
- Добавлен BFF-слой для зависимостей тем:
  - `POST` принимает `prerequisiteTopicId`, проксирует в backend как `depends_on_topic_id`;
  - `DELETE` удаляет связь по паре `topicId + dependencyTopicId`;
  - backend-ошибки проходят через общий нормализатор с кодом и сообщением.
- На `/roadmap` добавлена отдельная панель dependency management:
  - выбор темы и prerequisite темы;
  - добавление связи;
  - отображение ошибок мутаций.
- На каждом dependency-chip темы добавлена кнопка удаления связи.
- После add/delete зависимости вызывается `roadmap.reload()`, поэтому ребра графа и признаки блокировок пересчитываются и отображаются без ручного refresh страницы.
- Добавлены стили для dependency панели и remove-кнопок в `frontend/app/globals.css` (включая dark/mobile ветки).

**Tests / Verification**
- Запущено:
  - `cd frontend && npm run lint` — PASS.
  - `cd frontend && npx tsc --noEmit` — PASS.
- Не запускалось локально:
  - Ручной browser smoke для сценариев create/delete зависимостей и проверки `cycle_detected`/`self_dependency` на `/roadmap`.

**Impact**
- UI/UX: пользователь может добавлять и удалять связи между темами непосредственно на `/roadmap`.
- API/BFF: добавлены новые frontend маршруты зависимостей в `/api/roadmap/topics/[topicId]/dependencies*`.
- DAG-инварианты: сохранены, проверка циклов и self-dependency остается на backend; frontend только показывает нормализованные ошибки.
- Обратная совместимость: да, существующее чтение roadmap и отрисовка графа не ломаются.

**Risks & Rollback**
- Риск: без ручного e2e smoke не подтверждено браузерно поведение всех пользовательских сценариев add/remove.
- Риск: локализация ошибок опирается на `code` из backend-ответа; при изменении кодов потребуется синхронизация сообщений.
- Rollback plan:
  - удалить BFF-маршруты `frontend/app/api/roadmap/topics/[topicId]/dependencies/*`;
  - убрать dependency management UI/handlers из `frontend/src/components/roadmap-view.tsx`;
  - удалить добавленные dependency CSS-классы из `frontend/app/globals.css`.
