## TSK-0081 — Roadmap wheel-zoom: внутри графа не прокручивать страницу
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-21
- Updated: 2026-03-21 (Reviewer)

### Context
- На roadmap колесо мыши уже меняет масштаб графа, но одновременно продолжает прокручивать страницу.
- По пользовательскому контракту поведение должно быть разделено по зоне курсора:
  - внутри окна с карточками колесо меняет масштаб roadmap;
  - вне окна с карточками колесо прокручивает страницу как обычно.
- В `frontend/src/components/roadmap-view.tsx` desktop-ветка `handleGraphWheel` уже вызывает `event.preventDefault()`, но по фактическому сценарию этого недостаточно.

### Acceptance Criteria
- [x] При desktop viewport и положении курсора внутри `roadmap-graph` wheel изменяет только scale/transform графа и не меняет scroll страницы.
- [x] При положении курсора вне `roadmap-graph` поведение страницы остаётся стандартным: wheel scroll работает как раньше.
- [x] Mobile/tablet ветка не деградирует: scroll и доступность roadmap остаются рабочими при ширине `<= 960px`.
- [x] После исправления не ломаются pan, canvas controls и клики по карточкам после wheel-zoom.
- [x] Добавлены/обновлены frontend-проверки и выполнен ручной desktop browser smoke на `/roadmap`.

### Constraints
- Не ломать текущую desktop-логику pan/zoom из `TSK-0078`.
- Не ухудшать mobile scroll fallback.
- Не использовать git-команды.

### Change Summary
**Problem**
- В desktop roadmap колесо мыши внутри графа должно было менять только zoom, но page scroll продолжал срабатывать.
- После перехода на native wheel-listener во время live smoke обнаружился второй дефект: listener вешался до появления `.roadmap-graph` после загрузки данных и не активировался на реальной сцене.

**Root Cause Hypothesis**
- React `onWheel` с `preventDefault()` оказался ненадёжной точкой для подавления document scroll в этом canvas-сценарии.
- Компонент roadmap сначала рендерит loading/error ветки; `useEffect([])` для native wheel-listener видел `graphRef.current === null` и больше не перевешивался после появления графа.
- В проекте не было отдельной unit-проверки контракта `desktop zoom blocks page scroll / mobile keeps default scroll`.

**Evidence**
- В `frontend/src/components/roadmap-view.tsx` убран React `onWheel`; добавлен native `wheel` listener с `{ passive: false }` на `.roadmap-graph` и lifecycle-привязка через `roadmap.state.status`/`stages.length`.
- В `frontend/src/lib/roadmap-graph.ts` добавлен helper `getRoadmapWheelZoomBehavior`, а в `frontend/src/lib/roadmap-graph.test.ts` — новые кейсы для desktop/mobile/zero-delta.
- В `frontend/app/globals.css` добавлен desktop `overscroll-behavior: contain` для `.roadmap-graph` и mobile override `overscroll-behavior: auto`.
- Desktop browser smoke artifact `frontend/output/playwright/tsk-0081-review-1774095983221.json` подтверждает:
  - inside graph: `scale 1 -> 1.12`, `scrollY 65 -> 65`;
  - pan after wheel: offset `(-66.6, -33.6) -> (-196.6, 46.4)`;
  - outside graph: `scrollY 65 -> 168` без изменения transform;
  - click after wheel/control: переход на `/topics?topicId=topic-b`.
- Команды:
  - `cd frontend && npm run test` -> PASS (`53 passed, 0 failed`);
  - `cd frontend && npm run lint` -> PASS;
  - `cd frontend && npm run build` -> PASS.

**Solution**
- Вынесен контракт обработки wheel в `getRoadmapWheelZoomBehavior`, чтобы единообразно различать desktop/mobile поведение и покрыть его unit-тестами.
- Для `.roadmap-graph` добавлен native non-passive `wheel` listener, который:
  - блокирует page scroll только на desktop (`> 960px`);
  - обновляет scale/offset по anchor point под курсором;
  - не вмешивается в mobile/tablet fallback.
- Исправлен lifecycle listener-а: effect теперь перевешивается, когда roadmap переходит в состояние с реальным графом после загрузки.
- Добавлена CSS-подстраховка против scroll chaining на desktop и сохранён прежний mobile scroll через media override.

**Tests / Verification**
- `cd frontend && npm run test` -> PASS (`53 passed, 0 failed`).
- `cd frontend && npm run lint` -> PASS.
- `cd frontend && npm run build` -> PASS.
- Desktop browser smoke (`Edge + playwright-core`, `http://127.0.0.1:3037/roadmap`, mock backend `127.0.0.1:8084`, viewport `1440x700`) -> PASS:
  - inside graph wheel меняет только zoom;
  - outside graph wheel скроллит страницу;
  - pan после wheel работает;
  - canvas control `+` работает;
  - mouse-click по карточке после wheel/control открывает `/topics?topicId=topic-b`.
- Примечание по окружению:
  - `npm run test` и `npm run build` в sandbox падали на `spawn EPERM`; финальный PASS получен повторным запуском вне sandbox.

**Impact**
- Затронут только frontend roadmap UX: поведение wheel-zoom, scroll chaining и lifecycle подключения desktop wheel-listener.
- API, данные и backend-контракты не менялись.
- Mobile/tablet fallback сохранён: при ширине `<= 960px` helper не блокирует scroll, а CSS возвращает `overscroll-behavior: auto`.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: если roadmap в будущем будет монтировать `.roadmap-graph` по другому lifecycle-сценарию, зависимости effect для listener-а могут потребовать пересмотра.
- Риск: browser-specific нюансы wheel events на экзотических тачпадах/браузерах могут потребовать дополнительного smoke вне Edge.
- Rollback plan:
  - вернуть изменения в `frontend/src/components/roadmap-view.tsx`;
  - удалить helper/test changes в `frontend/src/lib/roadmap-graph.ts` и `frontend/src/lib/roadmap-graph.test.ts`;
  - убрать `overscroll-behavior` правки из `frontend/app/globals.css`.
