## TSK-0069 — Убрать описание из карточки темы roadmap
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-18
- Updated: 2026-03-18

### Context
- Владелец продукта запросил упростить карточку темы на roadmap: в карточке должно отображаться только название темы без текстового описания.
- Сейчас описание темы рендерится в карточке, что визуально перегружает граф и снижает читаемость карты зависимостей.

### Acceptance Criteria
- [x] В карточке темы на странице roadmap отображается только название темы (без description) для всех статусов темы.
- [x] Высота/верстка карточки не ломаются после удаления description на desktop и mobile.
- [x] Кликабельность карточки и переход к деталям темы остаются без изменений.
- [x] Счетчики задач/материалов, прогресс и статус-бейдж продолжают отображаться корректно.

### Constraints
- Не менять контракт backend API и структуру данных roadmap.
- Не затрагивать экран деталей темы и другие страницы, кроме поведения карточек roadmap.
- Изменения должны быть минимальными и сфокусированными только на удалении description из карточки.
- Не использовать git-команды.

### Change Summary
**Problem**
- В карточке темы roadmap отображался текст `description`, из-за чего карточка визуально перегружалась.

**Root Cause Hypothesis**
- Шаблон карточки в `roadmap-view.tsx` явно рендерил `topic.description` внутри блока `roadmap-topic-center`.

**Evidence**
- Код: `frontend/src/components/roadmap-view.tsx` (рендер карточки темы) содержал `<p className="roadmap-topic-description">{topic.description}</p>`.
- Проверка сборки/линтинга:
  - `Set-Location frontend; npm run lint` → PASS, `No ESLint warnings or errors`.
  - `Set-Location frontend; npm run build` → PASS, `Compiled successfully`.

**Solution**
- Удален рендер абзаца с описанием темы из карточки roadmap в `frontend/src/components/roadmap-view.tsx`.
- Остальные элементы карточки не изменялись: заголовок темы, статус-бейдж, прогресс, счетчики задач/материалов, обработчики клика и навигация.

**Tests / Verification**
- `Set-Location frontend; npm run lint` — PASS.
- `Set-Location frontend; npm run build` — PASS.
- Ручная проверка desktop/mobile — NOT RUN (нужно выполнить в браузере по шагам из секции Verification).

**Impact**
- Влияние: только UI карточки темы на странице roadmap.
- Backend API/контракты данных: без изменений.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: возможна избыточная пустота карточки на отдельных разрешениях; функциональный риск низкий.
- Rollback: вернуть удаленную строку рендера `topic.description` в `frontend/src/components/roadmap-view.tsx`.
