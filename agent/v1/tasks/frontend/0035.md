## TSK-0035 — Добавить BFF-роуты для stage CRUD и покрыть их проверками
- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Для надежной работы UI stage CRUD нужен предсказуемый BFF-слой между frontend и backend API.
- Сейчас BFF для roadmap в основном покрывает чтение roadmap и операции с topics, что усложняет единый UX ошибок для stage-операций.

### Acceptance Criteria
- [x] Добавлены BFF-роуты для операций `create/update/delete` этапов roadmap.
- [x] Роуты валидируют входные данные и возвращают единообразные сообщения об ошибках.
- [x] Роуты корректно проксируют авторизацию/сессию и backend-ошибки в формате, пригодном для UI.
- [x] Добавлены тесты или проверяемые сценарии для BFF stage CRUD (минимум happy path + validation error).

### Constraints
- Не менять внешний контракт backend `/api/v1/roadmap/stages*`.
- Не смешивать с задачами визуального редизайна.

### Verification
- Command/Test:
  - `cd frontend && npm run lint`
  - Expected: линтер проходит без ошибок.
  - Result: PASS (`✔ No ESLint warnings or errors`)
  - (если есть тесты) `cd frontend && npm test`
  - Expected: тесты BFF stage CRUD проходят.
  - Result: NOT RUN (в `frontend/package.json` отсутствует script `test`)
  - Manual:
  - Expected: UI stage CRUD получает корректные ответы и тексты ошибок через BFF.
  - Result: NOT RUN (добавлены проверяемые сценарии в `frontend/scripts/roadmap-stage-bff-verification.md`)

### Change Summary
**Problem**
- Для stage CRUD не хватало стабильного BFF-слоя с единообразной валидацией и ошибками, что делало обработку UI-сценариев менее предсказуемой.

**Root Cause Hypothesis**
- BFF-поддержка stage-операций была добавлена частично и требовала доработки качества: консистентной валидации, корректного edge-case поведения и формализации проверяемых сценариев.

**Evidence**
- В `frontend/app/api/roadmap/stages/[stageId]/route.ts` обнаружена некорректная строка импорта (`вimport`), что нарушало корректность файла.
- В stage-роутах отсутствовала явная проверка пустого `stageId`.
- После правок команда `cd frontend && npm run lint` проходит успешно.

**Solution**
- Доработаны BFF-роуты stage CRUD:
  - `frontend/app/api/roadmap/stages/route.ts` (create),
  - `frontend/app/api/roadmap/stages/[stageId]/route.ts` (update/delete).
- Исправлена поврежденная строка импорта в `stages/[stageId]/route.ts`.
- Добавлена явная валидация `stageId` для `PUT/DELETE` с единообразным ответом `422` и сообщением `Stage id is required.`.
- Сохранено проксирование backend-ошибок и сессионных cookie через `createBackendClient` + `createBackendErrorResponse`.
- Добавлены проверяемые сценарии happy-path/validation/cancel в `frontend/scripts/roadmap-stage-bff-verification.md`.

**Tests / Verification**
- Запущен `cd frontend && npm run lint` — PASS.
- `npm test` не запускался: test-script в проекте отсутствует.
- Для stage CRUD подготовлены проверяемые сценарии (happy path + validation error).

**Impact**
- BFF stage CRUD стал предсказуемее для UI по валидации и ошибкам.
- Backend-контракт `/api/v1/roadmap/stages*` не изменен.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: если backend начнет требовать дополнительную валидацию, BFF может возвращать backend-error чаще, чем локальные 422.
- Rollback: удалить добавленные проверки `stageId` и вернуть предыдущие версии stage-роутов; удалить сценарии верификации `frontend/scripts/roadmap-stage-bff-verification.md`.
