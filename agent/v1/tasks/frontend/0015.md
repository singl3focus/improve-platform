### Context
- Во frontend есть неиспользуемые модули, которые не импортируются в рабочий код: `frontend/src/lib/materials-library-store.ts`, `frontend/src/lib/tasks-board-store.ts`, `frontend/src/lib/topic-workspace-store.ts`, `frontend/src/lib/mock-api-mode.ts`.
- Эти файлы увеличивают объем поддержки и усложняют рефакторинг, потому что создают ложный контекст о действующих потоках данных.
- Требуется удалить мертвый код без изменения поведения приложения.

### Acceptance Criteria
- [x] Удалены неиспользуемые модули: `materials-library-store.ts`, `tasks-board-store.ts`, `topic-workspace-store.ts`, `mock-api-mode.ts`.

- [x] Поведение пользовательских экранов не изменилось (dashboard, roadmap, topics, tasks, materials).
- [x] `npm run lint`, `npm run build`, `npm run verify:mvp` проходят успешно.

### Constraints
- Не менять backend API-контракты.
- Не менять UX/тексты/внешний вид экранов в рамках этой задачи.
- Изменения ограничить удалением мертвого кода и минимальными правками импортов (если понадобятся).

### Change Summary
**Problem**
- Для задачи удаления мертвого кода не хватало подтверждения по AC: в прошлой проверке `verify:mvp` был FAIL, поэтому не был доказан критерий "поведение экранов не изменилось".

**Root Cause Hypothesis**
- Неполная/устаревшая верификация (без зеленого `verify:mvp`) не позволяла закрыть критерии, хотя сами файлы уже были удалены и не использовались.

**Evidence**
- Поиск `rg -n "materials-library-store|tasks-board-store|topic-workspace-store|mock-api-mode" frontend -g "*.ts" -g "*.tsx" -g "*.mjs"` после удаления дал `NO_MATCHES`.
- Проверка `Test-Path` подтвердила отсутствие всех 4 файлов (`False` для каждого пути).
- `npm run lint`, `npm run build` и `npm run verify:mvp` проходят.
- `verify:mvp` содержит PASS-сценарии для `dashboard`, `roadmap`, `topics`, `tasks`, `materials`.

**Solution**
- Удалены файлы:
  - `frontend/src/lib/materials-library-store.ts`
  - `frontend/src/lib/tasks-board-store.ts`
  - `frontend/src/lib/topic-workspace-store.ts`
  - `frontend/src/lib/mock-api-mode.ts`
- Дополнительные правки импортов не потребовались, так как ссылок на модули в коде не найдено.

**Tests / Verification**
- `rg ...` по именам удаленных модулей — PASS (`NO_MATCHES`).
- `Test-Path` для 4 удаленных файлов — PASS (`False` для каждого).
- `npm run lint` — PASS.
- `npm run build` — PASS (после повторного запуска вне sandbox из-за `spawn EPERM` в sandbox).
- `npm run verify:mvp` — PASS по всем smoke-сценариям, включая `dashboard/roadmap/topics/tasks/materials`.

**Impact**
- Удален мертвый frontend-код и снижена когнитивная нагрузка в `frontend/src/lib`.
- Backend API-контракты и UI не изменялись.
- Обратная совместимость: да, так как удалены только неиспользуемые модули.

**Risks & Rollback**
- Риск: если есть внешние/нерепозиторные зависимости на удаленные модули, они сломаются (внутри репозитория ссылок не найдено).
- Rollback plan: вернуть удаленные 4 файла в `frontend/src/lib/` с прежним содержимым.
