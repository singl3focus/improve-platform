### Change Summary
**Problem**
- В backend оставалась legacy endpoint-ветка `POST /api/v1/roadmap/topics/with-dependency` и связанный стек (`handler/usecase/repo/model`), которая не используется текущим frontend BFF и усложняет поддержку roadmap-кода.

**Root Cause Hypothesis**
- Ветка была добавлена ранее как промежуточный API для атомарного создания темы с зависимостью, но фактический UI/BFF поток использует `POST /api/v1/roadmap/topics` и отдельные dependency endpoints.

**Evidence**
- Регистрация legacy endpoint присутствовала в `backend/internal/server/server.go` (route `"/api/v1/roadmap/topics/with-dependency"`).
- Реализация legacy стека была в `backend/internal/roadmap/model.go`, `backend/internal/roadmap/handler.go`, `backend/internal/roadmap/usecase.go`, `backend/internal/roadmap/repo.go` (символы `CreateTopicWithDependency*`).
- Поиск использования в BFF/UI: `with-dependency` отсутствует в `frontend/app/api/**` и `frontend/scripts/**` (поиск: no matches).
- Smoke-проверка `npm run verify:mvp` после удаления прошла полностью, что подтверждает отсутствие зависимости текущего UI/BFF от удаленной ветки.

**Solution**
- Удален route `POST /api/v1/roadmap/topics/with-dependency` из `backend/internal/server/server.go`.
- Удалены устаревшие символы:
  - `CreateTopicWithDependencyRequest` и интерфейсные методы `CreateTopicWithDependency` из `backend/internal/roadmap/model.go`.
  - Handler `CreateTopicWithDependency` из `backend/internal/roadmap/handler.go`.
  - Usecase-метод `CreateTopicWithDependency` из `backend/internal/roadmap/usecase.go`.
  - Repo-метод `CreateTopicWithDependency` и helper `ensureTopicExistsInUserScope` из `backend/internal/roadmap/repo.go`.
- Обновлены тесты под актуальный API-поверхностный контракт:
  - удалены legacy mock/test ветки из `backend/internal/roadmap/handler_test.go`;
  - удалены legacy mock/test ветки из `backend/internal/roadmap/usecase_test.go`.

**Tests / Verification**
- `cd backend && go test ./...` -> PASS.
- `cd backend && go test ./internal/server ./internal/roadmap` -> PASS.
- `cd backend && golangci-lint run ./...` -> PASS (`0 issues`).
- `cd frontend && npm run verify:mvp` -> PASS (все сценарии smoke).

**Impact**
- API: удален устаревший endpoint `POST /api/v1/roadmap/topics/with-dependency`.
- Активные BFF-контракты (`/api/v1/roadmap`, `/api/v1/roadmap/topics`, `/api/v1/roadmap/topics/{topicID}/dependencies`, tasks/materials/history/auth) не изменены.
- Данные/миграции/безопасность: без изменений.
- Обратная совместимость: для текущего frontend/BFF сохранена; для внешних клиентов, использующих legacy endpoint, совместимость отсутствует (теперь 404).

**Risks & Rollback**
- Риск: внешние не-BFF интеграции, использующие `with-dependency`, перестанут работать.
- Rollback plan: вернуть удаленный route и связанный стек `CreateTopicWithDependency*` в `backend/internal/server/server.go` и `backend/internal/roadmap/*`.

**Suggested Commit Message**
- `backend: drop unused roadmap with-dependency endpoint and legacy handlers`
