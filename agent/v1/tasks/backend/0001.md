- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-14
- Updated: 2026-03-14

### Context
- Сейчас backend-код отсутствует, нужен минимальный каркас сервиса для дальнейшей доменной реализации.
- MVP требует отдельный backend-сервис с REST JSON API.

### Acceptance Criteria
- [x] Создан исполняемый backend-сервис на Go с роутером Chi и базовой структурой модульного монолита.
- [x] Есть endpoints `GET /healthz` и `GET /readyz` с проверкой подключения к БД для `readyz`.
- [x] Конфиг читается из env, добавлен пример конфигурации для локального запуска.
- [x] Формат ошибок HTTP унифицирован и подготовлен к интеграции с задачей `TSK-0010`.

### Constraints
- Без ORM, только подготовка слоя для pgx/SQL.
- Явные границы модулей; не смешивать бизнес-логику и HTTP.
- Не добавлять лишние абстракции вне MVP.

### Change Summary

**Problem**
- В репозитории отсутствовал backend-код. Для реализации доменной логики MVP необходим минимальный каркас Go-сервиса с HTTP-маршрутизатором, конфигурацией и health-проверками.

**Root Cause Hypothesis**
- Не применимо (greenfield-задача).

**Evidence**
- До начала работы в репозитории не было директории `backend/`, файлов `*.go`, `go.mod`, `go.sum`.

**Solution**
- Создан Go-модуль `improve-platform` в директории `backend/`.
- Структура проекта: `cmd/server/` (entry point), `internal/config/` (конфиг из env через viper), `internal/server/` (HTTP-сервер + Chi-роутер + graceful shutdown), `internal/server/handler/` (health-хендлеры), `pkg/httpresp/` (унифицированный JSON-ответ и формат ошибки).
- `GET /healthz` — всегда возвращает `200 {"status":"ok"}`.
- `GET /readyz` — проверяет `Ping` к БД через интерфейс `DBPinger`; при недоступности — `503 {"error":{"code":"database_unavailable","message":"..."}}`.
- Конфиг (`internal/config/config.go`) читает `APP_PORT`, `DATABASE_URL`, `LOG_LEVEL` из environment variables через `github.com/spf13/viper`; `DATABASE_URL` обязателен.
- Формат HTTP-ошибок (`pkg/httpresp/httpresp.go`): единый `{"error":{"code":"...","message":"..."}}`, готов к расширению в TSK-0010.
- Chi middleware: `RequestID`, `RealIP`, `Recoverer`.
- Structured JSON logging через `log/slog`.
- Graceful shutdown по SIGINT/SIGTERM.
- Добавлен `.env.example` с примером конфигурации для локального запуска.

**Tests / Verification**
- `go test -v ./...`: 8 тестов, все PASS.
  - `internal/config`: TestLoad_Defaults, TestLoad_CustomPort, TestLoad_MissingDatabaseURL.
  - `internal/server/handler`: TestHealthz_ReturnsOK, TestReadyz_DBUp, TestReadyz_DBDown.
  - `pkg/httpresp`: TestJSON_SetsContentTypeAndStatus, TestError_ReturnsUnifiedFormat.
- `go build -buildvcs=false ./...`: exit code 0.
- `go vet ./...`: exit code 0.
- curl-тест `/healthz` не запускался локально: нет работающего PostgreSQL; рекомендуется проверить после поднятия окружения.

**Impact**
- Добавлена новая директория `backend/` со всем содержимым.
- API: два новых endpoint (`/healthz`, `/readyz`).
- Обратная совместимость: не применимо (первый код в репозитории).

**Risks & Rollback**
- Риск минимальный: greenfield-каркас без бизнес-данных.
- Откат: удалить директорию `backend/`.
