- Для стабильного UX нужны предсказуемые доменные ошибки backend по операциям зависимостей (self/cycle/duplicate/not-found) с единым маппингом в handler.

### Change Summary
**Problem**
- Ошибки операций зависимостей roadmap были частично неоднородны для drag-сценариев frontend: `self/cycle/duplicate` маппились, но отсутствие связи при `DELETE` возвращалось как `topic_not_found`, что делало контракт двусмысленным.

**Root Cause Hypothesis**
- В доменной модели и repo-слое не было отдельной ошибки для кейса «зависимость не найдена», поэтому `RemoveDependency` использовал `ErrTopicNotFound` как fallback для `RowsAffected() == 0`.

**Evidence**
- `backend/internal/roadmap/repo.go`: `RemoveDependency` до изменения возвращал `ErrTopicNotFound` при отсутствии удаляемой связи.
- `backend/internal/roadmap/handler.go`: не было отдельного маппинга для `dependency_not_found`.
- Тесты на `duplicate/self/cycle` были неполные и не покрывали явно `dependency_not_found` для `DELETE`.

**Solution**
- Добавлена новая доменная ошибка `ErrDependencyNotFound` в `backend/internal/roadmap/model.go`.
- В `backend/internal/roadmap/repo.go` (`RemoveDependency`) `RowsAffected() == 0` теперь маппится в `ErrDependencyNotFound` вместо `ErrTopicNotFound`.
- В `backend/internal/roadmap/handler.go` добавлен единый error mapping:
  - `ErrSelfDependency` -> `400 self_dependency`
  - `ErrCycleDetected` -> `409 cycle_detected`
  - `ErrDependencyExists` -> `409 dependency_exists`
  - `ErrDependencyNotFound` -> `404 dependency_not_found`
  - `ErrTopicNotFound` -> `404 topic_not_found`
- Добавлены тесты `roadmap`:
  - `backend/internal/roadmap/usecase_test.go`:
    - `TestAddDependency_Duplicate`
    - `TestAddDependency_TopicNotFound`
    - `TestRemoveDependency_NotFound`
  - `backend/internal/roadmap/handler_test.go`:
    - `TestHandler_AddDependency_SelfRejected`
    - `TestHandler_AddDependency_DuplicateRejected`
    - `TestHandler_AddDependency_TopicNotFound`
    - `TestHandler_RemoveDependency_NotFound`

**Tests / Verification**
- Выполнено: `cd backend && go test ./...`
- Результат: PASS (все пакеты, включая `internal/roadmap`, `internal/task`, `internal/material`, `internal/server`).
- Manual API smoke: не запускался локально.

**Impact**
- API-контракт dependency-ошибок в `roadmap` стал однозначным и предсказуемым для UI drag-сценариев.
- Изменения локализованы в roadmap-домене и его тестах; happy-path и бизнес-правила DAG не изменялись.
- Обратная совместимость: сохранена для успешных сценариев; для `DELETE` отсутствующей связи уточнён error code (`dependency_not_found` вместо `topic_not_found`).

**Risks & Rollback**
- Риск низкий: изменения касаются только error-contract и тестового покрытия.
- Риск интеграции: frontend, если был захардкожен на `topic_not_found` в этом узком кейсе `DELETE`, должен использовать новый код `dependency_not_found`.
- Rollback: удалить `ErrDependencyNotFound`, вернуть старый маппинг в `repo.go`/`handler.go`, откатить добавленные тесты.
