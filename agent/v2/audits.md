# AUDITS

## Краткий промпт для audit-цикла агента

Ниже короткий промпт, который можно давать агенту для функционального, UX и perf-аудита:

```md
Тебе нужно провести полный аудит сценария/экрана в проекте и довести задачу до проверяемого результата.

Обязательный цикл работы:
1. Сначала воспроизведи сценарий через кодовую базу и доступные UI-инструменты.
2. Изучи релевантные файлы и зафиксируй рабочую гипотезу, где корневая причина: данные, React/state, layout, paint/compositing, CSS, overlay, сеть, BFF.
3. Внеси минимальные и контролируемые изменения, которые адресуют именно корневую причину, а не симптомы.
4. После правок обязательно прогоняй все уровни проверки:
   - build;
   - unit/integration tests;
   - e2e или targeted Playwright-сценарии;
   - самостоятельную функциональную UI-проверку;
   - самостоятельную performance-проверку через trace/DevTools/доступные браузерные инструменты.
5. Если на любом этапе видишь регрессию, не скрывай её: исправь и повтори цикл проверки.
6. Не считай работу завершённой, пока сам не подтвердил итог через инструменты.
7. В финальном отчёте разделяй:
   - симптомы;
   - корневую причину;
   - конкретные изменения;
   - результаты проверок;
   - остаточные риски.

Если задача касается производительности, обязательно измерь состояние до и после, а не опирайся только на субъективное ощущение.
```
