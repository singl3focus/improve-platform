## TSK-0052 — Перевести Roadmap в node-only граф без stage-блоков
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-17

### Context
- Owner требует, чтобы roadmap визуально был графом из мини-карточек тем, а не колонками/карточками этапов.
- Stage должен использоваться только как приписка внутри карточки темы (в правом верхнем углу), без отдельного визуального блока этапа.
- Связи в roadmap происходят только между темами.

### Acceptance Criteria
- [x] На экране `/roadmap` отсутствуют визуальные контейнеры этапов (`stage lanes`/отдельные stage-блоки).
- [x] Каждая тема отображается мини-карточкой в граф-сцене и содержит stage-приписку в правом верхнем углу.
- [x] Редактирование темы запускается иконкой карандаша в карточке темы (без обязательного открытия отдельного stage-контекста).
- [x] Создание/удаление связей между темами работает как и раньше: можно связать карточки, удалить существующую связь.
- [x] Переход в тему (`/topics?topicId=...`) по клику на карточку темы сохраняется.
- [x] На desktop и mobile визуал воспринимается как граф из узлов, без растягивания карточек темы на всю ширину экрана.

### Constraints
- Stage не удаляется из доменной модели и API, используется только как метка в карточке темы.
- Не менять backend API в рамках этой задачи.
- Не добавлять в эту задачу zoom/pan механику (выделено в отдельную техническую задачу).
- Сохранить существующие обработчики статусов/ошибок roadmap-модалок, если они не противоречат новой структуре.

### Change Summary
**Problem**
- На `/roadmap` темы были визуально сгруппированы в stage-lane контейнеры, что конфликтовало с требованием owner о node-only графе.
- Редактирование темы запускалось текстовой кнопкой внутри карточки, а не иконкой карандаша.

**Root Cause Hypothesis**
- Текущий JSX рендерил граф по `stages.map(...)` с отдельным `section` на stage и списком тем внутри.
- В карточке темы отсутствовал отдельный icon trigger для edit-сценария.

**Evidence**
- Изменён рендер графа в `frontend/src/components/roadmap-view.tsx`: stage-группировка заменена на единый список `allTopics`.
- Добавлены стили node-only списка и stage-метки в `frontend/app/globals.css`.
- Проверки:
  - `Set-Location frontend; npm run lint` → PASS
  - `Set-Location frontend; npm run test` → PASS (8/8)
  - `Set-Location frontend; npm run build` → PASS
  - Manual UI (desktop/mobile) → PASS:
    - edit по карандашу в карточке темы,
    - link/unlink между темами,
    - переход в `/topics?topicId=...`,
    - node-only структура без stage-lane,
    - карточки не full-width в mobile.

**Solution**
- Переведён рендер roadmap-сцены на единый набор узлов (`allTopics`) без визуальных stage-lane блоков.
- В карточку темы добавлена stage-приписка в правом верхнем блоке (`roadmap-topic-stage-tag`).
- Добавлена иконка-карандаш для редактирования темы прямо в карточке (`roadmap-topic-edit-trigger`).
- Сохранены существующие механики: клик по карточке (`/topics?topicId=...`), drag link/unlink, удаление связи и текущие обработчики ошибок/статусов.

**Tests / Verification**
- Запущено:
  - `Set-Location frontend; npm run lint` → PASS
  - `Set-Location frontend; npm run test` → PASS (8/8)
  - `Set-Location frontend; npm run build` → PASS
  - Manual UI checklist из секции Verification (desktop + mobile) → PASS.

**Impact**
- UI roadmap: stage-контейнеры убраны из визуального представления, stage остаётся меткой в topic-card.
- Backend API и доменная модель stage/topic/dependency не изменены.
- Обратная совместимость: да (контракты API не затронуты).

**Risks & Rollback**
- Риск: возможны визуальные нюансы плотности карточек на малых экранах, требующие ручной донастройки после UX-проверки.
- Rollback: вернуть предыдущий рендер stage-lane и стили в `frontend/src/components/roadmap-view.tsx` и `frontend/app/globals.css`.

**Suggested Commit Message**
- `frontend: switch roadmap to node-only graph with stage tag on topic cards`
