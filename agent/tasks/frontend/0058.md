## TSK-0058 — Завершить roadmap UI под полноценный граф и модальное управление темой
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-17
- Updated: 2026-03-17

### Context
- Owner подтвердил требования по roadmap UX:
  - редактирование темы открывается только в модальном окне;
  - удаление темы выполняется внутри этого же модального окна;
  - кнопку `Связать` на карточке нужно убрать;
  - создание связей должно выполняться drag-жестом между нодами.
- В текущем UI часть действий темы остается в карточке, а link-кнопка присутствует.

### Acceptance Criteria
- [x] Клик `Редактировать` на карточке темы открывает модальное окно редактирования темы.
- [x] В модальном окне темы доступны и работают `Сохранить` и `Удалить`; удаление из карточки убрано.
- [x] Кнопка `Связать`/`Link` удалена из карточек тем.
- [x] Создание связей выполняется drag-жестом между нодами (desktop), с корректной обработкой ошибок `self/cycle/duplicate`.
- [x] Визуал карточки темы доведен до референса `design/example_graph` (включая структуру центрального блока) без регресса читаемости на mobile.

### Constraints
- Не возвращать stages/колоночную stage-компоновку в UI.
- Не ломать текущую навигацию в тему и отображение blocked/progress/метрик.
- Не использовать git-команды.

### Change Summary
**Problem**
- В roadmap-карточке темы редактирование/удаление были частично встроены в карточку, что не соответствовало ожидаемому UX с единым модальным управлением.
- На карточке присутствовала текстовая кнопка `Связать/Link`, тогда как требование — создание связей drag-жестом между нодами.
- Ошибка дублирующей зависимости не была вынесена в отдельное доменное сообщение.

**Root Cause Hypothesis**
- Логика редактирования темы была реализована inline-формой внутри карточки, а не в модальном диалоге.
- UI drag-ручки для зависимости использовал текстовую кнопку и допускал сценарии вне drag-жеста.
- Маппинг кодов ошибок зависимостей покрывал `self` и `cycle`, но не `duplicate`.

**Evidence**
- `frontend/src/components/roadmap-view.tsx:2125` — добавлено модальное окно редактирования темы с формой и действиями `Сохранить/Удалить`.
- `frontend/src/components/roadmap-view.tsx:2412` — `Редактировать` на карточке открывает модалку через `startTopicEditing(topic, event.currentTarget)`.
- `frontend/src/components/roadmap-view.tsx:2364` — на карточке оставлена drag-ручка зависимости в виде иконки (без текста `Связать/Link`).
- `frontend/src/components/roadmap-view.tsx:1736` и `frontend/src/components/roadmap-view.tsx:1744` — при завершении pointer-сценария без валидного drag/source состояние источника связи очищается.
- `frontend/src/components/roadmap-view.tsx:379` и `frontend/src/components/roadmap-view.tsx:395` — добавлена обработка duplicate-кода зависимости с отдельным сообщением.
- `frontend/scripts/roadmap-drag-review-backend.mjs` — review-mock теперь возвращает `409` с `code: dependency_exists`, если связь уже существует; duplicate-сценарий стал воспроизводимым.
- `frontend/src/components/roadmap-view.tsx:2461` и `frontend/app/globals.css:1422` — переработана структура и стили центрального блока карточки темы под визуальный референс.

**Solution**
- Перенесено редактирование темы из inline-формы в модальное окно, добавлены кнопки `Сохранить`, `Отмена`, `Удалить` внутри модалки.
- Удалены действия удаления из тела карточки темы; удаление доступно только в модальном окне темы.
- Текстовая кнопка `Связать/Link` заменена на компактную drag-ручку-иконку; визуально текстовая link-кнопка на карточке убрана.
- Для сценария создания зависимости добавлена явная обработка duplicate-ошибки (`duplicate_dependency`/`dependency_exists`/`already_exists`).
- Подправлена композиция центрального блока карточки темы (`roadmap-topic-center`) и стили drag-ручки для ближе соответствия `design/example_graph` без изменения данных/метрик.
