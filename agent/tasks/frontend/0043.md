## TSK-0043 — Roadmap: после кнопок сразу колонки этапов, без верхних промежуточных блоков
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- По требованию owner после верхних action-кнопок в roadmap должны сразу идти колонки этапов.
- Визуальные промежуточные блоки над колонками должны быть убраны.
- При этом заголовок и счётчик тем внутри каждой колонки этапа должны сохраниться.

### Acceptance Criteria
- [x] После блока кнопок «Создать этап» и «Добавить тему» сразу начинается секция колонок этапов.
- [x] Верхний strip этапов над колонками удалён.
- [x] Между action-кнопками и колонками отсутствуют дополнительные промежуточные информационные блоки.
- [x] Заголовок этапа и количество тем внутри каждой колонки остаются отображаемыми.
- [x] Функциональность колонок и карточек тем не деградирует.

### Constraints
- Не удалять заголовок и счётчик у самой колонки этапа.
- Не менять бизнес-логику topic/stage CRUD.
- Не изменять API и структуру ответа roadmap.

  - `Set-Location frontend; npm run build`
  - Expected: production build проходит успешно.

  - Result: NOT RUN

### Change Summary
**Problem**
- После верхних action-кнопок на `/roadmap` рендерились дополнительные промежуточные блоки (включая верхний strip этапов), из-за чего колонки этапов не начинались сразу под кнопками.

**Root Cause Hypothesis**
- В success-ветке roadmap рендерились информационные блоки (`dashboard-error/success`) и `roadmap-stage-strip` перед секцией `roadmap-graph` с колонками.

**Evidence**
- До изменения в `frontend/src/components/roadmap-view.tsx` перед `roadmap-graph` присутствовали:
  - блоки ошибок/успеха для topic/stage/dependency,
  - `roadmap-stage-strip` с `roadmap-stage-pill`.
- После изменения:
  - `roadmap-stage-strip` удалён из рендера;
  - колонки этапов стартуют в `roadmap-graph` сразу после header-buttons (с учётом modal-overlay);
  - feedback-блоки перенесены ниже секции колонок.
- Проверка командами:
  - `Set-Location frontend; npm run lint` -> PASS;
  - `Set-Location frontend; npm run build` -> PASS.

**Solution**
- В `frontend/src/components/roadmap-view.tsx` удалён рендер верхнего strip этапов (`roadmap-stage-strip`).
- Блоки `topicMutationError`, `stageMutationError`, `stageMutationSuccess`, `dependencyMutationError` перенесены ниже `roadmap-graph`, чтобы между action-кнопками и колонками не было промежуточных информационных секций.
- Заголовок этапа и счётчик тем внутри каждой колонки сохранены в `roadmap-stage-header`.
- Бизнес-логика CRUD и API-контур не изменялись.

**Tests / Verification**
- Запущено:
  - `Set-Location frontend; npm run lint` -> PASS.
  - `Set-Location frontend; npm run build` -> PASS.
- Не запускалось локально:
  - ручной UI smoke `/roadmap` для визуальной проверки порядка блоков и поведения колонок/карточек.

**Impact**
- UI roadmap: упрощён верх страницы, колонки этапов отображаются сразу после блока action-кнопок.
- API/данные/маршрутизация: без изменений, обратная совместимость сохранена.

**Risks & Rollback**
- Риск: без ручного UI smoke возможны визуальные нюансы в крайних состояниях (например, отображение feedback при ошибках).
- Rollback plan:
  - вернуть `roadmap-stage-strip` и исходный порядок feedback-блоков в `frontend/src/components/roadmap-view.tsx`.
