## TSK-0023 — Убрать раздел «Темы» из меню и исключить тупиковый вход
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- В текущем меню есть раздел `/topics`, но без `topicId` экран не дает полезного действия и просит выбрать тему.
- Фактический рабочий сценарий уже идет через переход в тему из Roadmap/Tasks/Materials по конкретному `topicId`.
- Нужно убрать бесполезную точку входа из навигации и оставить только целевой вход в тему.

### Acceptance Criteria
- [x] Пункт «Темы» удален из бокового меню (`PrivateShell`), в навигации остаются только рабочие разделы.
- [x] Прямой переход в тему по ссылкам вида `/topics?topicId=...` продолжает работать.
- [x] Переход на `/topics` без `topicId` больше не оставляет пользователя в тупиковом состоянии (редирект на `/roadmap`).
- [x] Локализационный `navigation`-copy синхронизирован с новым набором пунктов меню.

### Constraints
- Не удалять сам маршрут `/topics`: он нужен как целевой экран темы по `topicId`.
- Не ломать переходы в тему из Roadmap, Kanban и библиотеки материалов.
- Изменения ограничить навигацией и поведением входа в `/topics` без параметров.

### Change Summary
**Problem**
- В боковом меню оставался пункт `/topics`, который без `topicId` открывал тупиковое состояние «тема не выбрана».
- Нужен целевой вход в тему только по ссылкам с `topicId`, без бесполезной точки входа из навигации.

**Root Cause Hypothesis**
- `PrivateShell` включал `/topics` в основной список навигации, хотя рабочий сценарий темы зависит от query-параметра `topicId`.
- Страница `/topics` не делала server-side guard на отсутствие `topicId`.

**Evidence**
- `frontend/src/components/layout/private-shell.tsx:9` — в `NAV_ITEMS` удален элемент `{ href: "/topics", key: "topicsLabel" }`.
- `frontend/src/lib/ui-copy.ts:20` — в `NavigationCopy` и RU/EN словарях удален `topicsLabel`, копия навигации синхронизирована с актуальным меню.
- `frontend/app/(private)/topics/page.tsx:23` — добавлена проверка `topicId`; при отсутствии вызывается `redirect("/roadmap")`.
- Ссылки целевого входа в тему сохранены:
  - `frontend/src/components/roadmap-view.tsx:369`
  - `frontend/src/components/roadmap-view.tsx:506`
  - `frontend/src/components/tasks-kanban-view.tsx:251`
  - `frontend/src/components/materials-library-view.tsx:299`
- Проверка командами:
  - `npm run lint` -> PASS.
  - `npx tsc --noEmit` -> PASS.

**Solution**
- Удален пункт `Topics` из основного меню `PrivateShell`.
- Синхронизированы тип и словари локализации `navigation`: удален `topicsLabel` в RU/EN.
- На серверной странице `/topics` добавлен guard:
  - если `topicId` отсутствует или пустой, выполняется редирект на `/roadmap`;
  - если `topicId` передан, рендерится `TopicWorkspaceView` как раньше.

**Tests / Verification**
- Запущено:
  - `cd frontend && npm run lint` — PASS (`✔ No ESLint warnings or errors`).
  - `cd frontend && npx tsc --noEmit` — PASS (exit code `0`).
- Не запускалось локально:
  - Ручная проверка в браузере сценариев меню/редиректа и переходов по `/topics?topicId=<id>`.

**Impact**
- UI/UX: убрана тупиковая точка входа из боковой навигации.
- Маршрутизация: `/topics` без `topicId` теперь направляет на `/roadmap`.
- Совместимость: входы в тему из Roadmap/Kanban/Materials через `/topics?topicId=...` сохранены.
- API/данные/безопасность: без изменений.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: без ручного браузерного smoke не подтверждено визуально поведение active-state/заголовка при открытии `/topics?topicId=...`.
- Rollback plan:
  - вернуть `topicsLabel` в `frontend/src/lib/ui-copy.ts`;
  - вернуть пункт `/topics` в `frontend/src/components/layout/private-shell.tsx`;
  - убрать редирект в `frontend/app/(private)/topics/page.tsx`.
