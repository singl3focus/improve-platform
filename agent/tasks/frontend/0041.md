- Status: Finished
- Type: Business
- Approval: Approved

### Context
- Сейчас создание зависимости инициируется кнопкой Link в действиях карточки темы и сопровождается отдельным верхним блоком «Управление зависимостями».
- По требованию owner связь должна инициироваться с границы карточки темы, а отдельный блок «Управление зависимостями» должен быть убран.

### Acceptance Criteria
- [x] Инициация создания связи выполняется через интерактивную зону на границе карточки темы (без кнопки Link в action-блоке карточки).
- [x] Для active-link сценария сохраняется текущий принцип выбора target-темы кликом/тапом, с созданием зависимости через существующий API-контур.
- [x] Визуальный preview-стрелки во время связи сохраняется.
- [x] Отдельный UI-блок «Управление зависимостями» удалён из roadmap-экрана.
- [x] Негативные кейсы `self_dependency` и `cycle_detected` продолжают отображаться корректными сообщениями.
- [x] Удаление существующих зависимостей из карточки темы продолжает работать.

### Constraints

- Не менять маршрутизацию страницы roadmap.
- Не ухудшать доступность клавиатурного сценария открытия темы.

### Change Summary
**Problem**
- Создание зависимости на roadmap было привязано к кнопке `Link` в action-блоке карточки и сопровождалось отдельной верхней панелью «Управление зависимостями», что не соответствовало обновлённому UX-требованию.

**Root Cause Hypothesis**
- Точка входа dependency-link flow осталась в `roadmap-topic-actions`, а связанный информационный/управляющий блок был выделен в отдельную секцию вместо карточки темы.

**Evidence**
- До правки точка входа linking была в action-кнопке `roadmap-topic-link-button` и отдельной секции `roadmap-dependency-link-panel` в `frontend/src/components/roadmap-view.tsx`.
- В стилях присутствовали отдельные классы панели/кнопки (`.roadmap-dependency-link-panel`, `.roadmap-topic-link-button`) в `frontend/app/globals.css`.
- Проверка сборки/качества после изменений:
  - `Set-Location frontend; npm run lint` -> PASS.
  - `Set-Location frontend; npm run build` -> PASS.

**Solution**
- В `frontend/src/components/roadmap-view.tsx` удалён отдельный блок «Управление зависимостями» из roadmap-экрана.
- Запуск linking перенесён в интерактивный edge-handle (`roadmap-topic-link-handle`) на границе каждой topic-card.
- Удалена кнопка `Link` из `roadmap-topic-actions`.
- Существующий dependency flow сохранён:
  - drag/tap source->target через текущие обработчики `beginDependencyLinkDrag`/`createDependencyFromSourceToTarget`;
  - preview-стрелка (`roadmap-connection-preview`) без изменения;
  - удаление связи (`handleDependencyDelete`) без изменения;
  - показ ошибок `self_dependency`/`cycle_detected` через существующий `getDependencyErrorMessage` без изменения API-контракта.
- В `frontend/app/globals.css` удалены стили старой dependency-панели и action-кнопки link, добавлены стили нового edge-handle на границе карточки.

**Tests / Verification**
- Запущено:
  - `Set-Location frontend; npm run lint` -> PASS.
  - `Set-Location frontend; npm run build` -> PASS.
- Не запускалось локально:
  - ручной UI smoke `/roadmap` (drag/tap linking, preview, remove dependency, отсутствие старой панели).

**Impact**
- UI/UX roadmap: изменён способ старта создания зависимости (с action-кнопки на edge-handle карточки).
- API/данные/роутинг: без изменений, обратная совместимость сохранена.
- Безопасность: без изменений.

**Risks & Rollback**
- Риск: возможные UX-расхождения в pointer/tap сценариях на отдельных устройствах без ручного smoke.
- Rollback plan:
  - вернуть рендер старой секции `roadmap-dependency-link-panel` и кнопки `roadmap-topic-link-button` в `frontend/src/components/roadmap-view.tsx`;
  - откатить стили edge-handle и восстановить стили старой панели/кнопки в `frontend/app/globals.css`.
