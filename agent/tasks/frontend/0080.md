### Change Summary
**Problem**
- На roadmap связи рендерились как линии без различимого arrowhead в фактическом UI, хотя `markerEnd` уже был подключен.
- Для связей, приходящих в правую грань карточки, наконечник мог уходить под выступающий `roadmap-topic-link-handle`, из-за чего направление связи читалось хуже.

**Root Cause Hypothesis**
- Базовый SVG marker был слишком компактным для текущей геометрии roadmap.
- Для всех сторон карточки использовался одинаковый gap `18px`, хотя правая грань имеет дополнительный выступающий control (`right: -9px`, `width: 28px`) и требует большего clearance.

**Evidence**
- Код рендера marker уже существовал в `frontend/src/components/roadmap-view.tsx`, но использовал компактную геометрию `12x10` с `refX="11"`.
- CSS `frontend/app/globals.css` задает `roadmap-topic-link-handle` с `right: -9px` и `width: 28px`, то есть control выступает вправо за карточку и может перекрывать наконечник при обычном gap.
- Геометрия связей строится через `offsetConnectionAnchorPoint` в `frontend/src/lib/roadmap-layout.ts` и `getTopicAnchorPoint` в `frontend/src/components/roadmap-view.tsx`; до правки для правой стороны не было отдельного offset contract.
- `Set-Location frontend; npm run test` → PASS (`59/59`).
- `Set-Location frontend; npm run build` → PASS (`Compiled successfully`, `Generating static pages (32/32)`).

**Solution**
- Добавлен helper `getConnectionAnchorOffsetDistance` в `frontend/src/lib/roadmap-graph.ts`, который оставляет стандартный gap для `left/top/bottom`, но увеличивает clearance для `right`.
- `frontend/src/lib/roadmap-layout.ts` и `frontend/src/components/roadmap-view.tsx` переведены на единый расчёт anchor offset, чтобы обычные связи и drag-preview использовали одинаковый contract.
- В `frontend/src/components/roadmap-view.tsx` увеличены размеры обычного и preview marker (`16x12`, `refX="15"`), чтобы наконечник был заметнее без изменения поведения ребер.
- Обновлены unit-тесты в `frontend/src/lib/roadmap-graph.test.ts` и `frontend/src/lib/roadmap-layout.test.ts`, включая кейс с правой гранью карточки и дополнительным clearance под `link-handle`.

**Tests / Verification**

**Risks & Rollback**
- Риск: arrow gap справа стал больше, поэтому линия чуть дальше отходит от правой грани карточки; визуально это компенсируется лучшей читаемостью наконечника и отсутствием перекрытия handle.
- Rollback: вернуть прежний marker size в `frontend/src/components/roadmap-view.tsx` и убрать helper `getConnectionAnchorOffsetDistance`, вернув единый gap в `frontend/src/lib/roadmap-graph.ts`.
