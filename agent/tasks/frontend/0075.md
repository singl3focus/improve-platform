## TSK-0075 — Roadmap tree-UX: создать справа/слева/ниже через меню темы без наложения карточек
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-20
- Updated: 2026-03-20

### Context
- Владелец зафиксировал целевое поведение roadmap-графа как дерево: у темы в меню троеточия должны быть действия `создать справа`, `создать слева`, `создать ниже`.
- Важное требование: карточки тем не должны накладываться друг на друга, даже при последовательном создании нескольких тем.
- Владелец подтвердил, что прежний child-create полностью заменяется действием `создать ниже`.

### Acceptance Criteria
- [x] В меню троеточия карточки темы доступны три действия: `создать справа`, `создать слева`, `создать ниже`.
- [x] Семантика направлений реализована в UI: новые темы отображаются в ожидаемом направлении относительно исходной темы.
- [x] Для всех трех направлений после создания визуально отображается dependency-стрелка от текущей темы к созданной.
- [x] Алгоритм раскладки и рендера гарантирует отсутствие наложения карточек в сценариях множественного последовательного создания.
- [x] Удалены/заменены все упоминания и UX-поведение старого `создать дочернюю`; вместо него используется `создать ниже`.
- [x] Добавлены/обновлены автотесты layout/position/graph для защиты от регрессий по коллизиям и направленному созданию.

### Constraints

### Change Summary
**Problem**
- В roadmap topic-menu был только legacy сценарий `Create child`, тогда как требовались три действия направления (`left/right/below`).
- Старый layout-подход не давал явной directional-семантики и был уязвим к коллизиям при последовательном создании тем.

**Root Cause Hypothesis**
- UI и копирайт оставались в child-create модели из предыдущей итерации.
- Раскладка карточек опиралась на legacy вычисления (`stage/level`) без единого placement-алгоритма, что усложняло гарантию anti-overlap для directional flow.

**Evidence**
- `frontend/src/components/roadmap-view.tsx`:
  - заменены copy-ключи и действия меню на `topicMenuCreateLeft/topicMenuCreateRight/topicMenuCreateBelow`;
  - добавлены directional заголовки/подписи/кнопки (`topicCreateDirectionTitle`, `topicCreateDirectionSubtitle`, `topicCreateDirectionButton`, `topicCreatingDirectionButton`);
  - legacy `startChildTopicCreate` заменён на `startDirectionalTopicCreate` и `topicCreateAnchor`.
- `frontend/src/lib/roadmap-layout.ts`:
  - добавлен `buildTopicGridPlacementById` с directional размещением (`left/right/below`) и проверкой занятых ячеек;
  - `buildTopicGridColumnById` переведён на reuse placement-алгоритма.
- `frontend/app/api/roadmap/topics/route.ts`:
  - расширен create-payload (`position`), добавлена валидация и поддержка явной позиции для directional create.
- `frontend/src/lib/roadmap-layout.test.ts`:
  - добавлены кейсы directional placement и anti-overlap для последовательных созданий.

**Solution**
- Реализован tree-UX с тремя действиями создания темы в меню карточки: слева, справа, ниже.
- В create-flow передаётся направленная позиция относительно текущей темы; затем создаётся dependency `current -> created`.
- Рендер графа переведён на placement-based раскладку без пересечений карточек.
- Обновлены тесты для защиты от регрессий по directional semantics и коллизиям.

**Tests / Verification**
- Запущено:
  - `npm run test` — PASS (`29 passed, 0 failed`).
  - `npm run build` — PASS.
  - `npm run verify:mvp` — PASS.
- Ручной браузерный сценарий:
  - login -> `/roadmap`;
  - quick-create первой темы;
  - через меню темы созданы `left/right/below`;
  - визуально и по DOM-проверке подтверждено отсутствие наложений (`overlap=false`) и наличие dependency-связей.

**Impact**
- Затронуты frontend roadmap UI, roadmap layout и BFF create-route для тем.
- Backend API контракт не ломается: `position` остаётся опциональным.
- Обратная совместимость: да.

**Risks & Rollback**

- Rollback:

  - вернуть изменения в `frontend/src/lib/roadmap-layout.ts` и `frontend/src/lib/roadmap-layout.test.ts`;
  - вернуть изменения в `frontend/app/api/roadmap/topics/route.ts`.
