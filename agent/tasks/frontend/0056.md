### Context
- Owner требует визуальное соответствие карточек тем референсу из `design/example_graph`.
- Критично соблюсти структуру карточки с блоком `div.flex.flex-col.items-center.gap-4` и близкий визуальный ритм.

### Acceptance Criteria
- [x] Структура карточки темы приведена к референсному паттерну (включая центральный вертикальный контейнер и ключевые внутренние блоки).

- [x] Карточка сохраняет функциональность: клик для перехода, действия редактирования, управление зависимостями.
- [x] На desktop и mobile карточка не растягивается неконтролируемо и остается читаемой.
- [x] Текущие классы/стили stage-lane не возвращаются в UI.

### Change Summary
**Problem**
- Карточка темы roadmap визуально и структурно расходилась с референсом из `design/example_graph`: не было централизованного вертикального контейнера для контента, ритм отступов и плотность карточки были менее выражены.

**Root Cause Hypothesis**
- Текущая разметка карточки исторически строилась вокруг горизонтального top-блока и распределённого контента, без выделенного центрального блока как в референс-паттерне `flex flex-col items-center gap-4`.

**Evidence**
- Разметка карточки обновлена в `frontend/src/components/roadmap-view.tsx` (блок карточки внутри `data-roadmap-topic-id`): добавлен контейнер `roadmap-topic-core` и в него перенесены ключевые элементы (stage tag, title, description, progress, counters).
- Стили карточки обновлены в `frontend/app/globals.css`: скорректированы `roadmap-topic-card`, `roadmap-topic-top`, добавлен `roadmap-topic-core`, усилена иерархия карточки (border/padding/min-height/shadow), центрирование и стабильная ширина прогресс-блока.
- Проверки сборки/линтинга:
  - `npm run lint` -> PASS.
  - `npm run build` -> PASS.

**Solution**
- Перестроена внутренняя структура карточки темы в roadmap:
  - сохранены действия редактирования/меню/связей;
  - добавлен центральный вертикальный контейнер контента `roadmap-topic-core`;
  - stage tag перенесён в центральный блок для композиционного соответствия референсу.
- Подправлены стили карточки под референсный визуальный ритм:
  - увеличены толщина рамки, радиус, внутренние отступы и минимальная высота;
  - усилен hover-shadow;
  - выровнен и центрирован основной контент карточки;
  - прогресс и счётчики закреплены по ширине/центру для стабильной читаемости.

**Tests / Verification**
- `Set-Location frontend; npm run lint` -> PASS.
- `Set-Location frontend; npm run build` -> PASS.
- Manual UI -> PASS (структура/визуал карточки, кликабельность, действия edit/menu/dependency, мобильная читаемость).

**Impact**
- Область влияния: только UI карточек roadmap.
- API/контракты/бизнес-логика: без изменений.
- Безопасность/данные/перфоманс: без существенного влияния.
- Обратная совместимость: да.
