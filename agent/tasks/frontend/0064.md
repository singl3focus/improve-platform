### Change Summary
**Problem**
- После stage-removal оставались технические хвосты в roadmap smoke: сценарии stage-free topic/dependency падали (`404`), а baseline проверки всё ещё включали stage-route контур.

**Root Cause Hypothesis**
- В mock backend сценариях topic create не имел корректного fallback для пустого roadmap без stage.
- В quick-create smoke отсутствовал безопасный reset состояния `roadmapMode` между checks.
- Smoke включал stage-route проверку как обязательную, что противоречило целевому stage-free baseline.

**Evidence**
- `frontend/scripts/mvp-verification-smoke.mjs`: в topic create mock-ветке при отсутствии stage возвращался `404 stage_not_found`.
- `frontend/scripts/mvp-verification-smoke.mjs`: quick-create сценарий переводил `roadmapMode` в `not_found`, но не гарантировал reset в `finally` при падении.
- До правки smoke падал на 3 сценариях roadmap (`quick-create`, `topic create/update/delete`, `dependency add/remove`) с `404`.
- Проверки:
  - `npm run lint` -> PASS.
  - `npm run test` -> PASS.
  - `npm run build` -> PASS.
  - `npm run verify:mvp` -> PASS.

**Solution**
- В `frontend/scripts/mvp-verification-smoke.mjs` добавлен `ensureFallbackStage(state)` для stage-free topic create в пустом roadmap.
- В quick-create smoke добавлен `try/finally` для гарантированного возврата `roadmapMode` в `ok`.
- Удалён обязательный smoke-check stage routes; целевой roadmap smoke сфокусирован на stage-free topic/dependency flow.
- Дополнительно подтверждено, что в target UI flow (`roadmap-view`) нет прямых вызовов stage endpoint.

**Tests / Verification**
- `Set-Location frontend; npm run lint` -> PASS.
- `Set-Location frontend; npm run test` -> PASS (18/18).
- `Set-Location frontend; npm run build` -> PASS.
- `Set-Location frontend; $env:BACKEND_API_URL='http://127.0.0.1:8080'; npm run start -- --port 3025` -> PASS.
- `Set-Location frontend; npm run verify:mvp` -> PASS (все smoke-сценарии, включая stage-free roadmap).
- Static check по target flow (`roadmap-view` + smoke script) -> PASS.

**Impact**
- Roadmap smoke baseline стабилизирован для stage-free контура.
- Проверка stage-free topic/dependency flow стала воспроизводимой и проходит локально.
- Stage endpoint больше не является обязательной частью целевого smoke-flow.
- Обратная совместимость: stage API в приложении остаётся доступным, но не влияет на target stage-free smoke.

**Risks & Rollback**
- Риск: fallback stage в mock может маскировать ошибки legacy stage-only реализации в изолированных тестах.
- Rollback: убрать `ensureFallbackStage` и вернуть stage-required поведение topic create в smoke mock.
