## TSK-0032 — Добавить блок истории на dashboard с переходом в подробности
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Пользователь не понимает, где в UI отображается история действий.
- По решению owner достаточно блока истории на dashboard (без отдельного пункта в основном меню).

### Acceptance Criteria
- [x] На `/dashboard` добавлен отдельный блок «История» с последними событиями.
- [x] Из блока есть явный переход в подробный экран/раздел истории.
- [x] Показаны состояния `loading`, `error`, `empty` в стиле dashboard.
- [x] Источник данных — существующий BFF/API-слой, без прямого UI вызова backend.

### Constraints
- Не добавлять новый основной пункт в левом меню в рамках этой задачи.
- Не смешивать с полным редизайном dashboard.

### Change Summary
**Problem**
- На dashboard отсутствовал отдельный блок истории действий и не было явного перехода в подробный экран истории.

**Root Cause Hypothesis**
- История (`/api/v1/history`) реализована в backend, но во frontend отсутствовали BFF-роут, типы, тексты и UI-компоненты для вывода истории на dashboard.

**Evidence**
- В backend доступен endpoint `GET /api/v1/history` (маршрутизация в `backend/internal/server/server.go`, handler в `backend/internal/history/handler.go`).
- До изменений во frontend не было маршрута `frontend/app/api/dashboard/history/route.ts` и приватного экрана истории `frontend/app/(private)/dashboard/history/page.tsx`.
- После реализации `npm run lint` и `npm run build` проходят успешно.

**Solution**
- Добавлен BFF-роут `frontend/app/api/dashboard/history/route.ts`, который проксирует вызовы в backend `/api/v1/history`, ограничивает `limit`, маппит snake_case -> camelCase и использует существующую обработку backend-ошибок.
- Добавлены типы истории: `BackendHistoryEventResponse` в `frontend/src/lib/backend-learning-contracts.ts` и `DashboardHistoryEvent` в `frontend/src/lib/dashboard-types.ts`.
- Расширен `useDashboardViewModel` (`frontend/src/components/hooks/use-dashboard-view-model.ts`) новым ресурсом `history` через существующий слой `useDashboardResource` (`/api/dashboard/history?limit=6`).
- В `frontend/src/components/dashboard-view.tsx` добавлен новый блок «История» с состояниями `loading/error/empty`, выводом последних событий и переходом на подробный экран `/dashboard/history`.
- Добавлен подробный экран истории: `frontend/src/components/dashboard-history-view.tsx` + маршрут `frontend/app/(private)/dashboard/history/page.tsx`.
- Обновлены локализованные тексты RU/EN в `frontend/src/lib/ui-copy.ts`.
- В `frontend/src/components/layout/private-shell.tsx` улучшена логика активного пункта меню для вложенных маршрутов (`/dashboard/history` подсвечивает раздел Dashboard).

**Tests / Verification**
- Запущен `cd frontend && npm run lint` — PASS (`✔ No ESLint warnings or errors`).
- Запущен `cd frontend && npm run build` — PASS (успешная production-сборка).
- Manual сценарий не запускался в этом цикле.

**Impact**
- Добавлен новый UI-блок и экран истории в приватной зоне frontend.
- Добавлен новый BFF endpoint `/api/dashboard/history`.
- API-контракты backend и структура данных не изменялись; обратная совместимость сохранена.

**Risks & Rollback**
- Риск: отображение зависит от корректности payload событий `history` и доступности backend endpoint.
- Rollback: удалить интеграцию блока истории из `frontend/src/components/dashboard-view.tsx`, удалить `frontend/app/api/dashboard/history/route.ts`, `frontend/src/components/dashboard-history-view.tsx` и `frontend/app/(private)/dashboard/history/page.tsx`, убрать связанные типы/копирайтинг.
