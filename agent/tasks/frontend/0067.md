## TSK-0067 — Очистить репозиторий от временных и лишних артефактов
- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-18
- Updated: 2026-03-18

### Context
- Owner запросил очистку репозитория от временных и ненужных файлов.
- В workspace присутствуют build/test/runtime артефакты, которые не должны храниться в рабочем дереве задач.

### Acceptance Criteria
- [x] Удалены временные артефакты frontend (например `.test-dist`, временные `.err/.out/.log`, локальные review-логи), не являющиеся исходниками проекта.
- [x] Проверены и обновлены ignore-правила для предотвращения повторного появления артефактов в рабочем дереве.
- [x] Очистка не затрагивает исходный код, документацию требований и актуальные task-файлы.
- [x] После очистки `npm run lint` и `npm run build` в `frontend` остаются PASS.

### Constraints
- Удалять только артефакты, чья ненужность подтверждена по контексту проекта.
- Не удалять диагностические файлы, если они явно используются в автоматизированной верификации.
- Не использовать git-команды.

### Change Summary
**Problem**
- В `frontend` оставались runtime/test/review артефакты (`.test-dist`, временные `.log/.err/.out`, `tsconfig.tsbuildinfo`), что засоряло рабочее дерево.

**Root Cause Hypothesis**
- В проекте отсутствовали ignore-правила для части локально генерируемых артефактов, а ранее созданные временные файлы не были удалены.

**Evidence**
- Перед очисткой на верхнем уровне `frontend` присутствовали: `.test-dist`, `.dev-3030.err.log`, `.dev-3030.out.log`, `review-dev.err`, `review-dev.out`, `review-next-start.err`, `review-next-start.out`, `tsconfig.tsbuildinfo`.
- `frontend/.gitignore` до изменения не содержал правил для `.test-dist`, `*.tsbuildinfo`, `review-*.err`, `review-*.out`, `.dev-*.log`.
- Проверка после изменения: `npm run lint` -> PASS, `npm run build` -> PASS.

**Solution**
- Удалены подтвержденные временные артефакты в `frontend`:
  - папка `.test-dist`;
  - файлы локальных dev/review логов `.dev-3030.*.log`, `review-*.err`, `review-*.out`;
  - файл `tsconfig.tsbuildinfo`.
- Обновлен `frontend/.gitignore`: добавлены `.test-dist`, `*.tsbuildinfo`, `review-*.err`, `review-*.out`, `.dev-*.log`.

**Tests / Verification**
- Запущено: `Set-Location frontend; npm run lint`.
- Результат: PASS.
- Запущено: `Set-Location frontend; npm run build`.
- Результат: PASS.
- Выполнена статическая проверка содержимого `frontend` после удаления и обновления ignore-правил.

**Impact**
- Влияние: только housekeeping frontend-артефактов и ignore-правил.
- API/данные/безопасность: без изменений.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: низкий, т.к. удалялись только временные файлы.
- Rollback plan: при необходимости удалить добавленные ignore-правила из `frontend/.gitignore`; артефакты будут сгенерированы повторно при следующих запусках команд.

**Suggested Commit Message**
- frontend: clean temporary artifacts and ignore generated review logs
