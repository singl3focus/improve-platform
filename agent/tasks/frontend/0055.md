## TSK-0055 — Добавить создание дочерней ноды через меню из трех точек с автосвязью
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-17
- Updated: 2026-03-17

### Context
- Owner требует сценарий: от каждой ноды можно создать следующую ноду через меню с тремя точками, и зависимость ставится сразу от исходной ноды.
- Текущий UX требует отдельных действий для создания темы и связывания зависимостью.

### Acceptance Criteria
- [x] У каждой карточки темы есть меню действий (три точки) с пунктом создания дочерней темы.
- [x] При создании дочерней темы из выбранной ноды автоматически создается зависимость `новая тема зависит от выбранной ноды`.
- [x] Новая нода появляется на следующем уровне графа с фиксированным расстоянием согласно layout-правилам.
- [x] Если создание темы успешно, а добавление зависимости неуспешно, пользователь получает понятную ошибку и не теряет контекст.
- [x] В сценарии создания из ноды не возникает дублей связей и не нарушаются текущие ограничения DAG (цикл запрещен).

### Constraints
- Сохранить существующие возможности ручного редактирования/удаления темы.
- Не менять существующие тексты ошибок backend-кодов (`self_dependency`, `cycle_detected`) без необходимости.
- Если для атомарности нужен новый backend endpoint, не реализовывать его в этой задаче (использовать отдельную backend-задачу).

### Change Summary
**Problem**
- У карточек roadmap не было action-меню «три точки» и сценария создания дочерней темы с автосвязью, поэтому пользователю нужно было выполнять несколько отдельных действий.

**Root Cause Hypothesis**
- `RoadmapView` поддерживал только общий модальный сценарий `Add topic` и отдельное ручное создание зависимостей.
- В состоянии UI отсутствовал контекст «родительская тема -> создание дочерней темы».

**Evidence**
- `frontend/src/components/roadmap-view.tsx`: до изменений не было menu-trigger/dropdown на карточке и логики child-create с автосвязью.
- `frontend/app/globals.css`: отсутствовали стили action-меню карточки темы.
- `frontend/src/components/roadmap-view.tsx`: в ReMake-доработке обновлён `useEffect` очистки `dependencySourceTopicId` — контекст source больше не сбрасывается во время промежуточного `loading` при `roadmap.reload()`.

**Solution**
- Добавлено меню действий (три точки) в карточку темы с пунктом `Создать дочернюю` / `Create child`.
- Реализован child-create поток:
  - определение следующего этапа относительно родительской темы,
  - открытие модалки создания темы в child-режиме с отдельным title/subtitle/button,
  - создание темы и автоматическое создание зависимости `newTopic <- parentTopic`.
- Добавлена проверка edge-case: если следующего этапа нет, показывается понятная ошибка.
- Добавлена обработка частичной ошибки: если тема создана, но автосвязь не создалась, пользователь получает явное сообщение, контекст родителя сохраняется для ручного завершения связи.
- Исправлен ReMake-баг потери контекста при partial-failure: `dependencySourceTopicId` очищается только при `roadmap.state.status === success` и реальном отсутствии source-темы в данных.
- Обновлён helper создания темы: теперь извлекается `topicId` из ответа API для последующего вызова API зависимостей.
- Добавлены CSS-стили dropdown-меню карточки.
