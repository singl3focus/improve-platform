## TSK-0038 — Редактирование этапа roadmap через модальное окно из карточки этапа
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Сейчас редактирование этапа выполняется через inline-формы на странице roadmap.
- По требованию owner редактирование этапа должно запускаться из окна этапа по иконке карандаша справа сверху и открываться поверх страницы (modal), без перехода на отдельную страницу.
- В этом же modal-сценарии должно быть доступно удаление этапа.

### Acceptance Criteria
- [x] В карточке/окне этапа добавлена кнопка-иконка "карандаш" в правом верхнем углу.
- [x] По нажатию открывается modal overlay поверх текущего `/roadmap`, без смены маршрута.
- [x] В modal доступны операции: редактирование этапа (минимум title/position) и удаление этапа.
- [x] Реализованы закрытие modal по кнопке отмены и по `Esc`, с корректным управлением фокусом.
- [x] Ошибки API для update/delete этапа отображаются внутри modal и не ломают основной экран roadmap.

### Constraints
- Не открывать отдельную страницу для edit/delete этапа.
- Не менять backend API-контракт stage CRUD.
- Не смешивать с задачей DnD-связей между темами.

### Change Summary
**Problem**
- Редактирование этапа roadmap было реализовано только inline в `roadmap-stage-strip`, без modal UX из карточки этапа.
- Удаление этапа не было частью единого modal-сценария редактирования из окна этапа.

**Root Cause Hypothesis**
- Stage edit/delete логика была привязана к inline-форме внутри strip-плашек этапов, а не к карточке этапа (`roadmap-stage-column`).
- В UI не существовало modal overlay и связанного механизма фокус-менеджмента для stage edit.

**Evidence**
- Обновлен компонент `frontend/src/components/roadmap-view.tsx`:
  - добавлен icon-trigger `Pencil` в `roadmap-stage-header` каждой карточки этапа;
  - удален inline stage-edit form из `roadmap-stage-strip`;
  - добавлен `roadmap-modal` с формой редактирования (title/position) и кнопкой удаления этапа;
  - добавлены `Esc`-закрытие, закрытие по Cancel и возврат фокуса на кнопку-триггер;
  - ошибки `stageMutationError` рендерятся внутри modal при активном редактировании.
- Обновлены стили `frontend/app/globals.css`:
  - `roadmap-stage-edit-trigger`, `roadmap-modal-overlay`, `roadmap-modal`, `roadmap-modal-*` + dark theme варианты.
- Web UI smoke подтвердил сценарий без смены маршрута:
  - open modal: `path=/roadmap`, `modalVisible=true`, delete-кнопка присутствует;
  - after `Esc`: `modalVisible=false`, `path=/roadmap`, фокус возвращен на trigger;
  - after `Cancel`: `modalVisible=false`, `path=/roadmap`;
  - save update: title этапа изменяется (`Stage Modal Updated`) без перехода с `/roadmap`.

**Solution**
- Реализован modal-flow редактирования этапа из карточки этапа:
  - кнопка-иконка карандаша в правом верхнем углу stage-card header;
  - modal overlay поверх текущего `/roadmap` без роут-переходов;
  - операции update/delete доступны прямо в modal.
- Фокус-менеджмент:
  - при открытии modal фокус устанавливается в поле title;
  - по `Esc`/Cancel modal закрывается и фокус возвращается к кнопке, которая открыла modal.
- Ошибки update/delete отображаются внутри modal и не переносятся в основной контент страницы во время открытого modal.

**Tests / Verification**
- Запущено:
  - `cd frontend && npm run lint` — PASS.
  - `cd frontend && npm run build` — PASS.
  - Web UI smoke `/roadmap` — PASS (open modal, save update, delete, cancel, esc, route stability).
- Не запускалось локально:
  - Отдельный автотест e2e-файл (Playwright/Cypress) в репозитории не добавлялся в scope этой задачи.

**Impact**
- UX: редактирование/удаление этапа перенесено в единый modal-сценарий из карточки этапа.
- API-контракт stage CRUD: без изменений (используются существующие `/api/roadmap/stages/*`).
- Маршрутизация: без изменений, работа остается на `/roadmap`.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: удаление этапа по-прежнему подтверждается `window.confirm`; UX подтверждения может потребовать отдельной продуктовой доработки.
- Rollback plan:
  - вернуть `frontend/src/components/roadmap-view.tsx` к inline-редактированию в `roadmap-stage-strip`;
  - удалить modal-related классы из `frontend/app/globals.css`.
