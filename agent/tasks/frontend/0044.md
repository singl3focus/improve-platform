## TSK-0044 — Tasks: убрать inline-блок «Добавить задачу», вынести кнопку создания вправо над канбаном и открыть форму в modal
- Status: Finished

### Context
- Сейчас раздел задач содержит отдельный inline-блок «Добавить задачу» перед колонками канбана.
- По требованию owner этот блок должен быть удалён: кнопка создания задачи должна находиться справа сверху над канбаном, а форма должна открываться во всплывающем окне с текущим содержимым.

### Acceptance Criteria
- [x] Inline-блок «Добавить задачу» удалён из основного потока страницы задач.
- [x] Кнопка «Создать задачу» расположена справа в верхней части над канбан-колонками.
- [x] Клик по кнопке открывает modal с текущим содержимым формы создания задачи (поля/валидации/submit-flow сохранены).
- [x] После блока «Личный Kanban обучения ...» и верхней строки управления сразу отображаются колонки канбана, без отдельного create-блока между ними.
- [x] После успешного создания modal закрывается, новая задача появляется в колонке «Новая».
- [x] Закрытие modal поддержано (Cancel/Esc), маршрут не меняется.

### Constraints
- Не менять backend API-контракт задач.
- Не менять DnD-поведение канбан-колонок.
- Не менять фильтры по теме и дедлайну.

### Change Summary
**Problem**
- На странице `/tasks` форма создания задачи рендерилась отдельным inline-блоком между шапкой и канбан-колонками, что не соответствует требованию owner по компоновке экрана.

**Root Cause Hypothesis**
- Исторически создание задачи было реализовано как встроенная панель (`TasksCreatePanel`) в основном потоке `TasksKanbanView`, без modal-режима и без отдельной кнопки-триггера в верхней строке управления.

**Evidence**
- В `frontend/src/components/tasks-kanban-view.tsx` до изменения `TasksCreatePanel` рендерился сразу после `TasksBoardHeader`, до `tasks-board-columns`.
- После изменения:
  - в `TasksBoardHeader` добавлена кнопка `createButton` справа в верхней зоне управления;
  - inline-рендер `TasksCreatePanel` удалён из потока страницы и перенесён в modal-оверлей;
  - submit формы проходит через `handleCreateSubmit` с закрытием modal только при успешном создании.
- В `frontend/src/components/hooks/use-tasks-board-view-model.ts` `handleCreate` теперь возвращает `Promise<boolean>`, что позволяет корректно закрывать modal только при success.
- Проверки команд:
  - `npm run lint` → PASS;
  - `npm run build` → PASS.
- Manual smoke (фактический):
  - `/tasks` на локальном сервере `:3026` подтверждает отсутствие inline create-блока;
  - create-кнопка расположена справа в header-control зоне и открывает modal;
  - `Cancel`/`Esc`/overlay закрывают modal без смены маршрута;
  - после submit созданная карточка отображается в колонке `New`.

**Solution**
- Перенесён create-flow задач в modal в `frontend/src/components/tasks-kanban-view.tsx`:
  - добавлено состояние открытия/закрытия modal;
  - добавлен trigger-кнопка `Создать задачу` в верхней правой части header;
  - сохранено текущее содержимое формы (`TasksCreatePanel`) и существующая валидация/submit-flow.
- Добавлено закрытие modal:
  - по кнопке `Отмена`/`Cancel`;
  - по кнопке `×`;
  - по клавише `Esc`;
  - по клику на overlay.
- После успешного создания задача остаётся в текущем потоке данных и появляется в колонке `new` (как и раньше), затем modal закрывается.
- Добавлены стили для новой верхней строки управления и modal-варианта формы в `frontend/app/globals.css`.

**Tests / Verification**

  - `Set-Location frontend; npm run build` — PASS.
  - Manual UI smoke `/tasks` на `http://127.0.0.1:3026` — PASS (кнопка справа в header, modal open/close, успешное создание задачи в колонке `New`, URL без изменений).
