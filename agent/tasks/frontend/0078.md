## TSK-0078 — Roadmap canvas: pan/zoom и центрирование первой карточки для пустого roadmap
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-20
- Updated: 2026-03-21

### Context
- Текущий roadmap-граф поддерживает только scroll-контейнер и не дает полноценной навигации по сцене (drag-pan, wheel-zoom, контролируемый масштаб).
- Требуется поведение «как рабочее полотно»: перемещение зажатой мышью, масштаб колесом и кнопками, а также стартовое центрирование первой карточки только в сценарии пустого roadmap после создания первой темы.

### Acceptance Criteria
- [x] В roadmap реализован drag-pan сцены зажатой мышью (desktop), без поломки кликов по карточкам и управляющим кнопкам.
- [x] Масштаб сцены меняется колесом мыши и кнопками «+»/«-» в интерфейсе; масштаб ограничен безопасными min/max границами.
- [x] Граф автоматически расширяет рабочую область так, чтобы не обрезать карточки и связи при росте дерева.
- [x] Первая карточка центрируется по окну только для пустого roadmap (после quick-create первой темы); для непустого roadmap авто-рецентр не выполняется.
- [x] Добавлены/обновлены тесты (unit/integration, где применимо) для math трансформаций и граничных сценариев pan/zoom.

### Constraints
- Не ломать текущий рендер зависимостей и interactive controls на ребрах.
- Не ухудшать mobile-опыт: базовый scroll и доступность должны остаться рабочими.
- Не использовать git-команды.

### Change Summary
**Problem**
- Roadmap использовал только scroll-контейнер без desktop drag-pan и контролируемого zoom.
- SVG-сцена связей ограничивалась viewport-размером контейнера и могла визуально обрезаться при росте дерева.
- Для empty-roadmap после quick-create первой темы не было гарантированного одноразового центрирования карточки.

**Root Cause Hypothesis**
- Координатная система сцены не имела явной transform-модели (scale + offset), поэтому не было единой точки управления pan/zoom.
- Размер рабочей области брался из clientWidth/clientHeight, а не из полного scroll-графа.
- Для quick-create отсутствовал флаг/эффект, который выполняет центрирование только в empty-roadmap flow.
- Порог mobile/desktop был рассинхронизирован между JS (`900px`) и CSS (`960px`), что создавало промежуточную зону с непредсказуемым UX.

**Evidence**
- Кодовая точка интеграции canvas transform и контролов: `frontend/src/components/roadmap-view.tsx`.
- В `frontend/src/components/roadmap-view.tsx` обработчики `handleGraphPointerDown` и `handleGraphWheel` переведены на единый порог `ROADMAP_MOBILE_BREAKPOINT = 960`.
- Математические функции трансформации уже были в `frontend/src/lib/roadmap-graph.ts`; они использованы для zoom-anchor и нормализации pointer-точек.
- Команда: `npm --prefix frontend run test` → PASS (36/36).
- Команда: `npm --prefix frontend run build` → PASS (production build + lint/type checks).
- Команда: `go test ./...` (из `backend/`) → PASS (`ok` по пакетам, без падений).
- Ручная проверка `http://127.0.0.1:3000/roadmap`: quick-create центрирует первую тему (`translate(254.2px, -104.3px)`), повторный reload непустого roadmap не делает авто-рецентр (`translate(0px, 0px)`), кнопки `+/-/Центр` изменяют `scale` ожидаемо (`1 -> 1.12 -> 1 -> 0.88`), переход по карточке темы после трансформаций успешен (`/topics?topicId=...`).
- Ограничение среды подтверждено фактом: `window.innerWidth = 882`, `matchMedia("(max-width: 960px)") = true`, и попытка выставить viewport `1400x900` не меняет эти значения; поэтому desktop drag-pan/wheel во встроенном browser tool не активируется.

**Solution**
- Внедрена canvas transform-модель в roadmap: `scale`, `offsetX`, `offsetY`.
- Добавлены desktop pan/zoom-обработчики:
  - drag-pan по пустой области сцены,
  - wheel-zoom с anchor-point под курсором,
  - UI-кнопки `-`, `+`, `Центр`.
- Для JS-обработчиков pan/wheel установлен единый с CSS breakpoint `960px`, чтобы исключить рассинхрон desktop/mobile поведения.
- Добавлен одноразовый empty-roadmap флоу: после успешного quick-create ставится флаг и первая карточка центрируется в viewport.
- Нормализация dependency drag и anchor-точек обновлена с учетом текущей transform.
- Пересчитан размер графа через `scrollWidth/scrollHeight` (с fallback), чтобы сцена и связи не обрезались при росте дерева.
- Обновлены unit-тесты `roadmap-layout.test.ts` новым кейсом на расширение render size по scroll-измерениям.

**Tests / Verification**
- `npm --prefix frontend run test` → PASS.
- `npm --prefix frontend run build` → PASS.
- `go test ./...` (из `backend/`) → PASS.
- Manual browser check в локальном docker-стеке (`http://127.0.0.1:3000/roadmap`) → PARTIAL PASS:
  - подтвержден quick-create flow с центрированием первой темы,
  - подтверждено отсутствие авто-рецентра для непустого roadmap после reload,
  - подтверждена работа кнопок `+/-/Центр` по фактическому изменению `transform/scale`,
  - подтверждено, что переход по карточке темы не ломается после zoom/recenter,
  - desktop drag-pan/wheel остаются без устойчивого подтверждения в текущем browser tool; требуется отдельная ручная проверка в desktop-браузере при viewport > `960px`.
- Причина невозможности закрыть desktop-проверку в этой среде:
  - browser tool принудительно остаётся в mobile viewport (`window.innerWidth = 882`, `matchMedia("(max-width: 960px)") = true`),
  - попытка смены viewport на `1400x900` не меняет режим,
  - из-за этого desktop-обработчики панорамирования/масштабирования не активируются и не могут быть подтверждены как пользовательский сценарий.

**Impact**
- Затронут frontend roadmap UI/UX: навигация по сцене, zoom/pan, поведение quick-create на пустом roadmap.
- API-контракты и формат данных не менялись.
- Обратная совместимость: да (mobile fallback сохранен: scroll и доступность остались).

**Risks & Rollback**
- Риск: отличия desktop/mobilе поведения на граничных ширинах (переключение media-query 960px).
- Риск: возможные edge-cases pointer-событий на нестандартных тачпадах/браузерах.
- Риск приёмки: без внешнего desktop-прогона в браузере при ширине > `960px` остаётся неполное доказательство AC по desktop drag-pan/wheel.
- Rollback plan: откатить изменения в
  - `frontend/src/components/roadmap-view.tsx`,
  - `frontend/src/components/roadmap/use-roadmap-graph-layout.ts`,
  - `frontend/src/lib/roadmap-layout.ts`,
  - `frontend/src/lib/roadmap-layout.test.ts`,
  - `frontend/app/globals.css`.
