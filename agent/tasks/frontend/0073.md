- Status: Finished
- Type: Technical
- Approval: Approved

### Context
- Фикс коллизий `position` в roadmap критичен для UX и должен быть защищен автопроверками.
- Ранее регресс проявлялся в create-flow (обычная тема и дочерняя тема) и визуально выглядел как перезапись карточек.

### Acceptance Criteria
- [x] Добавлены автоматизированные проверки вычисления next position для roadmap topic create-flow: кейсы с gap и с дублирующимися позициями.
- [x] Добавлены проверки, подтверждающие отсутствие коллизии layout-позиции для roadmap карточек при данных с конфликтными `position`.
- [x] Добавлен проверяемый сценарий child-topic create, который подтверждает: после создания дочерней темы карточка рендерится отдельно и dependency flow сохраняется.
- [x] Проверки интегрированы в существующий verification pipeline frontend без нарушения текущих тестов.

### Constraints
- Не подменять бизнес-фиксы моками в production-коде.
- Новые проверки должны быть детерминированными и воспроизводимыми локально.
- Не расширять задачу на unrelated e2e-сценарии вне roadmap create/dependency scope.

### Change Summary
**Problem**
- В ReMake ревью выявило недоказанный `verify:mvp`: часть сценариев запускалась без подтвержденной auth-сессии для private BFF routes, из-за чего в окружении ревью получались `401`.
- В ручном quick-create на `/roadmap` фиксировался `500`, и create-flow обрывался при ошибках read-path (`/api/v1/roadmap`) либо последующих backend-ошибках.

**Root Cause Hypothesis**
- Smoke-скрипт делал запросы к защищенным маршрутам без явного установления валидной сессии (cookie jar + login step отсутствовали).
- Quick-create/topic-create в BFF жёстко зависели от успешного `GET /api/v1/roadmap`; при backend `5xx` create-flow завершался ошибкой вместо контролируемого fallback.

**Evidence**
- `frontend/scripts/mvp-verification-smoke.mjs` обновлён: добавлен cookie jar, обработка `Set-Cookie`, auth step (`/api/auth/login` + `/api/auth/session`) и сценарий `Auth login establishes session for protected BFF routes`.
- Mock backend внутри `mvp-verification-smoke.mjs` расширен auth-endpoints (`/api/v1/auth/login`, `/api/v1/auth/refresh`) и auth-gate для `/api/v1/**` (кроме `/api/v1/auth/**`), чтобы проверка private BFF routes была реальной, а не анонимной.
- `frontend/app/api/roadmap/quick-create/route.ts`: добавлен fallback `position=1` при `5xx` на read-path roadmap (для ветки `roadmap_exists`) вместо немедленного abort.
- `frontend/app/api/roadmap/topics/route.ts`: добавлен fallback `position=1` при `5xx` на `GET /api/v1/roadmap`.
- Повторный ручной прогон в браузере даёт `500` из backend `UseCase.CreateTopic` с `SQLSTATE 0A000` (вне BFF-кода).

**Solution**
- Сделан воспроизводимый auth-aware smoke pipeline для `verify:mvp` (login + cookie session + проверка protected BFF routes).
- Усилен встроенный mock backend smoke-скрипта, чтобы private API сценарии падали без авторизации и подтверждали корректный happy-path только после auth.
- Добавлен минимальный resilience fallback в roadmap create-flow BFF routes (`quick-create` и `topics create`) для backend read-path `5xx`.
