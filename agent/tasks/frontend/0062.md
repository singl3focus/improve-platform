## TSK-0062 — Удаление зависимости крестиком на линии связи Roadmap
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-17
- Updated: 2026-03-17

### Context
- Owner требует удалять связь прямо на линии графа, а не кнопкой внутри карточки темы.
- Текущее поведение удаляет зависимости только из блока prerequisites в карточке, что не соответствует ожидаемому UX полноценного графа.

### Acceptance Criteria
- [x] На каждой отображаемой связи графа Roadmap есть интерактивный control удаления (крестик) прямо на линии.
- [x] Удаление связи через крестик вызывает существующий API удаления зависимости и удаляет связь из графа без перезагрузки страницы.
- [x] Кнопка удаления связи из карточки темы убрана из UI.
- [x] Ошибки удаления зависимости (4xx/5xx/network) показываются пользователю в существующей error-области без silent fail.
- [x] Доступность сохранена: control удаления доступен с клавиатуры и имеет понятный aria-label.

### Constraints
- Не менять backend-контракт удаления зависимости.
- Не ломать текущий drag-and-drop сценарий создания зависимости.
- Не использовать git-команды.

### Change Summary
**Problem**
- Удаление зависимостей было доступно только внутри карточки темы в блоке prerequisites, а не на линии графа.
- Это не соответствовало ожидаемому UX удаления связи непосредственно на ребре.

**Root Cause Hypothesis**
- UI удаления связи был привязан к списку `topic.prerequisiteTopicIds` в карточке темы.
- SVG-линии графа рендерились без интерактивных контролов удаления.

**Evidence**
- `frontend/src/components/roadmap-view.tsx`: удаление зависимости вызывалось из кнопки в `roadmap-prerequisite-chip`.
- `frontend/src/components/roadmap-view.tsx`: рендер линий connections не содержал edge-control.
- Команды проверки:
  - `npm run lint` -> PASS.
  - `npm run build` -> PASS.

**Solution**
- Добавлен интерактивный control удаления на каждую отображаемую связь графа (крестик на линии).
- Edge-control вызывает существующий `handleDependencyDelete` и текущий API удаления зависимости.
- Добавлены aria-label для edge-control в ru/en локализации.
- Удалён control удаления зависимости из карточки темы (в prerequisites оставлены только названия зависимостей).
- Добавлены CSS-стили для edge-control, включая focus-visible и dark theme.

**Tests / Verification**
- `Set-Location frontend; npm run lint` -> PASS.
- `Set-Location frontend; npm run build` -> PASS.
- Manual UI не запускался в этой подзадаче (выполняется единым ручным прогоном после завершения оставшихся задач).

**Impact**
- UI Roadmap: изменён способ удаления зависимости (с карточки на линию графа).
- API/контракт backend: без изменений (используется текущий endpoint удаления зависимости).
- Обратная совместимость: да, по контрактам.

**Risks & Rollback**
- Риск: на плотных графах edge-control может визуально пересекаться с линиями.
- Rollback: вернуть удаление из блока prerequisites и убрать edge-controls из `roadmap-view.tsx` и связанных CSS-стилей.
