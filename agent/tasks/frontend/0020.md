## TSK-0020 — Quick-create первой темы на пустом roadmap
- Status: ReMake
- Type: Business
- Approval: Approved
- Created: 2026-03-15
- Updated: 2026-03-17 (Reviewer)

### Context
- Сейчас при пустом roadmap пользователь видит только сообщение "Roadmap пока пуст..." без действия для добавления первой темы.
- На экране `/topics` уже есть CTA "Откройте roadmap", но в `/roadmap` отсутствует быстрый сценарий первичного создания.
- Нужно реализовать полноценный quick-create на пустом roadmap, чтобы пользователь мог создать структуру и первую тему без ручных API-шагов.

### Acceptance Criteria
- [x] На пустом состоянии `/roadmap` отображается CTA для быстрого создания первой темы.
- [x] Quick-create сценарий создает необходимые сущности через backend API: roadmap (если отсутствует), stage и topic.
- [x] После успешного создания пользователь видит созданную тему на roadmap без ручной перезагрузки страницы.
- [x] При ошибках валидации/конфликтах/сетевых ошибках пользователь получает понятное сообщение, страница не ломается.
- [x] В frontend BFF добавлены необходимые endpoint-ы для quick-create потока (без прямого вызова backend из UI-слоя).

### Constraints
- Не менять backend бизнес-правила DAG и валидацию доменных сущностей.
- Фокус задачи только на quick-create первого пути в roadmap; не делать общий редактор roadmap.
- Изменения должны быть совместимы с текущими контрактами `/api/v1/roadmap`, `/api/v1/roadmap/stages`, `/api/v1/roadmap/topics`.

### Change Summary
**Problem**
- На пустом `/roadmap` отображалось только текстовое сообщение без действия для первичного старта.
- В BFF отсутствовал endpoint, который создаёт roadmap/stage/topic в одном quick-create потоке.

**Root Cause Hypothesis**
- `frontend/src/components/roadmap-view.tsx` реализовывал только чтение `GET /api/roadmap`, без мутаций для пустого состояния.
- `frontend/app/api/roadmap/*` содержал только `GET`-роут и не покрывал цепочку `roadmap -> stage -> topic`.

**Evidence**
- `frontend/src/components/roadmap-view.tsx:57` и `:92` — добавлены copy-блоки quick-create (RU/EN).
- `frontend/src/components/roadmap-view.tsx:159` — добавлен клиентский вызов `POST /api/roadmap/quick-create`.
- `frontend/src/components/roadmap-view.tsx:382` — добавлен `handleQuickCreate` с валидацией/ошибками/reload.
- `frontend/src/components/roadmap-view.tsx:591` — на пустом roadmap рендерится quick-create CTA + форма.
- `frontend/app/api/roadmap/quick-create/route.ts:37` — новый BFF endpoint `POST`.
- `frontend/app/api/roadmap/quick-create/route.ts:118` и `:143` — создание stage и topic через backend API.
- `frontend/app/globals.css:584` и `:589` — стили для empty-panel и quick-create блока.
- `frontend/scripts/mvp-verification-smoke.mjs:799` — добавлен smoke-сценарий quick-create.
- Команды:
  - `npm run lint` → PASS.
  - `npm run build` → PASS (вне sandbox).
  - `FRONTEND_BASE_URL=http://127.0.0.1:3030 npm run verify:mvp` → PASS (включая quick-create сценарий).
  - `npx tsc --noEmit` → PASS.

**Solution**
- Добавлен новый BFF endpoint `POST /api/roadmap/quick-create`:
  - валидирует payload quick-create;
  - загружает roadmap; если roadmap отсутствует (`roadmap_not_found`), создаёт его;
  - создаёт stage, если stages ещё нет;
  - создаёт первую topic в выбранном/созданном stage;
  - отдаёт `201` и нормализованные ошибки backend.
- В `RoadmapView` добавлен quick-create UI для пустого состояния:
  - CTA с формой (title + description);
  - клиентская валидация обязательного названия темы;
  - обработка loading/error без ломания страницы;
  - автоматический `roadmap.reload()` после успешного создания для отображения темы без ручной перезагрузки.
- Добавлены стили quick-create блока в `globals.css`.
- Расширен `verify:mvp`:
  - mock backend теперь поддерживает `POST /api/v1/roadmap`, `/api/v1/roadmap/stages`, `/api/v1/roadmap/topics`;
  - добавлен сценарий quick-create на пустом roadmap.

**Tests / Verification**
- Запущены `npm run lint`, `npm run build`, `npm run verify:mvp`, `npx tsc --noEmit`.
- `build` и валидный `verify:mvp` выполнялись вне sandbox из-за системной ошибки `spawn EPERM` в sandbox.
- Ручной браузерный smoke для `/roadmap` не запускался в этой сессии.

**Impact**
- UI: изменён пустой state `/roadmap` (добавлен quick-create поток).
- API (frontend BFF): добавлен новый endpoint `POST /api/roadmap/quick-create`.
- Test tooling: расширен `frontend/scripts/mvp-verification-smoke.mjs`.
- Backend контракт не изменён, используются существующие `/api/v1/roadmap`, `/api/v1/roadmap/stages`, `/api/v1/roadmap/topics`.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: ручной UI сценарий не подтверждён в браузере в этой сессии.
- Риск: запуск команд внутри sandbox нестабилен из-за `spawn EPERM`; для CI/локали рекомендуется обычный runtime без sandbox-ограничений.
- Rollback:
  - удалить `frontend/app/api/roadmap/quick-create/route.ts`;
  - откатить изменения в `frontend/src/components/roadmap-view.tsx`;
  - откатить стили `frontend/app/globals.css`;
  - откатить изменения в `frontend/scripts/mvp-verification-smoke.mjs`.
