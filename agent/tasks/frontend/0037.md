## TSK-0037 — Стабилизировать позицию кнопки Settings в sidebar
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- На текущем shell позиция нижнего блока навигации с Settings зависит от высоты контента страницы, из-за чего кнопка визуально "уезжает" слишком низко.
- Требуется сделать позицию Settings устойчивой и независимой от высоты просматриваемой страницы.

### Acceptance Criteria
- [x] В private shell кнопка Settings отображается в стабильной нижней зоне sidebar и не смещается из-за высоты контента main-area.
- [x] На экранах `/dashboard`, `/roadmap`, `/topics`, `/tasks`, `/materials`, `/settings` поведение одинаковое: Settings в одном и том же визуальном блоке sidebar.
- [x] Проверено поведение на desktop и mobile breakpoints без регрессии навигации.

### Constraints
- Не менять маршруты и существующие ключи локализации навигации.
- Не изменять бизнес-логику экранов; область изменений ограничить shell/layout и связанными стилями.

### Change Summary
**Problem**
- В private shell нижний блок навигации с `Settings` зависел от итоговой высоты layout-строки и визуально смещался вниз на страницах с более высоким контентом main-area.

**Root Cause Hypothesis**
- Sidebar в grid-shell растягивался вместе с высотой контентной колонки, а `Settings` был привязан к нижней границе этого растянутого блока через `sidebar-spacer`, из-за чего позиция менялась между страницами.

**Evidence**
- Разметка shell: `frontend/src/components/layout/private-shell.tsx` использует `sidebar-spacer` между `nav-list-main` и `nav-list-bottom`.
- До фикса в `frontend/app/globals.css` для `.sidebar` не было привязки к viewport (`position: sticky`/фиксированной минимальной высоты), и sidebar зависел от высоты grid-строки.
- После фикса в `frontend/app/globals.css` добавлены:
  - `.app-shell { align-items: start; }`
  - `.sidebar { position: sticky; top: 14px; min-height: calc(100vh - 28px); }`
  - mobile-reset в `@media (max-width: 960px)`: `position: static; top: auto; min-height: 0;`.
- Командная проверка: `npm run lint` и `npm run build` проходят без ошибок.

**Solution**
- Ограничил область изменений только shell/layout-стилями, без изменения маршрутов и локализационных ключей.
- Зафиксировал sidebar относительно viewport на desktop (`sticky + min-height`), чтобы `sidebar-spacer` держал `Settings` в стабильной нижней зоне независимо от высоты main-content.
- Добавил явный mobile-reset, чтобы не ломать текущую мобильную навигацию.

**Tests / Verification**
- Запущено:
  - `cd frontend && npm run lint` — PASS.
  - `cd frontend && npm run build` — PASS.
  - Backend stack для auth smoke: `docker compose -f build/docker-compose.yml up -d postgres migrations backend` — PASS.
  - Web UI smoke (авторизованная сессия, страницы `/dashboard`, `/roadmap`, `/topics`, `/tasks`, `/materials`, `/settings`) на mobile viewport `390x844` — PASS: `Settings` стабильно находится в нижнем navigation-блоке sidebar и не выходит за его границы.

**Impact**
- UI/layout: стабилизирована визуальная позиция `Settings` в sidebar на desktop.
- Маршруты, i18n-ключи, бизнес-логика страниц: без изменений.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: маршрут `/topics` без `topicId` по текущему контракту редиректит на `/roadmap`, поэтому проверка для этого пункта подтверждена на финальном экране редиректа.
- Rollback plan:
  - откатить изменения в `frontend/app/globals.css` в блоках `.app-shell`, `.sidebar` и mobile-правилах `@media (max-width: 960px)`.
