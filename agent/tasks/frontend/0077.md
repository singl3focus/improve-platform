## TSK-0077 — Roadmap directional create без деградации left в below
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-20
- Updated: 2026-03-20

### Context
- В текущем flow создание темы «слева» может терять направленную семантику из-за position-эвристики, особенно у темы с минимальной позицией.
- Требуется стабильное поведение tree-like: left/right/below должны соответствовать выбранному направлению, а не случайной визуальной перестановке.

### Acceptance Criteria
- [x] Создание темы через меню «Создать слева/справа/ниже» использует directional-контракт (`direction`, `relative_to_topic_id`), без локальной подмены направления через `position`.
- [x] Для кейса «родитель с минимальной позицией» действие «Создать слева» приводит к размещению темы слева от родителя в графе, а не ниже.
- [x] После directional-create автоматически отображается dependency-стрелка между текущей и созданной темой для всех направлений (в соответствии с правилом parent blocked / child ready).
- [x] Для любого directional-create (`left/right/below`) новая дочерняя тема сразу доступна для старта (не blocked из-за родителя), а текущая родительская тема становится blocked до завершения дочерней темы.
- [x] Добавлены/обновлены unit-тесты фронтенда на directional-flow и отсутствие регресса существующих сценариев roadmap create.

### Constraints
- Не ломать существующую навигацию в карточку темы и текущие контракты roadmap API.
- Не ослаблять проверки cycle/self/duplicate dependency.
- Не использовать git-команды.

### Change Summary
**Problem**
- В ReMake-ревью зафиксирован незакрытый инвариант directional create: для `left/right/below` нужно гарантировать, что новая тема сразу доступна (`isBlocked=false`), а текущая родительская тема становится blocked до выполнения дочерней.

**Root Cause Hypothesis**
- Directional create полагался только на backend side-effect по автозависимости, без явной frontend/BFF-проверки и без нормализующего шага на случай обратного направления зависимости.
- Не было unit-покрытия на инвариант направления автозависимости по данным roadmap payload.

**Evidence**
- `frontend/app/api/roadmap/topics/route.ts`: в directional path добавлен post-create шаг нормализации зависимости по roadmap snapshot.
- `frontend/src/lib/roadmap-topic-create.ts`: добавлен helper `buildDirectionalDependencyPlan(...)`, который вычисляет, нужно ли:
  - добавить `parent -> created` dependency,
  - удалить обратную `created -> parent` dependency.
- `frontend/src/lib/roadmap-topic-create.test.ts`: добавлены кейсы на expected/no-op, reverse-repair и staged payload (`prerequisiteTopicIds`).
- Ручной browser-check на `http://127.0.0.1:3025/roadmap` подтвердил инвариант для всех трех действий (`left/right/below`): parent blocked, created topics unblocked.

**Solution**
- В BFF `POST /api/roadmap/topics` (directional branch) добавлена best-effort нормализация зависимости после успешного создания темы:
  - читается snapshot `/api/v1/roadmap`;
  - вычисляется план правки через `buildDirectionalDependencyPlan`;
  - при необходимости создается связь `parent depends_on created`;
  - при необходимости удаляется обратная связь `created depends_on parent`.
- В helper-слой `frontend/src/lib/roadmap-topic-create.ts` добавлен парсер зависимостей как для flat payload (`topics[].dependencies`), так и для staged payload (`stages[].topics[].prerequisiteTopicIds`).
- Unit-тесты расширены на сценарии инварианта parent/child для directional create.
