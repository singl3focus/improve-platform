## TSK-0048 — Техническая декомпозиция graph-экрана roadmap и стабилизация интеракций
- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- После перехода на полноценный graph-view roadmap возрастает сложность клиентской логики (координаты, связи, drag-интеракции, фокус/клавиатура).
- Нужна техническая стабилизация и декомпозиция, чтобы экран оставался поддерживаемым и проверяемым.

### Acceptance Criteria
- [x] Критическая graph-логика вынесена из монолитного UI-компонента в отдельные модули/хуки (layout/interaction helpers).
- [x] Выделены и покрыты тестами утилиты, от которых зависит корректность графа (например, вычисление путей связей, нормализация координат, edge-cases drag).
- [x] Снижена связность: визуальные подкомпоненты и interaction-логика разделены, публичные интерфейсы модулей описаны типами.
- [x] Поведение пользовательских сценариев графа не деградирует относительно TSK-0047.
- [x] Шрифт приложения не меняется.

### Constraints
- Не изменять бизнес-правила roadmap в backend.
- Не проводить массовый рефакторинг вне roadmap graph scope.
- Сохранять обратную совместимость по API BFF маршрутов frontend.

### Change Summary
**Problem**
- После внедрения graph-view в TSK-0047 критическая логика графа (layout связей, drag-threshold, нормализация координат) оставалась в крупном `roadmap-view.tsx` и не имела автоматических тестов.

**Root Cause Hypothesis**
- Логика layout/interactions добавлялась итеративно в UI-компонент, из-за чего выросла связность между рендером и вычислениями, а ключевые helper-алгоритмы не имели изолированной проверки.

**Evidence**
- До изменений в `frontend/src/components/roadmap-view.tsx` локально содержались:
  - `useRoadmapGraphLayout`,
  - `buildConnectionPath`,
  - `parsePositiveInteger`,
  - inline drag-threshold (`Math.hypot >= 8`) и нормализация preview point.
- После изменений:
  - graph-layout hook вынесен в `frontend/src/components/roadmap/use-roadmap-graph-layout.ts`;
  - utility helpers вынесены в `frontend/src/lib/roadmap-graph.ts`;
  - добавлены unit-tests `frontend/src/lib/roadmap-graph.test.ts`.
- Проверки:
  - `npm run lint` -> PASS,
  - `npm run build` -> PASS,
  - `npm run test` -> PASS (5/5).

**Solution**
- Выделен отдельный hook `useRoadmapGraphLayout` с типизированным публичным интерфейсом (`status/data/graphRef/topicRefs` -> `connections/graphSize`).
- Вынесены типизированные graph utility функции:
  - `parsePositiveInteger`,
  - `buildConnectionPath`,
  - `normalizeGraphPoint`,
  - `isDragGesture`.
- `RoadmapView` переведен на использование вынесенных модулей вместо локальных реализаций, что уменьшило связность UI-компонента и стабилизировало interaction-ветки drag.
- Добавлен `npm run test` без новых библиотек (через `tsc + node:test`) и `tsconfig.test.json` для запуска unit-тестов graph-утилит.

**Tests / Verification**
