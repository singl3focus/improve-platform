## TSK-0019 — Добавить создание задач прямо на канбан-доске `/tasks`
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-15
- Updated: 2026-03-15

### Context
- Текущий экран канбана поддерживает загрузку задач и смену статуса, но не позволяет создавать новые задачи.
- По задаче owner №2 пользователь должен иметь возможность добавлять задачи в канбан-доску.
- Backend уже поддерживает `POST /api/v1/tasks`, поэтому требуется добавить соответствующий frontend BFF и UI-поток создания.

### Acceptance Criteria
- [x] На экране `/tasks` доступна форма добавления задачи с полями: `title` (обязательно), `description` (опционально), `topicId` (опционально), `deadline` (опционально).
- [x] После успешного создания задача появляется на доске в колонке `new` без ручного обновления страницы.
- [x] Валидационные ошибки создания (например, пустой `title`) отображаются в UI как пользовательская ошибка и не ломают текущую доску.
- [x] Добавлен `POST`-обработчик в `frontend/app/api/tasks/route.ts`, проксирующий создание в backend `/api/v1/tasks`.
- [x] Существующие функции канбана (фильтры, смена статуса) не регрессируют.
- [x] `npm run lint`, `npm run build`, `npm run verify:mvp` проходят успешно после изменений.

### Constraints
- Не менять backend API и модель задачи в рамках этой задачи.
- Не делать редизайн канбана; добавить минимально необходимый UX для создания.
- Не смешивать в этой задаче доработки несвязанных экранов.

### Change Summary
**Problem**
- На экране `/tasks` не было пользовательского потока создания задачи.
- BFF-роут `frontend/app/api/tasks/route.ts` поддерживал только `GET`, поэтому frontend не мог проксировать `POST /api/v1/tasks`.

**Root Cause Hypothesis**
- Отсутствовали UI-форма и client-side обработчик создания в канбане.
- Отсутствовал `POST`-обработчик BFF для задач.
- В smoke-проверках не было сценария на `POST /api/tasks`.

**Evidence**
- `frontend/src/components/tasks-kanban-view.tsx:241` — добавлен `TasksCreatePanel` с полями `title/description/topicId/deadline`.
- `frontend/src/components/hooks/use-tasks-board-view-model.ts:268` — добавлен `handleCreate` с вызовом `POST /api/tasks`.
- `frontend/src/components/hooks/use-tasks-board-view-model.ts:313` — новая задача добавляется в локальный state (`tasks: [normalizedTask, ...current.data.tasks]`) без ручного обновления страницы.
- `frontend/app/api/tasks/route.ts:134` — добавлен `export async function POST`.
- `frontend/scripts/mvp-verification-smoke.mjs:558` — добавлен smoke-сценарий `Tasks POST route keeps success and validation errors`.
- Команды:
  - `npm run lint` → PASS.
  - `npm run build` → PASS.
  - `FRONTEND_BASE_URL=http://127.0.0.1:3030 npm run verify:mvp` → PASS.

**Solution**
- Добавлена форма создания задачи на `/tasks`:
  - обязательный `title`,
  - опциональные `description`, `topicId`, `deadline`.
- Во view-model канбана добавлены:
  - draft-состояние формы,
  - `createTask` (`POST /api/tasks`),
  - `handleCreate` с показом пользовательских ошибок и без ломания доски,
  - мгновенное добавление созданной задачи в локальный список с учётом активных фильтров.
- В BFF `frontend/app/api/tasks/route.ts` добавлен `POST`:
  - валидация payload,
  - проксирование в backend `POST /api/v1/tasks`,
  - возврат созданной задачи в формате канбана.
- В `verify:mvp` добавлены:
  - поддержка mock backend `POST /api/v1/tasks`,
  - проверка success + validation error для `POST /api/tasks`.
- Добавлены стили формы создания в `frontend/app/globals.css`.

**Tests / Verification**

- Запущено `npm run build` — PASS.
