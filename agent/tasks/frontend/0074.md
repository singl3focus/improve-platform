### Change Summary
**Problem**
- В `ReMake` ревьюер зафиксировал незакрытые Acceptance Criteria, пустой `Change Summary` и `NOT RUN` во всех проверках.
- В frontend оставались устаревшие/неиспользуемые roadmap-ветки, которые мешали сопровождению (legacy child-create copy + неиспользуемая layout-функция).

**Root Cause Hypothesis**
- Ранее задача была закрыта без формального заполнения артефактов и без фактических запусков обязательных проверок.
- После эволюции roadmap-UX часть старых строк/веток осталась в коде как технический хвост.

**Evidence**
- `frontend/src/components/roadmap-view.tsx`: удалены неиспользуемые stage-management copy-ключи (`stageManageTitle`, `stageManageSubtitle`, `stageFieldTitle`, `stageFieldPosition`, `stageTitlePlaceholder`, `stageCreateButton`, `stageCreatingButton`, `stageEditButton`, `stageDeleteButton`, `stageSaveButton`, `stageCancelButton`, `stageUpdatingButton`, `stageDeletingButton`, `stageTitleRequired`, `stageCreateFailed`, `stageUpdateFailed`, `stageDeleteFailed`, `stageDeleteConfirm`, `stageCreateSuccess`, `stageUpdateSuccess`, `stageDeleteSuccess`).
- `frontend/src/components/roadmap-view.tsx`: удалена неиспользуемая helper-ветка `buildTopicLevelById(...)` и связанные старые вычисления `stagePositionById/topicLevelById`.
- `frontend/src/lib/roadmap-layout.ts`: устранено дублирование layout-логики через единый placement builder (`buildTopicGridPlacementById`) и reuse в `buildTopicGridColumnById`.

**Solution**
- Удалён/упрощён устаревший и неиспользуемый roadmap-код в frontend без изменения контрактов backend.
- Обновлена roadmap-layout реализация к единому пути вычисления позиций карточек.
- Задача 0074 документирована по обязательному шаблону и подтверждена фактическими прогонами.

**Tests / Verification**
- Запущено:
  - `npm run lint` — PASS.
  - `npm run test` — PASS (`29 passed, 0 failed`).
  - `npm run build` — PASS.
  - `npm run verify:mvp` — PASS (ключевые dashboard/roadmap/tasks/materials сценарии).
- Ручная браузерная проверка private-навигации и auth-redirect — PASS.

**Impact**
- Затронуты только frontend roadmap-компонент и layout-утилиты.
- API-контракты, миграции и backend-данные не изменены.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: низкий; изменения локальны и покрыты тестами/смоуком.
- Rollback: вернуть правки в `frontend/src/components/roadmap-view.tsx` и `frontend/src/lib/roadmap-layout.ts`/`frontend/src/lib/roadmap-layout.test.ts`.
