## TSK-0007 — Экран материалов как личная библиотека
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-14
- Updated: 2026-03-14

### Context
- Материалы в MVP должны восприниматься как curated library, а не файловый менеджер.
- Нужен экран, где можно быстро видеть ресурс, связанную тему и прогресс по материалу.
- Для визуального направления использовать `design/_5/code.html` как референс библиотеки материалов.

### Acceptance Criteria
- [x] Экран материалов показывает карточки с `title`, `description`, `topic`, `progress`, `position`.
- [x] Доступны создание/редактирование/удаление материала и изменение прогресса материала.
- [x] Поддерживается переход к теме материала.
- [x] Отображение порядка материалов внутри темы согласовано с backend `position`.
- [x] Визуальная композиция библиотеки (поиск/фильтры, карточки, прогресс) согласована с `design/_5/code.html` без требования полного визуального совпадения.

### Constraints
- Не добавлять жесткую типизацию материалов в MVP UI.
- Прогресс материала не должен визуально интерпретироваться как прогресс темы.
- Сценарии должны работать на server state (TanStack Query) без дублирующего глобального стейта.
- `design/_5/code.html` использовать как пример визуала; функциональные требования берутся из этой задачи и `docs/bussines/v1/domain.md`.
- Не внедрять дополнительные сценарии из макета, если они не описаны в acceptance criteria.

### Change Summary
**Problem**
- После Re-review экран `materials` использовал локальный shim вместо официального `@tanstack/react-query`, что нарушало constraints задачи.
- После переключения на официальный пакет проявилась несовместимость generic-сигнатуры `useMutation` в `materials-library-view`.

**Root Cause Hypothesis**
- В `frontend/tsconfig.json` был alias `@tanstack/react-query -> src/vendor/tanstack-react-query`, а в `frontend/package.json` отсутствовала зависимость на официальный пакет.
- Локальный shim принимал другие generic-параметры в `useMutation`, поэтому типы из старой реализации не совпали с API официальной библиотеки.

**Evidence**
- `frontend/package.json`: добавлен dependency `@tanstack/react-query` (`rg` вернул строку `12`).
- `frontend/tsconfig.json`: alias на `src/vendor/tanstack-react-query` удален (`rg` не нашел совпадений).
- `frontend/src/vendor/tanstack-react-query.tsx`: удален.
- `frontend/src/components/materials-library-view.tsx`: generic-типизация `useMutation` приведена к официальной сигнатуре v5 (`<TData, TError, TVariables>`).
- `npm run lint` PASS, `npm run build` PASS (вне sandbox), runtime smoke `/api/materials` PASS (`200/201/200/200/200`, порядок `1,2 -> 1,2,3 -> 1,2`, `empty=200`, `error=500`).

**Solution**
- Добавлен официальный пакет `@tanstack/react-query` в `frontend/package.json` и синхронизирован lockfile через `npm install`.
- Удален alias `@tanstack/react-query` из `frontend/tsconfig.json`.
- Удален локальный shim `frontend/src/vendor/tanstack-react-query.tsx`.
- Обновлена типизация мутаций в `frontend/src/components/materials-library-view.tsx`:
  - `useMutation<LibraryMaterial, Error, CreateMaterialPayload>`
  - `useMutation<LibraryMaterial, Error, PatchMaterialPayload>`
  - `useMutation<void, Error, string>`
- Импорты `AppQueryProvider`/`MaterialsLibraryView` продолжают использовать `@tanstack/react-query`, теперь это официальный пакет.

**Tests / Verification**
- Запущено:
  - `npm install` — PASS (`added 2 packages`).
  - `npm run lint` — PASS.
  - `npm run build` — PASS (sandbox: `spawn EPERM`, вне sandbox: PASS).
  - `rg -n '"@tanstack/react-query"' frontend/package.json` — PASS.
  - `rg -n '"@tanstack/react-query": \["src/vendor/tanstack-react-query"\]' frontend/tsconfig.json` — PASS (совпадений нет).
  - Runtime smoke `GET/POST/PATCH/PATCH/DELETE` для `/api/materials` — PASS.
  - Runtime smoke `mode=empty` и `mode=error` — PASS (`200` и `500`).
- Не запускалось локально:
  - Полный browser smoke `/materials` в `npm run dev` (desktop/mobile) с ручной проверкой перехода по topic-link.

**Impact**
- Dependency/runtime: query-слой теперь работает на официальном `@tanstack/react-query`, без локального shim.
- UI/server-state: поведение экрана `materials` (поиск/фильтр/CRUD/progress/topic-link) сохранено.
- API контракт: без изменений (`/api/materials` и `/api/materials/[materialId]`).
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: server-state реализован как in-memory mock; изменения не переживают перезапуск процесса.
- Риск: backend-контракт по материалам может отличаться от текущего mock API.
- Риск: обновление версии `@tanstack/react-query` в будущем может потребовать дополнительных правок типизации.
- Rollback plan:
  - вернуть `frontend/package.json` и `frontend/package-lock.json` до добавления `@tanstack/react-query`;
  - восстановить alias в `frontend/tsconfig.json` и файл `frontend/src/vendor/tanstack-react-query.tsx`;
  - откатить типизацию `useMutation` в `frontend/src/components/materials-library-view.tsx` к предыдущей версии shim.

- `frontend: switch materials query layer to official tanstack package`
