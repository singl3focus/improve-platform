## TSK-0033 — Улучшить UX ползунка прогресса материалов
- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Текущий ползунок прогресса работает неудобно: значение изменяется в основном кликами, а drag-перетаскивание воспринимается нестабильно.
- Нужно обеспечить предсказуемое взаимодействие мышью, тачем и клавиатурой.

### Acceptance Criteria
- [x] Ползунок корректно тянется drag-жестом (мышь/тач) с плавным обновлением значения.
- [x] Доступно управление клавиатурой (`ArrowLeft/ArrowRight`, `Home/End`) с корректными шагами.
- [x] Обновление прогресса на сервер выполняется без лишнего спама запросами (стабильная стратегия commit/debounce).
- [x] Визуальная индикация текущего значения синхронна с фактическим сохраненным значением.

### Constraints
- Не менять API-контракт прогресса материалов.
- Не ухудшить доступность и навигацию с клавиатуры.

### Change Summary
**Problem**
- Ползунок прогресса материалов отправлял PATCH на каждый шаг `onChange`, что приводило к лишним запросам и делало drag-взаимодействие нестабильным.

**Root Cause Hypothesis**
- Логика обновления прогресса была реализована как прямой server-commit на каждом изменении значения range без debounce/commit стратегии.

**Evidence**
- В `frontend/src/components/materials-library-view.tsx` range вызывал обновление в `onChange` сразу на каждое движение.
- В `frontend/src/components/hooks/use-materials-library-view-model.ts` `handleProgressUpdate` сразу вызывал `patchMaterialMutation` для каждого шага.
- После изменений `npm run lint` и `npm run build` проходят успешно.

**Solution**
- В `frontend/src/components/hooks/use-materials-library-view-model.ts` заменена модель обновления прогресса на:
  - optimistic preview в кэше query (`updateProgressInCache`),
  - очередь значений по `materialId`,
  - debounce таймер (`PROGRESS_COMMIT_DEBOUNCE_MS`),
  - последовательный flush commit (`flushProgressCommit`) без request-spam.
- Добавлены отдельные обработчики `handleProgressPreview` и `handleProgressCommit`.
- В `frontend/src/components/materials-library-view.tsx` range обновлен:
  - `onChange` -> preview,
  - commit на `onPointerUp`, `onBlur` и `onKeyUp` для клавиш управления range,
  - добавлен `aria-valuetext` для доступности.

**Tests / Verification**
- Запущен `cd frontend && npm run lint` — PASS.
- Запущен `cd frontend && npm run build` — PASS.
- Manual сценарий не запускался в этом цикле.

**Impact**
- Улучшен UX ползунка прогресса материалов: плавный drag-preview и стабильный server-commit.
- API-контракт прогресса материалов не изменен.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: при нестабильной сети возможны задержки фиксации значения до debounce/commit.
- Rollback: вернуть прежнюю прямую схему обновления из `onChange` и удалить queue/debounce-логику в `use-materials-library-view-model.ts`.
