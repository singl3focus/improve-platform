## TSK-0005 — Экран темы как рабочее пространство
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-14
- Updated: 2026-03-14

### Context
- Экран темы — место ежедневной работы: статус, прогресс, checklist/задачи, материалы, зависимости и даты.
- Нужна ясная структура без перегрузки.
- Для визуального направления использовать `design/_3/code.html` как референс структуры экрана темы.

### Acceptance Criteria
- [x] Экран темы показывает `title`, `description`, `status`, `progress`, ключевые даты и блок зависимостей.
- [x] Встроен блок checklist/tasks по теме с изменением статусов элементов.
- [x] Встроен блок материалов темы с отображением порядка и прогресса материалов.
- [x] После изменения checklist UI обновляет прогресс темы из server state.
- [x] Визуальная структура (hero-блок, блок зависимостей, план/checklist, материалы) согласована с `design/_3/code.html`, но без требований буквального копирования.

### Constraints
- Прогресс темы берется с backend и не редактируется вручную на frontend.
- Нельзя скрывать бизнес-ограничения зависимостей.
- Локальное состояние не должно расходиться с server state после мутаций.
- `design/_3/code.html` использовать только как пример UI; функциональное поведение определяется задачей и доменными правилами.
- Не добавлять в scope экрана внешние блоки (сообщество, футер с внешними разделами и т.п.), если они не требуются текущей задачей.

### Change Summary
**Problem**
- Экран `/topics` оставался заглушкой и не покрывал MVP-сценарий рабочей темы: отсутствовали зависимости, checklist с изменением статусов, материалы и серверный пересчет прогресса.

**Root Cause Hypothesis**
- Не был реализован topic workspace UI-компонент и API-контур для темы.
- Отсутствовал server-state слой, который принимает мутации checklist и возвращает пересчитанный `status/progress`.
- В `globals.css` не было layout/стилей для структуры `hero + dependencies + checklist + materials`.

**Evidence**
- До изменений `frontend/app/(private)/topics/page.tsx` рендерил placeholder.
- Новый workspace UI реализован в `frontend/src/components/topic-workspace-view.tsx`:
  - hero с датами и progress (`topic-date-grid`, progress track),
  - блок dependencies,
  - checklist с изменением статуса и последующим reload из сервера,
  - блок materials с отображением `position` и `progress`.
- Server-state логика вынесена в `frontend/src/lib/topic-workspace-store.ts`:
  - пересчет прогресса `calculateProgress`,
  - вычисление статуса `calculateStatus`,
  - мутация checklist `updateChecklistItemStatus`.
- API:
  - `GET /api/topics/[topicId]` — `frontend/app/api/topics/[topicId]/route.ts`,
  - `PATCH /api/topics/[topicId]/checklist/[itemId]` — `frontend/app/api/topics/[topicId]/checklist/[itemId]/route.ts`.
- Runtime smoke подтвердил server-state обновления: `0 -> 33` и финально `100% + completed`.

**Solution**
- Подключён полноценный экран темы:
  - `frontend/app/(private)/topics/page.tsx` теперь рендерит `TopicWorkspaceView`.
- Добавлен client-компонент `TopicWorkspaceView`:
  - загрузка темы по `topicId` из query-параметра (`/topics?topicId=...`);
  - обработка `loading/error/success`;
  - отображение `title/description/status/progress`, ключевых дат и зависимостей;
  - редактирование статусов checklist (`todo/in_progress/done`) через `PATCH`;
  - после мутации выполняется reload темы для синхронизации с server state;
  - материалы рендерятся в порядке `position` с прогрессом по каждому материалу.
- Добавлен mock server-state хранилище тем и checklist/materials в `frontend/src/lib/topic-workspace-store.ts` (включая темы из roadmap id-набора).
- Добавлены API маршруты темы и checklist-мутаций с валидацией входных данных.
- Добавлены стили topic workspace в `frontend/app/globals.css` с responsive-поведением.

**Tests / Verification**
- Запущено:
  - `npm run lint` — PASS.
  - `npm run build` — PASS (после повторного запуска вне sandbox из-за `spawn EPERM`).
  - Runtime smoke `GET/PATCH/GET` для одной checklist-мутации — PASS (`0 -> 33`, `not_started -> in_progress`).
  - Runtime smoke полного выполнения checklist — PASS (`status=completed`, `progress=100`).
- Не запускалось локально:
  - Ручной браузерный UI-smoke в `npm run dev` (desktop/mobile визуальная проверка интеракций).

**Impact**
- API: добавлены новые frontend endpoints:
  - `GET /api/topics/[topicId]`,
  - `PATCH /api/topics/[topicId]/checklist/[itemId]`.
- UI: `/topics` теперь рабочий экран темы с зависимостями, checklist и материалами.
- Данные: прогресс темы вычисляется серверным состоянием из checklist и не редактируется напрямую на frontend.
- Обратная совместимость: да, изменения изолированы в модуле темы.

**Risks & Rollback**
- Риск: текущий server-state реализован как in-memory mock; после рестарта процесса данные checklist откатываются к начальным.
- Риск: backend-контракт может отличаться от текущей формы payload (id полей, статусы, вложенность).
- Rollback plan:
  - вернуть `frontend/app/(private)/topics/page.tsx` к placeholder-состоянию;
  - удалить `frontend/src/components/topic-workspace-view.tsx`,
    `frontend/src/lib/topic-workspace-types.ts`,
    `frontend/src/lib/topic-workspace-store.ts`,
    `frontend/app/api/topics/[topicId]/route.ts`,
    `frontend/app/api/topics/[topicId]/checklist/[itemId]/route.ts`;
  - убрать topic workspace стили из `frontend/app/globals.css`.
