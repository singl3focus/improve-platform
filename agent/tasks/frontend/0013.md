## TSK-0013 — Персонализировать dashboard: приветствие по `full_name` и дата `Сегодня dd.mm.yyyy`
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-15
- Updated: 2026-03-15

### Context
- Owner запросил добавить на dashboard персональный блок: приветствие с `full_name` и текущую дату в формате `Сегодня X.X.X`.
- Сейчас header dashboard статичный и не использует данные пользователя.
- Функция зависит от доступности `full_name` в сессии/пользовательском API.

### Acceptance Criteria
- [x] На dashboard отображается приветствие с именем пользователя: `Привет, <full_name>`.
- [x] Под приветствием отображается дата формата `Сегодня dd.mm.yyyy` (пример: `Сегодня 15.03.2026`).
- [x] Значение `full_name` берется из реального auth/session источника, без hardcoded строки.
- [x] Если имя временно недоступно (ошибка/пустое значение), отображается безопасный fallback без падения UI.
- [x] Изменение не ломает загрузку и интерактивность существующих dashboard-виджетов.

### Constraints
- Не использовать mock-имя пользователя.
- Формат даты фиксированный `dd.mm.yyyy` для этой задачи.
- Не смешивать с общей локализацией интерфейса и переключением темы (это отдельная задача).

  - Result: NOT RUN (локально не запускался, нужен живой auth backend для полного сценария с реальным `full_name`).

### Change Summary
**Problem**
- Dashboard не показывал персонализацию пользователя: приветствие по имени отсутствовало, дата в формате `Сегодня dd.mm.yyyy` не выводилась.
- Auth user-модель в frontend сессии не включала `full_name`, из-за чего UI не мог безопасно взять имя из session-источника.

**Root Cause Hypothesis**
- В `DashboardView` не было запроса к auth/session для user-профиля.
- `AuthUser` и `extractUser(...)` сохраняли только `id/email`, без `full_name`.
- В dashboard отсутствовал отдельный верхний персональный блок.

**Evidence**
- `frontend/src/components/dashboard-view.tsx`:
  - добавлен запрос `useDashboardResource("/api/auth/session")`;
  - добавлены `greetingLabel` (`Привет, <full_name>` / fallback `Привет!`);
  - добавлен `todayLabel` в формате `Сегодня dd.mm.yyyy`;
  - добавлен рендер персонального блока `dashboard-personal`.
- `frontend/src/lib/auth/contracts.ts`: `AuthUser` расширен полем `full_name?: string`.
- `frontend/app/api/auth/_shared.ts`: `extractUser(...)` теперь извлекает `full_name`/`fullName` и возвращает в формате `full_name`.
- `frontend/app/globals.css`: добавлены стили `dashboard-personal*`.
- Команды проверки:
  - `npm run lint` -> PASS.
  - `npm run build` -> PASS (после повторного запуска вне sandbox).

**Solution**
- Расширен frontend auth-профиль сессии:
  - `AuthUser` теперь поддерживает `full_name`;
  - BFF `extractUser` маппит имя из backend payload (`full_name` или `fullName`).
- В `DashboardView` добавлен персональный блок сверху:
  - приветствие берется из реального `/api/auth/session`;
  - при наличии имени показывается `Привет, <full_name>`;
  - при пустом имени/ошибке session — безопасный fallback `Привет!`.
- Добавлена дата в фиксированном формате `Сегодня dd.mm.yyyy` через `Intl.DateTimeFormat("ru-RU", { day: "2-digit", month: "2-digit", year: "numeric" })`.
- Существующие dashboard-виджеты (`hero`, `grid`, клики, retry, loading/error/empty`) оставлены без изменения поведения.

**Tests / Verification**
- Запущено:
  - `npm run lint` — PASS.
  - `npm run build` — PASS (`Compiled successfully`, `Generating static pages (26/26)`).
- Техническая оговорка:
  - первый `npm run build` в sandbox завершился `spawn EPERM`; повторный запуск вне sandbox прошёл успешно.
- Не запускалось:
  - ручной e2e в браузере с авторизацией и проверкой отображения реального `full_name`.

**Impact**
- UI/UX: на dashboard добавлен персональный заголовок с приветствием и текущей датой.
- Auth/session data: frontend теперь переносит `full_name` в `user` из auth session-ответа.
- API-контракты и URL не изменены (`/api/auth/*` и backend пути остаются прежними).
- Обратная совместимость: да; при отсутствии имени работает fallback без падения UI.

**Risks & Rollback**
- Риск: если backend не возвращает `full_name` в session payload, всегда будет fallback `Привет!` (без персонального имени).
- Риск: без ручного e2e не подтвержден визуальный результат в реальной авторизованной сессии.
- Rollback plan:
  - удалить персональный блок и вычисления `greetingLabel/todayLabel` из `dashboard-view.tsx`;
  - вернуть `AuthUser` и `extractUser(...)` к версии без `full_name`;
  - удалить добавленные `dashboard-personal*` стили из `globals.css`.
