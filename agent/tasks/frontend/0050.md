## TSK-0050 — Отображать все ошибки модальных окон внутри самих окон
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Owner подтвердил правило: все ошибки, возникающие в конкретном окне (модалке), должны отображаться внутри этого окна, а не на фоне страницы.
- Сейчас часть ошибок рендерится на уровне страницы после секции модалки, что ухудшает UX и контекст ошибки.

### Acceptance Criteria
- [x] Для create-модалок задач и материалов ошибки создания, возникающие в этих окнах, отображаются внутри соответствующего окна.
- [x] Ошибка `mutationError` не показывается фоновым баннером страницы, пока открыта create-модалка, в которой произошла ошибка.
- [x] После закрытия create-модалки её локальные ошибки сбрасываются и не переносятся в другой контекст.
- [x] Сценарии update/delete, которые выполняются вне модалок в текущем UI, продолжают работать без регрессий отображения ошибок.
- [x] UX поведения кнопок окна не ломается: повторная отправка, закрытие, отмена.
- [x] Шрифт приложения не меняется.

### Constraints
- Не менять backend-коды ошибок.
- Не скрывать критические ошибки загрузки страницы/экрана, не связанные с конкретным окном.
- Сохранить существующие тексты ошибок, если не требуется обратное.

### Change Summary
**Problem**
- Ошибки мутаций, возникающие в create-модалках задач и материалов, рендерились отдельным блоком на фоне страницы, а не внутри активного окна.

**Root Cause Hypothesis**
- `mutationError` был общий для экрана и отображался вне модального контейнера; у модалок не было собственного рендера ошибок и явного сброса ошибки при закрытии.

**Evidence**
- До изменения `mutationError` выводился только на уровне страницы в `frontend/src/components/tasks-kanban-view.tsx` и `frontend/src/components/materials-library-view.tsx`.
- В `frontend/src/components/hooks/use-tasks-board-view-model.ts` и `frontend/src/components/hooks/use-materials-library-view-model.ts` отсутствовал публичный метод сброса `mutationError` для жизненного цикла модальных окон.
- В текущем UI `update/delete` не используют отдельные модальные окна: удаление задачи идёт через `window.confirm` в `frontend/src/components/tasks-kanban-view.tsx`, а редактирование/удаление материалов выполняется в inline-форме/кнопках в `frontend/src/components/materials-library-view.tsx`.
- Команды проверки:
  - `Set-Location frontend; npm run lint` → PASS (`No ESLint warnings or errors`).
  - `Set-Location frontend; npm run build` → PASS (`Compiled successfully`).

**Solution**
- Добавлен `clearMutationError` в оба view-model хука (`tasks` и `materials`) и возвращён из хука.
- В обоих create-модалках добавлен рендер `mutationError` внутри модального контента.
- В обоих экранах фоновый error-блок для `mutationError` теперь рендерится только когда create-модалка закрыта.
- В обработчиках открытия/закрытия модалок (включая `Escape` и успешный submit) добавлен вызов `clearMutationError`, чтобы локальные ошибки не переносились между контекстами.
- В рамках ReMake синхронизированы Acceptance Criteria и Verification под фактический UI-scope: модальное требование относится к create-потоку; update/delete остаются вне модалок.

**Tests / Verification**
- `Set-Location frontend; npm run lint` — PASS.
- `Set-Location frontend; npm run build` — PASS.
- Manual UI — PASS:
  - `/tasks` create: ошибка отображается внутри модалки, без фонового дубля; после закрытия/повторного открытия сброшена.
  - `/materials` create: ошибка отображается внутри модалки, без фонового дубля; после закрытия/повторного открытия сброшена.
  - update/delete вне модалок: проверены рабочие сценарии удаления задачи и inline update/delete материала.

**Impact**
- API/контракты backend: без изменений.
- Данные/миграции: без изменений.
- Безопасность: без изменений.
- UX: ошибки create-потока, возникшие в активной create-модалке, видны в контексте окна и не дублируются фоновым баннером при открытой модалке.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: если ошибка мутации появится при открытой модалке не из её действия, она также отобразится внутри окна (ожидаемое безопасное поведение для текущего потока).
- Rollback: вернуть изменения в `frontend/src/components/tasks-kanban-view.tsx`, `frontend/src/components/materials-library-view.tsx`, `frontend/src/components/hooks/use-tasks-board-view-model.ts`, `frontend/src/components/hooks/use-materials-library-view-model.ts`.
