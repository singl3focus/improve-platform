### Context
- Owner утвердил объединение в одну задачу визуальных правок private-интерфейса.
- Текущая реализация имеет расхождения с целевым UI:
  - в календарной ленте Dashboard остается пустое место справа от последнего видимого дня;
  - в логотипе присутствуют подзаголовок App/Приложение, цветная метка и рамка;
  - блок заголовка "Дорожная карта обучения ..." оформлен с границей, в отличие от более "чистого" приветственного блока Dashboard;
  - действие "Выйти" находится в header-кнопке, а не под "Настройки";
  - часть кнопок визуально растягивается шире текста;
  - базовый шрифт интерфейса не Montserrat.

### Acceptance Criteria
- [x] На Dashboard блок дней календаря распределяет видимые дни равномерно на всю доступную ширину секции без пустой области справа от последнего показываемого дня.
- [x] В брендинге sidebar отображается только название платформы без подзаголовка App/Приложение, без цветной точки/иконки и без рамки вокруг блока логотипа.
- [x] Header-блок "Дорожная карта обучения ..." визуально без границ (border/outline), в стиле "чистого" приветственного блока Dashboard.
- [x] Действие "Выйти" перенесено под кнопку/пункт "Настройки" в нижней части sidebar и представлено как текстовый action (не button-стиль из header).
- [x] В рамках этой задачи кнопки в затронутых экранах (private shell + dashboard/roadmap изменяемые элементы) имеют ширину по содержимому текста и не растягиваются шире текста без необходимости.
- [x] Базовый шрифт интерфейса переключен на Montserrat (fallback допускается), и это применено консистентно в private-layout и связанных экранах.

### Change Summary
**Problem**
- Private shell и dashboard/roadmap имели визуальные расхождения с согласованным UI: лишние элементы брендинга, logout в header, border в roadmap-header, неравномерная лента календаря и локальные случаи растягивания кнопок.

**Root Cause Hypothesis**
- Причина в исходной верстке и CSS-правилах: бренд-блок содержал дополнительные элементы, logout был реализован как header-button, календарные элементы имели фиксированную минимальную ширину, roadmap-header имел явный border, часть кнопок в grid/flex-контейнерах не имела принудительного `fit-content`.

**Evidence**
- `frontend/src/components/layout/private-shell.tsx`: бренд содержал `brand-mark` + подзаголовок, а logout-кнопка находилась в `<header className="header">`.
- `frontend/app/globals.css`: `.dashboard-calendar-day` использовал `flex-shrink: 0; min-width: 60px;`, `.roadmap-header` имел `border: 1px solid ...`, для части roadmap-кнопок не было явного `width: fit-content`.
- Проверка командами: `npm run lint` и `npm run build` прошли успешно.

**Solution**
- Упрощен sidebar-брендинг до одного названия платформы (без подзаголовка и декоративной метки).
- Logout перенесен в нижний блок sidebar под "Настройки" как текстовый action (`sidebar-text-action`), удален из header.
- Для dashboard-календаря настроено равномерное распределение карточек дней по ширине контейнера на desktop; на mobile оставлен горизонтальный скролл с фиксированной читаемой шириной дня.
- Для roadmap-header убран border, сохранен общий визуальный стиль блока.
- Для затронутых roadmap-кнопок добавлены ограничения ширины по контенту (`width: fit-content`, `justify-self: start` для grid-кейсов).
- Базовый шрифт интерфейса переключен на Montserrat через `next/font/google` + обновлен fallback в глобальном `body`.

**Tests / Verification**
- Запущено: `Set-Location frontend; npm run lint` — PASS.
- Запущено: `Set-Location frontend; npm run build` — PASS.
- Ручной UI smoke не запускался локально в браузере в рамках этой итерации; рекомендован запуск по сценариям из секции Verification.
