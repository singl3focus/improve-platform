## TSK-0051 — Разделить верх секции материалов на 2 блока по требованию owner
- Status: Finished
- Type: Business
- Approval: Approved
- Created: 2026-03-16
- Updated: 2026-03-16

### Context
- Owner запросил разделение текущего общего блока материалов на два:
  - верхний без рамок с текстом `Личная библиотека материалов`;
  - нижний с рамками, содержащий поисковую строку и фильтры/управление.
- Сейчас header и фильтры объединены в один рамочный блок.

### Acceptance Criteria
- [x] В секции `/materials` верхняя часть разделена на два визуально разных блока согласно требованию.
- [x] Первый блок без рамки содержит заголовок `Личная библиотека материалов` и связанный подзаголовок.
- [x] Второй блок с рамкой содержит поиск и фильтры (и action-кнопку, если она относится к control-зоне).
- [x] Существующие сценарии поиска, фильтрации и открытия окна создания материала не ломаются.
- [x] Шрифт приложения не меняется.

- Не объединять эту задачу с переработкой Kanban или roadmap.

### Change Summary
**Problem**
- В верхней части `/materials` заголовок и controls (поиск/фильтры/кнопка) были объединены в один рамочный блок, что не соответствует требованию owner о двух отдельных визуальных блоках.

**Root Cause Hypothesis**
- Компонент `MaterialsLibraryHeader` рендерил и заголовок, и control-зону внутри одного контейнера `panel materials-library-header`.

**Evidence**
- Структура в `frontend/src/components/materials-library-view.tsx` до изменения содержала единый контейнер header с классом panel.
- Стили в `frontend/app/globals.css` применялись к `.materials-library-header` как к единому блоку.
- Команды проверки:
  - `Set-Location frontend; npm run lint` → PASS (`No ESLint warnings or errors`).
  - `Set-Location frontend; npm run build` → PASS (`Compiled successfully`).

**Solution**
- В `MaterialsLibraryHeader` разделена разметка на два блока:
  - безрамочный верхний `materials-library-intro` (заголовок + подзаголовок),
  - нижний рамочный `panel materials-library-controls-panel` (поиск, фильтры, кнопка создания).
- В `frontend/app/globals.css` обновлены селекторы и layout:
  - `.materials-library-header` переведён на grid-раскладку верх/низ,
  - типографика заголовочного блока перенесена на `.materials-library-intro h2/p`,
  - добавлен `.materials-library-controls-panel` для рамочного control-блока,
  - mobile-правило сохранено (адаптивная геометрия не ухудшена).

**Tests / Verification**
- `Set-Location frontend; npm run lint` — PASS.
- `Set-Location frontend; npm run build` — PASS.
- Manual UI — NOT RUN (рекомендована ручная проверка в браузере desktop/mobile).

**Impact**
- API/backend: без изменений.
- Данные/миграции: без изменений.
- Безопасность: без изменений.
- UX: верхняя часть `/materials` соответствует требованию owner о двух визуальных блоках.
- Обратная совместимость: да.

**Risks & Rollback**
- Риск: возможны minor-визуальные отличия от ожидаемого отступа на отдельных брейкпоинтах до ручной UI-проверки.
- Rollback: вернуть изменения в `frontend/src/components/materials-library-view.tsx` и `frontend/app/globals.css`.
