### Context
- Материалы — отдельная сущность и важная часть MVP, не второстепенная.
- Материал принадлежит ровно одной теме, а его прогресс не должен подменять прогресс темы.

### Acceptance Criteria
- [x] Реализован API CRUD материалов с полями `title`, `description`, `topic_id`, `progress`, `position`.
- [x] Один материал может быть привязан только к одной теме; одна тема содержит много материалов.
- [x] Поддерживается изменение порядка материалов внутри темы (`position`).
- [x] Изменение прогресса материала не меняет автоматически прогресс темы.

### Constraints
- Жесткая типизация материалов (книга/курс/видео) не требуется в MVP.
- Проверки принадлежности пользователя обязательны для всех операций.
- Удаление материалов физическое, без soft delete.

### Change Summary
**Problem**
- Отсутствовал API для управления материалами темы (CRUD, порядок, ручной прогресс).

**Root Cause Hypothesis**
- Модуль material не существовал; требовалось создать с нуля по аналогии с task.

**Evidence**
- Таблица `materials` определена в `migrations/000001_initial_schema.up.sql` (строки 104-117), включая CHECK constraint `progress >= 0 AND progress <= 100` и composite FK `(topic_id, user_id) → topics(id, user_id)`.
- До изменений каталог `internal/material/` не существовал.

**Solution**
- Создан модуль `internal/material/` (4 файла):
  - `model.go` — доменная модель `Material`, DTO (`CreateRequest`, `UpdateRequest`, `MaterialResponse`), интерфейсы `Repository` и `Service`, sentinel errors (`ErrMaterialNotFound`, `ErrInvalidProgress`, `ErrTopicNotFound`).
  - `repo.go` — реализация `Repository` через `pgxpool.Pool`. `Create` обрабатывает FK violation (код `23503`) → `ErrTopicNotFound`. `ListByTopic` сортирует `ORDER BY position, created_at`. `Update`/`Delete` возвращают `ErrMaterialNotFound` при `RowsAffected() == 0`.
  - `usecase.go` — реализация `Service`. Валидирует `progress ∈ [0, 100]` перед записью. Не затрагивает прогресс темы.
  - `handler.go` — HTTP-обработчики: `CreateMaterial`, `GetMaterial`, `ListTopicMaterials`, `UpdateMaterial`, `DeleteMaterial`. Валидация: `topic_id` и `title` обязательны при создании, `title` обязателен при обновлении.
- Изменён `internal/server/server.go`:
  - Импорт `improve-platform/internal/material`.
  - Инициализация `matRepo`, `matUC`, `matH`.
  - Новые маршруты (все под JWT middleware):
    - `GET /api/v1/roadmap/topics/{topicID}/materials` — список материалов темы.
    - `POST /api/v1/materials` — создание.
    - `GET /api/v1/materials/{materialID}` — получение.
    - `PUT /api/v1/materials/{materialID}` — обновление.
    - `DELETE /api/v1/materials/{materialID}` — удаление.
- Создано 2 тест-файла:
  - `usecase_test.go` — 12 тестов: CreateMaterial (success, invalid progress ×2, topic not found), GetMaterial (success, not found), ListByTopic (success, empty), UpdateMaterial (success, invalid progress, not found), DeleteMaterial (success, not found).
  - `handler_test.go` — 11 тестов: CreateMaterial (success, missing title, missing topic_id, invalid progress), GetMaterial (success, not found), ListTopicMaterials (success), UpdateMaterial (success, missing title), DeleteMaterial (success), NoAuth.
