## TSK-0010 — Единая обертка ошибок и трассировка `op` между слоями
- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-14
- Updated: 2026-03-14

### Context
- Нужен единый и проверяемый способ прокидывания ошибок между `repo -> usecase -> http`.
- Владелец запросил обязательный паттерн: в каждой функции usecase/repo `const op = "layer.FuncName"` с добавлением в ошибку и трассировкой до логов HTTP-адаптера.

### Acceptance Criteria
- [x] Создан пакет `pkg/errors` с оберткой ошибки, хранением цепочки `op` и поддержкой `errors.Is/errors.As`.
- [x] В каждом новом/изменяемом методе `usecase` и `repo` используется `const op = "<layer>.<FuncName>"`, а возвращаемые ошибки оборачиваются через `pkg/errors`.
- [x] На каждом слое есть единая функция распознавания ошибок нижнего слоя (маппинг типов/кодов без дублирования).
- [x] В HTTP-адаптере при ответе с ошибкой пишется структурированный trace цепочки `op` и исходной причины.

### Constraints
- Не терять совместимость с `errors.Is/errors.As`.
- Ошибки должны быть безопасны: без утечки секретов в response.
- Логировать trace только на сервере; клиенту отдавать нормализованный формат ошибки.

### Change Summary

**Problem**
- По замечанию ревьюера в `auth` и `history` handler-слоях не было единой функции распознавания/маппинга ошибок: ветки обработки были inline и дублировались.

**Root Cause Hypothesis**
- При первичной реализации паттерна `pkg/errors` фокус был на `repo/usecase` и базовом HTTP fallback, из-за чего `auth` и `history` остались с локальной обработкой ошибок внутри каждого endpoint-метода.

**Evidence**
- `backend/internal/auth/handler.go`: до переделки `Register/Login/Me` содержали собственные inline-блоки `if err != nil` с отдельным маппингом и fallback-логированием.
- `backend/internal/history/handler.go`: до переделки одинаковая fallback-обработка была продублирована в двух ветках (`GetEntityHistory` и `GetUserHistory`).
- Reviewer Notes в этой задаче фиксировали оба отклонения как ReMake-блокеры.

**Solution**
- `backend/internal/auth/handler.go`:
  - добавлен единый `handleError(w, err, op authOperation)` для слоя `auth`;
  - добавлен тип `authOperation` с константами `register/login/me`, чтобы централизованно задавать маппинг domain errors и безопасные fallback-сообщения;
  - `Register`, `Login`, `Me` переведены на вызов `handleError(...)` без inline-дублирования.
- `backend/internal/history/handler.go`:
  - добавлен единый `handleError(w, err)` с серверным логированием `apperr.OpsTrace(err)` и нормализованным клиентским ответом;
  - обе error-ветки в `GetHistory` переведены на вызов `handleError(...)`.
- `backend/internal/auth/handler_test.go`:
  - добавлены тесты `TestHandler_Login_UserNotFound`, `TestHandler_Me_NotFound`, `TestHandler_Register_InternalError_LogsOpsTrace`.
- `backend/internal/history/handler_test.go`:
  - обновлён `TestHandler_GetHistory_ServiceError` (проверка fallback + логов с `ops` trace);
  - добавлен `TestHandler_GetHistory_UserHistoryServiceError` для второй ветки handler.

**Tests / Verification**
- `go test -v -count=1 ./...` (из `backend/`) — PASS.
- `go build -buildvcs=false ./...` (из `backend/`) — PASS (exit code 0).
- `go vet ./...` (из `backend/`) — PASS (exit code 0).
- Логирование fallback с `apperr.OpsTrace(err)` подтверждено тестами:
  - `internal/auth`: `TestHandler_Register_InternalError_LogsOpsTrace`;
  - `internal/history`: `TestHandler_GetHistory_ServiceError`.

**Impact**
- Изменены файлы:
  - `backend/internal/auth/handler.go`
  - `backend/internal/history/handler.go`
  - `backend/internal/auth/handler_test.go`
  - `backend/internal/history/handler_test.go`
- API-контракты и коды ответов для известных domain errors сохранены.
- Поведение fallback унифицировано и централизовано на уровне handler-слоя для `auth` и `history`.

**Risks & Rollback**
- Риск низкий: изменения локальны в handler-слое и покрыты unit-тестами.
- Возможный риск: некорректный выбор `authOperation` мог бы изменить fallback message; покрыто тестами на ключевые ветки.
- Откат: вернуть четыре изменённых файла к предыдущей версии.
