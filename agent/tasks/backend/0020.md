## TSK-0020 — Добавить атомарный API для создания темы roadmap с зависимостью от родителя
- Status: Finished
- Type: Technical
- Approval: Approved
- Created: 2026-03-17
- Updated: 2026-03-17

### Context
- Для нового UX фронтенда требуется создание дочерней темы из ноды с немедленной установкой зависимости.
- Два отдельных вызова (`create topic` + `add dependency`) дают риск частичного успеха и усложняют обработку ошибок на UI.

### Acceptance Criteria
- [x] Добавлен backend endpoint для атомарного сценария: создать тему и сразу связать ее зависимостью с родительской темой.
- [x] Операция выполняется транзакционно: при ошибке на шаге зависимости тема не остается созданной частично.
- [x] Endpoint соблюдает текущие ограничения roadmap (валидность stage/topic, запрет self/cycle, user scope).
- [x] Ошибки возвращаются в согласованном формате (`code` + `message`) для корректного отображения на frontend.
- [x] Добавлены/обновлены unit и handler тесты roadmap-модуля для нового endpoint и fail-сценариев транзакции.

### Constraints
- Не ломать существующие endpoint-ы `/api/v1/roadmap/topics` и `/api/v1/roadmap/topics/{topicID}/dependencies`.
- Не менять миграции БД.
- Сохранить обратную совместимость текущих roadmap response-контрактов.

### Change Summary
**Problem**
- Для frontend-сценария создания дочерней ноды требовался атомарный backend API; существующая схема из двух вызовов (`create topic` + `add dependency`) допускала частичный успех.

**Root Cause Hypothesis**
- В roadmap-модуле отсутствовал единый endpoint и транзакционная операция, объединяющая создание темы и добавление зависимости в одну DB-транзакцию.

**Evidence**
- До изменения в `backend/internal/server/server.go` не было отдельного маршрута под атомарный сценарий создания темы с зависимостью.
- До изменения в `backend/internal/roadmap/usecase.go` и `backend/internal/roadmap/repo.go` не было метода, который выполняет create topic + add dependency в рамках одной транзакции.
- После изменения добавлен endpoint `POST /api/v1/roadmap/topics/with-dependency`, usecase-метод `CreateTopicWithDependency` и repo-метод `CreateTopicWithDependency` с `BEGIN/COMMIT/ROLLBACK`.

**Solution**
- Добавлен новый request-контракт `CreateTopicWithDependencyRequest` и расширены интерфейсы `Service`/`Repository` в `backend/internal/roadmap/model.go`.
- Добавлен новый handler `CreateTopicWithDependency` в `backend/internal/roadmap/handler.go`:
  - валидация `stage_id`, `title`, `depends_on_topic_id`;
  - единый ответ в существующем формате ошибок (`code` + `message`) через `handleError`.
- Добавлена usecase-логика в `backend/internal/roadmap/usecase.go`:
  - проверка parent-topic в user scope через `GetTopicByID`;
  - проверка stage в user scope через `GetStagesByUserID`;
  - вызов транзакционного репозиторного метода;
  - возврат `TopicResponse` с заполненной зависимостью на родителя.
- Добавлен транзакционный метод в `backend/internal/roadmap/repo.go`:
  - `CreateTopicWithDependency` использует `tx` и выполняет создание темы + вставку зависимости атомарно;
  - при любой ошибке происходит rollback (включая ошибку шага зависимостей);
  - добавлены проверки существования `stage` и `depends_on_topic` в user scope внутри транзакции.
- Зарегистрирован новый маршрут в `backend/internal/server/server.go`: `POST /api/v1/roadmap/topics/with-dependency`.
- Обновлены тесты roadmap:
  - `backend/internal/roadmap/usecase_test.go`: добавлены тесты success, fail-сценария ошибки зависимости (контракт транзакционного rollback), и проверки stage scope;
  - `backend/internal/roadmap/handler_test.go`: добавлены тесты success, validation error и stage-not-found для нового endpoint.

**Tests / Verification**
- Запущено: `Set-Location backend; go test ./...`.
- Результат: PASS (все пакеты проходят, включая `internal/roadmap`, `internal/server`).
- Manual API smoke: NOT RUN.

**Impact**
- API: добавлен новый endpoint `POST /api/v1/roadmap/topics/with-dependency`.
- Обратная совместимость: да, существующие endpoint-ы `/api/v1/roadmap/topics` и `/api/v1/roadmap/topics/{topicID}/dependencies` не изменены.
- Данные: изменения схемы БД и миграции отсутствуют.
- Безопасность/изоляция: проверяется user scope для `stage` и parent topic.

**Risks & Rollback**
- Риск: новый endpoint добавляет альтернативный путь создания темы; возможны расхождения поведения при дальнейшем развитии старого и нового сценариев.
- Rollback: удалить маршрут из `backend/internal/server/server.go` и откатить изменения в `backend/internal/roadmap/model.go`, `backend/internal/roadmap/handler.go`, `backend/internal/roadmap/usecase.go`, `backend/internal/roadmap/repo.go`, а также соответствующие тесты.
