## TSK-0004 — API roadmap/topics/dependencies с инвариантами DAG
- Status: Finished
- Type: Business
- Approval: Approved

- Updated: 2026-03-14

### Context
- Roadmap является ключевой сущностью продукта и должна быть DAG, а не линейным списком.
- Нужны серверные проверки зависимостей тем и блокировок старта.

### Acceptance Criteria
- [x] Реализован CRUD для roadmap-структуры: этапы (как UI-группы) и темы.
- [x] Реализовано управление зависимостями тем; сервер запрещает создание циклов.
- [x] Перевод темы в активный статус запрещен, если prerequisite темы не завершены.
- [x] API возвращает признак блокировки темы и причины блокировки для UI.

### Constraints
- У пользователя одна roadmap.
- Этап не живет как отдельная бизнес-сущность вне тем (только группировка).
- Доменная валидация обязана быть на backend, а не только на клиенте.

### Change Summary

**Problem**
- В backend-сервисе отсутствовал API для управления roadmap-структурой — ключевой бизнес-сущностью MVP. Без серверной валидации DAG-зависимостей и блокировок тем невозможно обеспечить корректность данных.

**Root Cause Hypothesis**
- Не применимо (greenfield-функциональность; схема БД уже создана в TSK-0002).

**Evidence**
- До начала работы в `internal/` не было модуля `roadmap/`, маршрутизация не включала roadmap-эндпоинты.

**Solution**
Создан доменный модуль `internal/roadmap/` (5 файлов + 2 файла тестов):

Структура:
- `model.go` — доменные модели (`Roadmap`, `Stage`, `Topic`, `TopicDep`), request/response DTO, sentinel errors (9 шт.), интерфейсы `Repository` (17 методов) и `Service` (13 методов).
- `repo.go` — `Repo` на `*pgxpool.Pool`: полный CRUD для roadmaps, stages, topics, topic_dependencies. Обработка unique_violation для roadmap (один на пользователя) и dependency (дубликат). Фильтрация всех операций по `user_id`.
- `usecase.go` — `UseCase`: бизнес-логика включая:
  - **DAG cycle detection**: DFS по графу зависимостей `depMap[topicID] → []dependsOnTopicID`. При добавлении "A depends on B" проверяет, можно ли из B дойти до A — если да, цикл.
  - **Status transitions**: валидная матрица переходов `{not_started→in_progress, in_progress→paused|completed, paused→in_progress|not_started, completed→in_progress}`. Переход в `in_progress` проверяет блокировку.
  - **Blockage computation**: для каждой темы вычисляет `is_blocked` и `block_reasons` по prerequisite-темам со статусом ≠ `completed`.
  - **Auto-dates**: при первом переходе в `in_progress` устанавливает `start_date`; при переходе в `completed` устанавливает `completed_date`.
  - **Full roadmap assembly**: `GetFullRoadmap` загружает roadmap + stages + topics + dependencies и собирает вложенную структуру с blockage info.
- `handler.go` — `Handler` с 13 HTTP-обработчиками и централизованной функцией `handleError` для маппинга domain errors → HTTP status codes.
- `middleware.go` переиспользуется из `internal/auth/`.

Эндпоинты (13, все protected JWT middleware):
- Roadmap: `GET /api/v1/roadmap`, `POST /api/v1/roadmap`, `PUT /api/v1/roadmap`
- Stages: `POST /api/v1/roadmap/stages`, `PUT /api/v1/roadmap/stages/{stageID}`, `DELETE /api/v1/roadmap/stages/{stageID}`
- Topics: `POST /api/v1/roadmap/topics`, `GET /api/v1/roadmap/topics/{topicID}`, `PUT /api/v1/roadmap/topics/{topicID}`, `DELETE /api/v1/roadmap/topics/{topicID}`
- Status: `PATCH /api/v1/roadmap/topics/{topicID}/status`
- Dependencies: `POST /api/v1/roadmap/topics/{topicID}/dependencies`, `DELETE /api/v1/roadmap/topics/{topicID}/dependencies/{depTopicID}`

Формат ответа `GET /api/v1/roadmap`: вложенная структура `roadmap → stages[] → topics[]`, каждый topic содержит `is_blocked`, `block_reasons`, `dependencies`.

Изменённые файлы:
- `internal/server/server.go` — добавлена инициализация roadmap-модуля (`repo → usecase → handler`) и 13 маршрутов в protected-группе Chi.

**Risks & Rollback**
- Риск: DAG cycle detection выполняется в памяти без транзакционной блокировки. При одновременных запросах на добавление зависимостей теоретически возможен race condition. Для MVP однопользовательский сценарий делает это приемлемым; для production рекомендуется транзакция с SELECT FOR SHARE.
- Откат: удалить `internal/roadmap/`, откатить `server.go`.
